# P3: 대기자 리스트 + 사전 런칭 Hype 전략 — 신규 서비스 런칭 때마다 활용

> **작성일**: 2026-03-29 | **우선순위**: P3 | **예산**: 월 0원 | **담당**: 대표

---

## 핵심 컨셉

코스, 부트캠프, 새 도구, 커뮤니티 등 **신규 서비스 런칭 시마다** 대기자 리스트를 먼저 열어 사전 Hype를 만들고, 런칭 당일 폭발적 전환을 유도.

## 대기자 리스트 공식

```
[D-30] "곧 [서비스명]이 나옵니다" 티저
  → 대기자 리스트 랜딩페이지 오픈
  → 이메일 입력 = 얼리버드 혜택 약속

[D-14] "대기자 [X]명 달성!" 사회적 증거
  → FOMO 자극

[D-7] "다음 주 런칭" + 대기자 전용 프리뷰
  → 기대감 극대화

[D-Day] 대기자에게 **먼저** 오픈 (30분~24시간 선공개)
  → 대기자 전환율 극대화

[D+1] 일반 공개 + 대기자 수 공개 ("500명이 기다렸습니다")
```

## 대기자 리스트 인센티브

| 인센티브 | 효과 |
|---------|------|
| 얼리버드 가격 (30% OFF) | 가장 강력한 전환 트리거 |
| 한정 수량 (20명만) | 희소성 |
| 보너스 (프리미엄 템플릿) | 추가 가치 |
| 선공개 접근 | 특별감 |
| 추천 보상 ("3명 추천 → 추가 할인") | 바이럴 루프 |

## Claude Code 자동화

```
대기자 이메일 등록
  → Claude Code → Notion "대기자 DB" 저장
  → 자동 이메일 시퀀스:
    D+0: "등록 감사! 런칭 때 먼저 알려드립니다"
    D+7: "현재 대기자 [X]명! 기대해주세요"
    D-1: "내일 런칭! 준비하세요"
    D-Day: "지금 오픈! 대기자 전용 링크 →"
```

## KPI (런칭당)

| 지표 | 목표 |
|------|------|
| 대기자 수 | 100~500명 |
| 대기자 → 구매 전환율 | 15~25% |
| 런칭 첫 24시간 매출 | 전체 목표의 50%+ |
| 대기자 리스트 바이럴 계수 | 0.3+ |
