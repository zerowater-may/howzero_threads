## 1. 전략 배경 & 리서치 인사이트

### 왜 CRM/리텐션이 P2인가?

P1~P3 전략들이 리드를 쏟아붓고 있다. 그런데 **들어온 리드를 체계적으로 관리하지 않으면 90%가 증발한다.**

```
[현재 리드 유실 포인트]

숏폼 시청 → 구독 (OK)
구독 → 이메일 수집 (... 어디서 관리?)
이메일 구독 → 관심 표현 (... 누가 관심 있는지 모름)
AI 준비도 진단 완료 → (... 점수별 후속 조치?)
오딧 신청 → (... 예약 확인? 리마인더?)
오딧 완료 → (... 유료 제안 타이밍?)
유료 계약 → (... 만족도? 추가 니즈?)
계약 종료 → (... 재계약? 추천?)
```

**CRM 없이 이 퍼널을 운영하면 = 새는 양동이에 물 붓기**

### 핵심 데이터

- **Claude Code 고급 워크플로우로 고객 생애주기 전체 자동화 가능** — 건강점수, 이탈 예측, 업셀 식별까지 (Koanthic, 2026)
- **AI 기반 이탈 예방: 취소 수 주~수 개월 전에 위험 계정 식별, 자동 재참여 워크플로우 트리거** (AI Magicx, 2026)
- **G2 2026 전문가 설문: AI가 이탈 감소의 실행 격차를 좁히고 있다** (G2, 2026)
- **2026년 CRM 자동화: 리드 행동 기반 반복 작업 자동 처리 + 액션 트리거** (RSWebSols, 2026)
- **Agentic AI로 마케팅→리드 검증→개인화 서비스→이탈 방지까지 전자동** (Medium/Predict, 2026.01)

### 하우제로에 CRM이 특별한 이유

하우제로는 **AI 자동화를 파는 회사**다. 자체 CRM을 Claude Code으로 자동화하면:

1. **실전 쇼케이스**: "우리가 직접 Claude Code으로 CRM을 자동화해서 쓰고 있습니다" → 설득력 극대화
2. **케이스 스터디**: 자체 CRM 자동화 과정 자체가 콘텐츠가 됨
3. **템플릿 판매**: 이 CRM 워크플로우를 템플릿으로 판매 가능 (P2-디지털 프로덕트와 연결)

## 2. 리드/고객 생애주기 8단계 설계

### 2.1 전체 생애주기 맵

```
┌──────────────────────────────────────────────────────┐
│  Stage 1: 미인지 (Unknown)                            │
│  → 숏폼/SNS에서 처음 본 사람                          │
│  → 액션: 없음 (트래킹 불가)                           │
├──────────────────────────────────────────────────────┤
│  Stage 2: 인지 (Aware)                               │
│  → 유튜브 구독, SNS 팔로우                            │
│  → 액션: 리드마그넷 CTA 노출                         │
├──────────────────────────────────────────────────────┤
│  Stage 3: 리드 (Lead)                                │
│  → 이메일 수집 완료 (뉴스레터/진단/템플릿)             │
│  → 액션: 웰컴 시퀀스 자동 발송                       │
├──────────────────────────────────────────────────────┤
│  Stage 4: 관심 리드 (Engaged Lead)                   │
│  → 이메일 오픈 3회+ or 템플릿 다운 or 진단 완료       │
│  → 액션: 오딧 CTA 이메일 자동 발송                   │
├──────────────────────────────────────────────────────┤
│  Stage 5: 기회 (Opportunity)                         │
│  → 무료 AI 오딧 신청                                 │
│  → 액션: 예약 확인 + 사전 설문 자동 발송              │
├──────────────────────────────────────────────────────┤
│  Stage 6: 고객 (Customer)                            │
│  → 유료 계약 체결 (컨설팅/코스/구축)                  │
│  → 액션: 온보딩 시퀀스 + 킥오프 안내                  │
├──────────────────────────────────────────────────────┤
│  Stage 7: 활성 고객 (Active Customer)                │
│  → 프로젝트 진행 중 or 코스 수강 중                   │
│  → 액션: 주간 체크인 + 만족도 체크                    │
├──────────────────────────────────────────────────────┤
│  Stage 8: 졸업/종료 (Alumni)                         │
│  → 프로젝트 완료 or 코스 수료                         │
│  → 액션: 후기 요청 + 추천 인센티브 + 재계약 제안      │
└──────────────────────────────────────────────────────┘
```

### 2.2 단계별 자동화 트리거 & 액션

| 단계 | 트리거 | 자동 액션 | 채널 |
|------|--------|----------|------|
| **3→4** (리드→관심) | 이메일 오픈 3회+ / 링크 클릭 / 진단 완료 | 관심 리드 태그 부여 + 심화 콘텐츠 이메일 | 이메일 |
| **4→5** (관심→기회) | AI 준비도 점수 40+ / 오딧 CTA 클릭 | 오딧 예약 페이지 링크 + 한정 슬롯 알림 | 이메일+카카오 |
| **5→6** (기회→고객) | 오딧 완료 후 3일 | 유료 제안서 자동 생성 + 후속 이메일 시퀀스 | 이메일 |
| **6→7** (고객→활성) | 계약 체결 | 온보딩 체크리스트 + 킥오프 미팅 안내 | 이메일+캘린더 |
| **7→8** (활성→졸업) | 프로젝트 완료 / 코스 수료 | 후기 요청 + NPS 설문 + 추천 인센티브 | 이메일 |
| **이탈 위험** | 14일 미오픈 / 오딧 노쇼 / 응답 없음 | 재참여 이메일 + "도움 필요하신가요?" | 이메일+SMS |

## 3. Claude Code CRM 자동화 워크플로우

### 3.1 기술 스택

```
[CRM 자동화 스택 — 전부 무료/저비용]

┌─────────────┐     ┌──────────┐     ┌──────────┐
│   Claude Code        │────▶│  Notion  │────▶│  스티비   │
│ (자동화 엔진) │     │ (CRM DB) │     │ (이메일)  │
└──────┬──────┘     └──────────┘     └──────────┘
       │
       ├──▶ Cal.com (예약 관리)
       ├──▶ Slack/카카오톡 (내부 알림)
       ├──▶ Google Sheets (리포트)
       └──▶ Typeform (설문/진단)
```

| 도구 | 역할 | 비용 |
|------|------|------|
| **Claude Code** (셀프호스팅) | 전체 자동화 오케스트레이션 | 무료 |
| **Notion** | CRM 데이터베이스 (리드/고객/프로젝트) | 무료 |
| **스티비** | 이메일 시퀀스 + 세그먼트 발송 | 30,000원/월 |
| **Cal.com** | 오딧 예약 + 자동 리마인더 | 무료 |
| **Google Sheets** | 주간/월간 KPI 대시보드 | 무료 |
| **Typeform/Tally** | 사전 설문 + AI 준비도 진단 | 무료~30,000원/월 |

### 3.2 핵심 Claude Code 워크플로우 7개

#### WF1: 리드 수집 → Notion CRM 자동 등록

```
[트리거] 스티비 신규 구독 / Typeform 진단 완료 / Cal.com 예약
    → Notion DB에 새 레코드 생성
    → 필드: 이름, 이메일, 유입 채널, AI 준비도 점수, 단계(Stage 3)
    → Slack 알림: "새 리드! [이름] / [유입채널] / 점수 [XX]"
```

#### WF2: 리드 스코어링 자동화

```
[트리거] 매일 09:00 실행
    → 스티비 API: 각 리드의 이메일 오픈/클릭 데이터 수집
    → 스코어 계산:
        이메일 오픈 1회 = +5점
        링크 클릭 1회 = +10점
        AI 준비도 진단 완료 = +20점
        템플릿 다운로드 = +15점
        웹사이트 재방문 = +10점
    → Notion DB 점수 업데이트
    → 점수 50+ → Stage 4(관심 리드) 자동 승격
    → Slack 알림: "핫 리드! [이름] 점수 [XX] → 오딧 CTA 발송됨"
```

#### WF3: 오딧 예약 자동화

```
[트리거] Cal.com 오딧 예약 완료
    → Notion DB Stage 5(기회) 업데이트
    → 자동 이메일: 예약 확인 + 사전 설문 링크
    → D-1 리마인더 이메일 자동 발송
    → D-0 당일 리마인더 (카카오톡 or SMS)
    → 노쇼 시: D+1 "놓치셨나요? 재예약 링크" 자동 발송
```

#### WF4: 오딧 후 유료 전환 시퀀스

```
[트리거] 오딧 완료 표시 (Notion DB 수동 업데이트)
    → D+0: 오딧 리포트 PDF 자동 이메일
    → D+1: "이런 기업도 같은 점수에서 시작했습니다" (사례)
    → D+3: "지금 바로 개선할 수 있는 Quick Win 3가지" (가치 제공)
    → D+5: "유료 컨설팅/구축 서비스 안내" (전환 CTA)
    → D+7: 미전환 시 → "온라인 코스로 직접 해보기" (다운셀)
```

#### WF5: 고객 온보딩 시퀀스

```
[트리거] Notion DB Stage 6(고객) 업데이트
    → D+0: 환영 이메일 + 킥오프 미팅 예약 링크
    → D+1: 온보딩 체크리스트 + 필요 자료 요청
    → D+7: "첫 주 어떠셨나요?" 체크인
    → D+14: 중간 만족도 체크
    → Slack 알림: 대표에게 일정 리마인더
```

#### WF6: 이탈 위험 감지 & 재참여

```
[트리거] 매주 월요일 09:00 실행
    → Notion DB 스캔: 14일 이상 이메일 미오픈 리드
    → 자동 재참여 이메일: "최근에 공유한 AI 자동화 팁, 놓치셨나요?"
    → 30일 미오픈 시: "솔직히 물어볼게요 — 어떤 콘텐츠가 도움이 될까요?"
    → 60일 미오픈 시: 비활성 태그 → 월간 다이제스트만 발송
    → Slack 알림: "[X]명 이탈 위험 리드 재참여 시도 완료"
```

#### WF7: 졸업 후 추천 루프

```
[트리거] 프로젝트 완료 or 코스 수료 (Notion DB 업데이트)
    → D+0: 감사 이메일 + 후기 작성 요청 링크
    → D+3: NPS 설문 (1~10점)
    → NPS 9~10: 자동 추천 인센티브 이메일 (추천인/피추천인 각 혜택)
    → NPS 7~8: "더 개선할 점 있나요?" 피드백 요청
    → NPS 1~6: Slack 긴급 알림 → 대표 직접 연락
    → D+30: "그 후로 어떠세요?" 성과 확인 이메일
    → D+90: 재계약/추가 서비스 제안
```

## 4. Notion CRM 데이터베이스 설계

### 4.1 리드/고객 DB 스키마

| 필드 | 타입 | 설명 |
|------|------|------|
| 이름 | Text | 리드/고객 이름 |
| 이메일 | Email | 연락처 |
| 전화번호 | Phone | (선택) |
| **단계 (Stage)** | Select | 1~8단계 |
| **리드 점수** | Number | 행동 기반 자동 계산 (0~100) |
| 유입 채널 | Select | 유튜브/링크드인/뉴스레터/진단/파트너/검색/광고 |
| AI 준비도 점수 | Number | 자가진단 결과 (0~90) |
| 업종 | Select | 이커머스/서비스/제조/IT/기타 |
| 회사 규모 | Select | 1인/1~10명/10~50명/50명+ |
| 첫 유입일 | Date | 자동 기록 |
| 마지막 활동일 | Date | 자동 업데이트 |
| 오딧 예약일 | Date | Cal.com 연동 |
| 유료 계약 여부 | Checkbox | 전환 추적 |
| 계약 금액 | Number | 매출 추적 |
| NPS 점수 | Number | 졸업 후 수집 |
| 메모 | Text | 대표 수동 메모 |

### 4.2 파이프라인 뷰 (보드 뷰)

```
[미인지] → [인지] → [리드] → [관심 리드] → [기회] → [고객] → [활성] → [졸업]
            │         │          │           │         │         │
            3명       25명       15명        5명       3명       2명    ← 예시 수치
```
