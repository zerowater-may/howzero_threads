# P3: 위기 커뮤니케이션 + 평판 관리 — 부정 리뷰/이슈 대응 플레이북

> **작성일**: 2026-03-29 | **우선순위**: P3 | **예산**: 월 0원 | **담당**: 대표

---

## 핵심 컨셉

하우제로가 성장할수록 부정 리뷰, 오해, 경쟁자 공격, 고객 불만 공개 등 **평판 리스크**가 증가한다. 사전에 대응 플레이북을 만들어두면 패닉 없이 대처 가능.

## 위기 유형 + 대응 매트릭스

| 위기 유형 | 심각도 | 대응 시간 | 대응 방법 |
|----------|:---:|:---:|------|
| 고객 불만 SNS 공개 | 중 | 2시간 | DM 즉시 연락 + 공개 사과 + 해결 |
| 부정 유튜브 리뷰 | 중 | 24시간 | 댓글로 팩트 정정 + 직접 연락 |
| 경쟁사 비방 | 낮음 | 48시간 | 무시 or 팩트 기반 반박 (감정 배제) |
| 서비스 장애/에러 | 높음 | 1시간 | 즉시 공지 + 복구 + 보상 |
| 개인정보 관련 | 높음 | 즉시 | 법률 자문 → 공식 성명 |
| 허위 정보 유포 | 중 | 24시간 | 팩트 증거 공개 + 정정 요청 |

## 대응 원칙 (FACE)

```
F — Fast (빠르게): 2시간 내 1차 응답
A — Acknowledge (인정): "불편을 드려 죄송합니다"
C — Correct (수정): 구체적 해결 방안 제시
E — Exceed (초과 보상): 기대 이상의 대응으로 위기를 기회로
```

## 위기 대응 템플릿

### 고객 불만 SNS 공개 시

```
[DM]
"[이름]님, 불편을 드려 정말 죄송합니다.
 말씀하신 [문제]를 즉시 확인하겠습니다.
 30분 내로 해결 방안을 알려드리겠습니다."

[공개 댓글] (DM 후)
"[이름]님, 소중한 피드백 감사합니다.
 말씀하신 부분을 확인하여 [해결 방안]으로 조치했습니다.
 DM으로 상세 안내드렸으니 확인 부탁드립니다.
 앞으로 이런 일이 없도록 [개선 조치]하겠습니다."
```

### 서비스 장애 시

```
[전 채널 동시 공지]
"현재 [서비스]에 일시적 장애가 발생했습니다.
 원인: [간략 설명]
 복구 예상: [시간]
 영향 받으신 분들께 진심으로 사과드립니다.
 복구 완료 시 다시 안내드리겠습니다."

[복구 후]
"[서비스] 복구가 완료되었습니다.
 원인: [설명] / 재발 방지: [조치]
 이번 장애로 영향 받으신 분들께 [보상] 제공합니다."
```
