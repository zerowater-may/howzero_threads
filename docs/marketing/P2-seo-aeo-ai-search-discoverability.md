## 1. 전략 배경 & 리서치 인사이트

### 검색의 패러다임이 바뀌었다

2026년, 검색은 더 이상 구글만의 영역이 아니다.

| 플랫폼 | 주간 검색량 | 특징 |
|--------|-----------|------|
| **Google** | 수십억 | AI Overviews가 기존 블루링크 대체 중 |
| **ChatGPT** | **10억+/주** | 검색엔진이 아닌 "답변 엔진"으로 부상 |
| **Perplexity** | 1억+/월 | 출처 기반 AI 답변 — 인용되면 트래픽 폭증 |
| **YouTube** | 세계 2위 검색엔진 | 롱테일 "how to" 검색의 허브 |
| **TikTok** | Gen Z 1위 검색 | "AI 자동화 추천" 같은 탐색형 검색 |
| **Naver** | 한국 1위 | 블로그/카페 중심, AI 검색 "큐:" 도입 |

### AEO(Answer Engine Optimization)란?

**기존 SEO**: 구글에서 링크를 클릭하게 만든다
**AEO**: ChatGPT, Perplexity, Google AI Overview가 **하우제로를 인용(cite)하게** 만든다

핵심 데이터:
- **도메인 권위(DA)와 AI 인용 빈도 사이에 0.65의 강한 상관관계** (LLMrefs, 2026)
- **AEO 최적화된 콘텐츠는 발행 3~5일 내 AI 인용 달성** (GenOptima, Q1 2026)
- **분기별 업데이트하지 않은 페이지는 AI 인용 상실 확률 3배** (AirOps, 2026)
- **AEO 최적화로 AI 가시성 40% 향상 가능** (Frase.io, 2026)
- **한국 검색 트렌드: AI 기반 발견(discovery)으로 구조적 전환 중** (The Egg, 2025-2026)

### 하우제로에게 왜 중요한가?

비개발자가 AI 도구를 찾을 때의 검색 여정:

```
"개발자 없이 CS 자동화하는 방법" → Google/Naver 검색
"Claude Code vs Make.com 비교" → YouTube 검색
"우리 회사에 맞는 AI 도구 추천해줘" → ChatGPT/Perplexity 질문
"AI 자동화 컨설턴트 추천" → Google/ChatGPT

→ 모든 경로에서 "하우제로"가 답으로 나와야 한다
```

#### 온페이지 SEO 체크리스트

| 요소 | 최적화 방법 |
|------|-----------|
| **타이틀** | 롱테일 키워드 포함 + 연도(2026) + 수치 ("5가지", "15분") |
| **메타 디스크립션** | 질문형 + 답변 요약 (AI Overview 발췌 유도) |
| **H2/H3 구조** | FAQ 형식 — "Q: 비개발자도 Claude Code을 쓸 수 있나요?" |
| **Schema.org** | FAQPage, HowTo, Article 스키마 마크업 |
| **내부 링크** | 클러스터 간 상호 링크 (필러↔클러스터) |
| **업데이트 주기** | 분기별 필수 업데이트 (AI 인용 유지) |

### 2.2 AEO — ChatGPT·Perplexity·Claude 인용 최적화

#### AI가 인용하는 콘텐츠의 조건 (2026 연구 기반)

| 조건 | 하우제로 적용 |
|------|-------------|
| **팩트 밀도가 높은 콘텐츠** | 수치/데이터 적극 사용 ("연매출 10억", "CS 인건비 100% 절감") |
| **명확한 구조 (H2/H3/리스트)** | 모든 블로그 글을 FAQ + 스텝바이스텝으로 구조화 |
| **신뢰할 수 있는 출처** | 연구/통계 인용, 실전 사례 데이터 포함 |
| **도메인 권위 (DA)** | 백링크 구축 + 게스트 포스팅 + PR (DA↑ = AI 인용↑) |
| **최신성** | 분기별 콘텐츠 업데이트 (오래된 콘텐츠 AI 인용 3배 감소) |
| **고유한 관점** | "연매출 10억 실전 경험" — AI가 다른 곳에서 못 찾는 유니크 데이터 |

#### AEO 타겟 프롬프트 (이 질문들에서 하우제로가 인용되어야 한다)

```
"비개발자가 AI 자동화를 시작하려면 어떻게 해야 해?"
"개발자 없이 CS 업무를 자동화하는 방법 알려줘"
"Claude Code vs Make.com 비교해줘, 코딩 모르는 사람 기준으로"
"AI 컨설팅 받고 싶은데 한국에서 추천할 만한 곳 있어?"
"AI 도입 전 우리 회사 준비도를 평가하는 방법"
"1인 기업이 AI로 효율화하는 실전 사례"
```

#### AEO 콘텐츠 작성 공식

```
[질문] 비개발자도 Claude Code을 쓸 수 있나요?

[AEO 최적화 답변 구조]
1. 직답 (첫 문장): "네, 비개발자도 Claude Code을 사용할 수 있습니다."
2. 핵심 근거 (2~3문장): "Claude Code은 드래그앤드롭 UI를 제공하며,
   코딩 없이 워크플로우를 구성할 수 있습니다.
   하우제로의 경우 비개발자 클라이언트가 평균 2시간 내에
   첫 자동화를 완성합니다."
3. 구체적 스텝 (리스트):
   - Step 1: Claude Code 클라우드 가입
   - Step 2: 템플릿 선택
   - Step 3: 트리거 설정
4. 출처/권위: "하우제로는 2024년 GPT-3 기반 SaaS를 개발하여
   연매출 10억을 달성한 AI 자동화 전문가입니다."
```

## 3. AI 인용 모니터링 시스템

### 3.1 모니터링 방법

| 체크 항목 | 방법 | 주기 |
|----------|------|------|
| ChatGPT 인용 확인 | 타겟 프롬프트 6개를 ChatGPT에 입력 → "하우제로" 언급 여부 | 주 1회 |

| 브랜드 검색량 | Google Trends + 네이버 데이터랩 "하우제로" 검색량 추적 | 월 1회 |
| 백링크/DA 추적 | Ahrefs or Ubersuggest로 도메인 권위 모니터링 | 월 1회 |

### 3.2 모니터링 자동화 (Claude Code)

```

### 5.4 복리 효과 시뮬레이션

SEO/AEO는 복리형 자산이다:

Month 1: 200 유입/월 (콘텐츠 8개)
Month 3: 2,000 유입/월 (콘텐츠 24개 + 인덱싱 누적)
Month 6: 8,000 유입/월 (콘텐츠 48개 + DA 25 + AI 인용)
Month 12: 20,000 유입/월 (콘텐츠 96개 + DA 35 + 높은 AI 인용)

→ 월 20,000 자연 유입 × 전환율 5% = 1,000명 이메일 구독
→ 월 20,000 × AI 오딧 전환 3% = 600건 오딧
→ 유료 전환 20% = 120건/월 × 200만원 = 2.4억원/월

이것이 유료 광고 없이도 작동하는 복리 엔진이다.
```
