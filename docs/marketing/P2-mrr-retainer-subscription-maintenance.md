# P2: 월정액 유지보수 + 리테이너 구독 — 프로젝트형에서 반복 수익(MRR)으로

> **작성일**: 2026-03-29 | **우선순위**: P2 (높음 — 비즈니스 안정성의 핵심)
> **기간**: 2026 Q3 (첫 구축 고객 완료 후 즉시)
> **예산**: 월 5만원 (결제 인프라)
> **담당**: 하우제로 대표

## 1. 전략 배경

### 프로젝트형의 한계

```

       ₩200만     ₩0    ₩500만
```

```
     ┃              ████████████
```

### 핵심 데이터

- **컨설팅 리테이너: 고객이 월 $500~$5,000 고정 지불** — B2B 단일 리테이너가 MRR에 가장 큰 영향 (CozmosLabs)
- **성공적인 자동화 프랙티셔너 = 하이브리드 모델**: 구축 고정가 + 월간 최적화 리테이너 + 시간당 컨설팅 (GetAIPerks, 2026)
- **MRR = 신규 + 확장 - 축소 - 이탈**: 핵심은 이탈 최소화 (Ecosire)
- **구독 모델의 진짜 가치: 이탈률만 관리하면 매출이 자동으로 쌓임** (Ordway Labs)

### 왜 유지보수/리테이너인가?

하우제로가 구축한 자동화 시스템은 **지속적 관리가 필요**:
- API 변경 → 워크플로우 수정 필요
- 비즈니스 프로세스 변경 → 자동화 업데이트 필요
- 새로운 자동화 추가 니즈 발생
- 성과 모니터링 + 최적화
- 트러블슈팅 + 에러 대응

**고객은 이 관리를 혼자 못 한다 → 월정액으로 맡긴다 → MRR**

## 2. MRR 상품 설계

### 2.1 유지보수 리테이너 3-티어

| | 🥉 베이직 | 🥈 프로 ⭐ | 🥇 프리미엄 |
|---|---|---|---|
| **가격** | ₩300,000/월 | ₩600,000/월 | ₩1,200,000/월 |
| **워크플로우 모니터링** | 주 1회 자동 체크 | 일 1회 자동 체크 | 실시간 모니터링 |
| **에러 대응** | 48시간 내 | 24시간 내 | 4시간 내 |
| **워크플로우 수정** | 월 2회 | 월 5회 | 무제한 |
| **새 자동화 추가** | ❌ | 월 1개 (소규모) | 월 2개 |
| **성과 리포트** | 월간 이메일 리포트 | 주간 리포트 + 월간 통화 | 주간 리포트 + 격주 통화 |
| **전용 슬랙/카톡** | ❌ | ✅ 카카오톡 채널 | ✅ 전용 슬랙 채널 |
| **분기 전략 리뷰** | ❌ | ❌ | ✅ 분기 1회 전략 미팅 |
| **타겟** | 소규모 자동화 (1~3개) | 중규모 (3~5개) | 대규모 (5개+) |

### 2.2 컨설팅 리테이너 (자문형)

구축 없이 **지속 자문**만 원하는 고객용:

| | 어드바이저 |
|---|---|
| **가격** | ₩500,000/월 |
| **포함** | 월 2회 화상 미팅 (각 30분) + 이메일 무제한 Q&A |
| **내용** | AI 자동화 전략 자문, 도구 추천, 로드맵 검토 |
| **타겟** | 자체 팀이 실행하지만 전문가 자문이 필요한 기업 |

### 2.3 구독형 템플릿 + 커뮤니티 (소액 MRR)

| | 하우제로 플러스 |
|---|---|
| **가격** | ₩49,000/월 |
| **포함** | 월 3개 신규 Claude Code 템플릿 + 모든 기존 템플릿 접근 + 프리미엄 커뮤니티 |
| **타겟** | 직접 자동화를 하고 싶지만 매달 새 아이디어/템플릿이 필요한 사람 |

→ **P3-유료 커뮤니티(Skool)와 통합 가능**

## 3. 프로젝트 → 리테이너 전환 퍼널

```
[컨설팅/구축 프로젝트 완료]
    ↓
[D+0] "프로젝트 완료 축하합니다! 앞으로 이 시스템이
       지속적으로 잘 작동하려면 관리가 필요합니다."
    ↓
[D+3] "3가지 유지보수 옵션을 안내드립니다" (3-티어 제안)
    ↓
[D+7] "첫 달 50% 할인으로 체험해보세요" (첫 달 프로모)
    ↓
[리테이너 계약]
    월 자동 결제 시작
    ↓
[매월] 모니터링 + 리포트 + 수정
    → 가치를 지속 제공 → 이탈 방지
    ↓
[분기] 리뷰 미팅 (프로 이상)
    → 새로운 자동화 니즈 발굴 → 업셀
```

### 전환율 목표

```
프로젝트 완료 고객 중 리테이너 전환: 40%+

근거:
  - 고객은 시스템이 멈추면 큰 손실 (의존도 높음)
  - 혼자 관리할 기술력 없음 (비개발자)
  - 월 30~60만원 = 직원 인건비 대비 1/10
  - 첫 달 50% 할인 = 진입 장벽 제거
```

## 4. MRR 성장 시뮬레이션

```
[월별 MRR 성장 — 보수적 시나리오]

Month 1: 프로젝트 2건 완료 → 리테이너 1건 (₩600K)
  MRR: ₩600,000

Month 3: 누적 프로젝트 6건 → 리테이너 3건
  MRR: ₩1,800,000

Month 6: 누적 프로젝트 15건 → 리테이너 7건 (이탈 1건)
  MRR: ₩3,600,000

Month 12: 누적 프로젝트 30건 → 리테이너 14건 (이탈 3건)
  MRR: ₩7,200,000
  ARR: ₩86,400,000 ← 리테이너만으로 연 8,640만원

[이탈률 5%/월 가정]
  → 매월 신규 2건 + 이탈 0.5건 = 순증 1.5건
  → 24개월 후 MRR: ~₩15,000,000 (연 1.8억)
```

### MRR + 프로젝트 합산 매출

```
12개월 후:
  프로젝트 매출: 월 ₩6,000,000 (3건/월)
  리테이너 MRR: 월 ₩7,200,000
  기타 (코스/스토어/커뮤니티): 월 ₩3,000,000
  ──────────────────
  총 월매출: ₩16,200,000 (연 ₩1.94억)

→ MRR 비중: 44% ← 안정적 기반
→ 프로젝트가 0건이어도 MRR로 ₩7,200,000 보장
```

## 5. 이탈 방지 전략

### 5.1 이탈 원인 & 대응

| 이탈 원인 | 발생 확률 | 대응 |
|----------|----------|------|
| "필요 없어졌다" (자동화 안정) | 중간 | 매월 성과 리포트로 가치 가시화. "이번 달 ₩240만 절감" |
| "비싸다" | 중간 | ROI 리포트 제공. "₩60만 투자 → ₩240만 절감 = ROI 300%" |
| "직접 할 수 있을 것 같다" | 낮음 | "직접 하다 문제 생기면 복구 비용이 리테이너의 3배" 프레이밍 |
| "회사 사정 (예산 삭감)" | 낮음 | 베이직(₩30만)으로 다운그레이드 옵션 → 이탈보다 축소가 낫다 |

### 5.2 이탈 방어 자동화

```
[Claude Code 이탈 감지 워크플로우]

매주 월요일 체크:
    → 리테이너 고객 중 "30일 내 문의/요청 0건" 감지
    → 자동 이메일: "최근 자동화 시스템 잘 작동하고 있나요?
       이번 달 성과 리포트 보내드립니다."
    → 미응답 7일 → Slack 알림 → 대표 직접 전화

[해지 요청 시]
    → "해지 전 한 번만 통화할 수 있을까요?" (15분)
    → 이유 파악 → 가능하면 다운그레이드 제안
    → 불가피 시 → 깔끔한 해지 + "언제든 돌아오세요" + 3개월 후 체크인
```

### 5.3 업셀 자동화

```
[월간 리포트에서 업셀 기회 감지]

자동화 사용량 증가 → "워크플로우 추가가 필요해 보입니다"
에러 빈도 증가 → "프로 플랜의 일일 모니터링이 도움될 수 있습니다"
새 프로세스 발견 → "이것도 자동화 가능합니다. 견적 드릴까요?"

→ 리테이너 내 자연 업셀 = 추가 프로젝트 수주
→ 기존 고객에게 팔기가 신규보다 5배 쉬움
```

## 6. 기술 인프라

### 6.1 모니터링 시스템 (Claude Code)

```
[Claude Code 자동 모니터링 워크플로우]

매일/매주 실행 (티어별):
    → 고객 Claude Code 인스턴스 health check
    → 워크플로우 실행 로그 확인
    → 에러 감지 → 자동 알림 (Slack/카톡)
    → 주간 성과 집계 (실행 수/절감 시간/에러율)
    → 월간 리포트 자동 생성 (Claude API)
    → 고객에게 이메일 자동 발송
```
