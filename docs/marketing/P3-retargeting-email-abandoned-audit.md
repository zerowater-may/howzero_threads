# P3: 미완료 오딧/진단 리타겟팅 — "거의 다 왔어요!" 복구 시퀀스

> **우선순위**: P3 | **예산**: 월 3만원 | **담당**: Claude Code 자동화

---

## 핵심: AI 진단 시작했지만 완료 안 한 사람, 오딧 페이지 방문했지만 신청 안 한 사람을 복구

## 복구 시나리오

### 1. AI 진단 미완료 복구

```
진단 시작 (이메일 입력) → 5문항 중 3문항에서 이탈
  → D+0 (1시간 후): 이메일 "진단이 중단됐어요! 2분이면 끝납니다 → [이어하기]"
  → D+1: "AI 준비도 결과를 아직 못 보셨네요. 지금 마저 하시면 맞춤 가이드를 드립니다"
  → D+3: 미완료 시 → 일반 뉴스레터 편입
```

### 2. 오딧 페이지 방문 → 미신청 복구

```
오딧 페이지 방문 (Pixel 감지) → 신청 안 함
  → Meta/카카오 리타겟 광고: "무료 AI 오딧 — 이번 달 잔여 3슬롯"
  → (이메일 있으면) D+1: "어제 오딧 페이지를 보셨죠? 혹시 궁금한 점이?"
```

### 3. 코스 페이지 방문 → 미구매 복구

```
코스 페이지 방문 → 구매 안 함
  → 리타겟 광고: "아직 고민 중? 첫 모듈 무료 체험"
  → (이메일 있으면) D+2: "코스 관련 질문 있으시면 답장주세요"
```

## Claude Code 복구 워크플로우

```
[트리거] 진단 미완료 감지 (Typeform partial submission)
  or 오딧 페이지 방문 후 미신청 (GA4 이벤트)
  → 조건 체크: 이메일 있는가?
  → YES → 복구 이메일 시퀀스 자동 발송
  → NO → Meta Pixel 리타겟 오디언스에 추가
```

## 복구율 목표

| 시나리오 | 이탈 비율 | 복구 목표 |
|---------|:---:|:---:|
| 진단 미완료 | 40% | 20% 복구 |
| 오딧 페이지 미신청 | 70% | 10% 복구 |
| 코스 페이지 미구매 | 80% | 5% 복구 |
