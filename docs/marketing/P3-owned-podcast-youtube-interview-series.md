## 1. 전략 배경

### 게스팅 vs 자체 미디어

| | 팟캐스트 게스팅 (P3-PR) | 자체 팟캐스트 (이 기획서) |
|---|---|---|
| 통제권 | 호스트가 통제 | 내가 통제 |
| 오디언스 | 남의 오디언스 빌림 | 내 오디언스 구축 |
| 콘텐츠 자산 | 남의 채널에 남음 | 내 채널에 축적 |
| 관계 구축 | 1회성 | 게스트 초대 → 장기 관계 |
| B2B 세일즈 | 간접 | **게스트를 고객으로 전환** |
| 스케줄 | 호스트에 맞춤 | 내 스케줄대로 |

### 핵심 인사이트

- **"게스트를 인터뷰하면 → 관계가 생기고 → 협업을 탐색하게 된다"** — B2B 팟캐스트의 진짜 가치는 콘텐츠가 아니라 **관계** (Rise25, 2026)
- **1개 인터뷰 = 수십 개 콘텐츠**: 한 대화에서 숏폼, 블로그, 뉴스레터, SNS 포스트 모두 추출 (PodcastVideos, 2026)
- **작은 팟캐스트도 수익화 가능**: 니치 스폰서십 + 직접 파트너십 (BloodyFM, 2026)
- **유튜브 팟캐스트**: 영상+오디오 동시 발행 → 유튜브 SEO + 팟캐스트 플랫폼 동시 노출

### 왜 2027 Q1 런칭인가?

```
2026 Q2~Q3: 게스팅 10회+ 경험 축적 → 인터뷰 스킬 확보
2026 Q3~Q4: 구독자 1,000+, 이메일 500+ → 초기 청취자 확보
2026 Q4: 자체 팟캐스트 사전 제작 (3~5 에피소드)
2027 Q1: ← 런칭 (첫 날부터 3~5개 에피소드 동시 공개)
```

### 2.2 에피소드 유형 3가지

| 유형 | 비중 | 설명 | 예시 |
|------|:---:|------|------|
| **🎙️ 게스트 인터뷰** | 50% | 비개발자 AI 성공 사례 / 업종별 전문가 / 파트너 | "이커머스 셀러가 AI로 월 200시간 아낀 방법" |

| **🔥 핫테이크/트렌드** | 20% | AI 뉴스/트렌드에 대한 역발상 해석 | "GPT-5 나왔다고? 아직 GPT-4도 제대로 못 쓰잖아" |

### 2.3 게스트 전략: "게스트 = 잠재 고객 or 파트너"

B2B 팟캐스트의 핵심: **인터뷰 초대 = 가장 자연스러운 세일즈**

```
[게스트 초대 → 관계 → 전환 퍼널]

Step 1: 타겟 기업 대표/팀장을 게스트로 초대
  "비개발자인데 AI 도입한 경험을 공유해주세요"
  → 거절률 낮음 (자기 홍보 기회)

Step 2: 인터뷰 중 자연스럽게 현황 파악
  "현재 어떤 프로세스가 가장 비효율적이세요?"
  → 사실상 AI 오딧의 첫 질문

Step 3: 에피소드 공개 후 팔로업
  "인터뷰 감사했습니다. 말씀하신 CS 비효율,
   저희가 도움드릴 수 있을 것 같은데 15분 통화 가능하세요?"
  → 이미 30분+ 대화한 관계 → 전환율 극대화

Step 4: 게스트의 네트워크에 노출
  게스트가 에피소드 공유 → 게스트의 팔로워/동료에게 노출
  → 새로운 게스트 후보 + 잠재 고객 유입
```

### 2.4 게스트 타겟 리스트

| 카테고리 | 예시 게스트 | 왜 초대하는가 |
|---------|-----------|-------------|
| **비개발자 AI 성공 사례** | AI 도입한 중소기업 대표 | 사회적 증거 + 잠재 고객 |
| **업종별 전문가** | 이커머스/법률/부동산/병원 경영자 | 니치 콘텐츠 + 업종 확장 |
| **AI 도구 메이커** | Claude Code/Make.com/Lovable 한국 담당 | 도구 권위 + 파트너십 |
| **B2B 파트너** | 강사/코치/컨설턴트 | 크로스 프로모션 |
| **하우제로 고객** | 코스 졸업생/컨설팅 고객 | 케이스 스터디 + 후기 |
| **AI 오피니언 리더** | AI 관련 유튜버/블로거 | 오디언스 교환 |

## 3. 콘텐츠 재가공: 1 에피소드 = 15개+ 콘텐츠

```
[에피소드 1개 녹화 (45분)]
    │
    ├── 유튜브 롱폼 (영상) ──── 1개
    ├── Apple/Spotify (오디오) ── 1개
    ├── 유튜브 쇼츠 ────────── 5~8개 (핵심 클립)
    ├── 틱톡/릴스 ──────────── 3~5개 (리포맷)
    ├── 링크드인 포스트 ─────── 2개 (인사이트 텍스트)
    ├── 뉴스레터 소재 ────────── 1개 (핵심 요약)
    ├── 블로그 포스트 ────────── 1개 (트랜스크립트 기반 SEO)
    ├── 트위터 스레드 ────────── 1개
    └── 게스트 공유용 클립 ───── 2~3개 (게스트가 자발적 공유)

    = 에피소드 1개 → 콘텐츠 15~22개
    = 주 1개 에피소드 → 월 60~88개 콘텐츠
```

### Claude Code 자동 재가공 워크플로우

```
[에피소드 업로드]
    → OpusClip API: 자동 숏폼 5~8개 생성
    → Claude API: 트랜스크립트 → 블로그 포스트 자동 변환
    → Claude API: 트랜스크립트 → 링크드인 포스트 2개 생성
    → Claude API: 트랜스크립트 → 뉴스레터 소재 생성
    → Buffer API: 숏폼 + 포스트 예약 게시
    → Notion: 에피소드 DB에 자동 기록
    → 게스트에게 자동 이메일: 클립 + 에피소드 링크 + 공유 요청
```
