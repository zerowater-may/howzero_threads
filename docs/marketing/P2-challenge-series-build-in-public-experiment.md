# P2: 챌린지/공개 실험 시리즈 전략 — "30일 AI 자동화 챌린지" + Build in Public

> **작성일**: 2026-03-29 | **우선순위**: P2 (높음 — 참여+바이럴+신뢰 동시 달성)
> **기간**: 2026 Q2~Q3 (분기 1회 챌린지)
> **예산**: 월 10만원
> **담당**: 하우제로 대표

## 1. 전략 배경

### 왜 챌린지/공개 실험인가?

2026 틱톡/유튜브 트렌드: **"Reali-TEA" — 완벽한 콘텐츠보다 솔직하고 현실적인 콘텐츠가 폭발** (TikTok Trend Forecast 2026). 챌린지 시리즈는 3가지를 동시에 달성한다:

```
1. 시리즈 = 반복 시청 습관 형성 → 구독/팔로우 자연 증가
2. 챌린지 = 참여 유도 → 커뮤니티 활성화 + UGC 생산
3. Build in Public = 투명성 → 신뢰 폭발 (하우제로 핵심 철학)
```

### 핵심 인사이트

- **"바이럴 1개는 운, 시리즈는 비즈니스"** — 반복되는 시리즈 포맷이 지속 성장의 핵심 (12AM Agency, 2026)
- **30일 챌린지가 틱톡/유튜브에서 가장 높은 참여율** — 일일 업데이트 = 알고리즘 사랑 (TikTok Discover)
- **Build in Public = 수치까지 공개하는 투명성** — Indie Hackers 커뮤니티가 증명한 성장 전략
- **참여형 콘텐츠 = 저장/공유/댓글 동시 극대화** — 알고리즘 최고 신호 (SocialBu)

### 2.3 시리즈 C: "72시간 자동화 스프린트" (라이브 실험)

| 항목 | 설계 |
|------|------|
| **컨셉** | 실제 비즈니스 과제를 72시간 안에 자동화하는 과정을 실시간 공개 |
| **주기** | 분기 1회 |
| **포맷** | Day 1/2/3 각각 숏폼 3~5개 + 최종 롱폼 1개 |
| **예시** | "72시간 안에 CS 완전 자동화 시스템 만들기 — 라이브 타임랩스" |

#### 72시간 스프린트 구조

```
[Day 0] 미션 발표 (숏폼)
  "72시간 안에 이 회사의 CS를 완전 자동화하겠습니다.
   가능할까요? 지켜봐 주세요."

[Day 1] 진단 + 설계 (숏폼 3개)
  - "현재 CS 프로세스 분석 완료"
  - "자동화 포인트 5개 발견"
  - "Claude Code 워크플로우 설계 시작"

[Day 2] 구축 (숏폼 3개)
  - "워크플로우 3개 완성!"
  - "이메일 자동 분류 테스트 — 작동!"
  - "챗봇 연결 중... 에러 발생 😱"

[Day 3] 완성 + 결과 (숏폼 2개 + 롱폼 1개)
  - "에러 해결! 전체 시스템 가동"
  - "Before/After 결과: 주 40시간 → 8시간"
  - [롱폼] "72시간 CS 자동화 풀 과정 (30분)"

→ 극적인 스토리 아크: 미션→난관→해결→성공
→ MrBeast식 "결과를 보여주는" 콘텐츠 구조
```

[보상 체계]

### 3.2 UGC(사용자 생성 콘텐츠) 수확

```
[챌린지 참여자의 UGC]

참여자가 "#하우제로챌린지 Day 15 — 이메일 자동 분류 성공!" 포스팅
    ↓
하우제로가 해당 포스트를 리포스트/인용 (사회적 증거)
    ↓
다른 팔로워: "오 나도 해봐야겠다" (바이럴)
    ↓
참여자의 팔로워: "이게 뭐지?" → 하우제로 발견 (새 유입)

→ 챌린지 참여자 1명 = 하우제로의 무료 마케터
```
