## 1. 전략 배경 & 리서치 인사이트

### "AI 자동화"는 너무 넓다

현재 하우제로의 메시지: "비개발자를 위한 AI 자동화"
→ 좋은 포지셔닝이지만 **너무 범용적**이어서 특정 타겟의 마음을 못 잡는다.

```
[범용 메시지]
"AI 자동화로 업무 효율을 높이세요"
→ 이커머스 사장: "우리 업종에도 되나?"
→ 병원 원장: "의료는 다르지 않나?"
→ 부동산 중개인: "나한테도 해당되는 건가?"
→ 모두 "혹시 나도?"라고 생각하지만 확신이 없음

[니치 메시지]
"이커머스 CS 문의 자동화로 인건비 월 2,000만원 절감"
→ 이커머스 사장: "이거 완전 내 얘기잖아!" → 즉시 전환
```

### 핵심 데이터

- **니치 커뮤니티 퍼널: 범용보다 더 높은 신뢰의 고객을 전달** (BuckleyPlanet, 2026)
- **롱테일 고의도 키워드에 집중하면 권위 구축 + 검증된 리드** (NiftyPM, 2026)
- **AI 도입율 업종별 격차 큼**: 이커머스/리테일(높음) > 부동산/건설(3%) > 의료(4%) > 교육(3%) — **도입율 낮은 업종 = 블루오션** (Lyzr/AI Agent Statistics, 2026)
- **Claude Code 블로그: 15가지 실전 AI 에이전트 사례를 업종별로 분류** — 업종별 콘텐츠가 실제 전환을 만든다 (Claude Code Blog, 2026)
- **도메인 특화 AI가 범용 AI보다 정확하고 효과적** — 업종별 맞춤 솔루션이 2026 트렌드 (Idea Forge Studios)

### 니치 전략의 마법

```
[범용 퍼널]
  유입 1,000명 → 오딧 신청 3% = 30건 → 유료 20% = 6건

[니치 퍼널 (이커머스 특화)]
  유입 300명 (이커머스만) → 오딧 신청 10% = 30건 → 유료 35% = 10.5건

→ 유입이 1/3인데 전환은 1.75배!
→ 같은 노력으로 더 많은 매출
```

## 4. 케이스 스터디 팩토리 시스템

### 4.1 왜 "팩토리"인가?

케이스 스터디는 **가장 강력한 전환 무기**이지만, 대부분의 컨설턴트는 1~2개만 만들고 멈춘다. 하우제로는 **모든 프로젝트를 자동으로 케이스 스터디로 변환하는 시스템**을 구축한다.

### 4.2 케이스 스터디 자동 생산 워크플로우

```
[프로젝트 완료]
    ↓
[자동 트리거] Notion CRM에서 Stage 8(졸업) 감지
    ↓
[Day 0] 자동 이메일: "성과 측정 설문" (5문항)
    1. 자동화 전 해당 업무에 쓰던 시간?
    2. 자동화 후 해당 업무에 쓰는 시간?
    3. 절감된 비용(인건비/외주비)?
    4. 가장 만족스러운 자동화는?
    5. 하우제로를 추천하신다면 한 마디?
    ↓
[Day 3] 설문 수집 → 케이스 스터디 초안 자동 생성 (AI)
    ↓
[Day 5] 대표가 10분 편집 + 고객 승인 요청
    ↓
[Day 7] 승인 완료 → 자동 배포:
    ├── 블로그 게시 (해당 버티컬 랜딩에 링크)
    ├── 뉴스레터 소재로 큐잉
    ├── SNS 카드뉴스 자동 생성
    ├── 유튜브 숏폼 대본 생성
    └── 가격 페이지 사회적 증거로 추가
```
