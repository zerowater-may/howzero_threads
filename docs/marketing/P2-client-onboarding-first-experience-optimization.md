# P2: 고객 온보딩 자동화 + 첫 경험 최적화 — 첫 인상에서 Aha Moment까지

> **작성일**: 2026-03-29 | **우선순위**: P2 (높음 — 전환 후 리텐션의 핵심)
> **기간**: 2026 Q2 즉시
> **예산**: 월 5만원
> **담당**: 하우제로 대표 + Claude Code

---

## 1. 전략 배경

### 왜 온보딩 최적화인가?

고객이 결제한 후 **첫 72시간**이 리텐션을 결정한다. 이 기간에 "이거 돈 값 한다"를 느끼지 못하면 → 환불/이탈/불만족.

```
[온보딩 없이]
  결제 → "그래서 뭐 하면 되죠?" → 혼란 → 불만 → 이탈

[온보딩 있으면]
  결제 → 즉시 웰컴 이메일 → 체크리스트 → 킥오프 미팅 예약
  → D+1 첫 성과 → "와 이거 진짜 되네!" (Aha Moment)
  → 만족 → 추천 → 재구매
```

## 2. 서비스별 온보딩 설계

### 2.1 컨설팅/구축 고객 온보딩 (7일)

```
[D+0: 결제 직후] (자동)
  ✉️ 웰컴 이메일:
    "축하합니다! [패키지명] 시작합니다.
     앞으로 [기간] 동안 이렇게 진행됩니다:"
    → 전체 프로세스 타임라인 인포그래픽
    → 킥오프 미팅 예약 링크 (Cal.com)
    → 사전 설문 링크 (Typeform)
    → 준비물 체크리스트 (Notion 페이지)

  📱 카카오톡 알림:
    "결제 확인! 킥오프 미팅을 예약해주세요 → [링크]"

[D+1: 사전 설문 리마인더] (자동)
  "내일 킥오프 전 이 설문만 작성해주세요 (5분)"
  → 현재 프로세스, 사용 도구, 가장 큰 고통, 목표

[D+2~3: 킥오프 미팅] (대표 직접)
  30분 화상 미팅:
  → 설문 기반 맞춤 진행
  → 프로젝트 범위 확정
  → 일정 + 소통 채널 합의
  → "첫 자동화를 [날짜]까지 보여드리겠습니다" 약속

[D+5: 첫 성과 데모] (대표)
  "약속한 첫 자동화입니다. 직접 보시겠습니다."
  → 라이브 데모 → "이거 진짜 되네!" = Aha Moment

[D+7: 첫 주 체크인] (자동 + 대표)
  ✉️ "첫 주 어떠셨나요?" 이메일
  → 만족도 1~5점 퀵 설문
  → 불만/질문 있으면 즉시 대응
```

### 2.2 셀프페이스 코스 수강생 온보딩 (3일)

```
[D+0: 결제 직후] (자동)
  ✉️ 웰컴 이메일:
    "환영합니다! 코스를 최대한 활용하는 3가지 팁"
    1. "Module 1을 오늘 안에 시청하세요 (15분)"
    2. "커뮤니티에 자기소개 올리기"
    3. "이번 주 목표 1가지 정하기"
    → 코스 바로가기 링크
    → 커뮤니티 참여 링크

[D+1: 시작 독려] (자동)
  "Module 1 보셨나요? 아직이라면 오늘 꼭 보세요!
   첫 모듈만 보면 나머지는 술술 됩니다."

[D+3: 진도 체크] (자동)
  Module 1 미수강 시:
    "아직 시작 안 하셨네요! 바쁘시죠?
     딱 15분이면 됩니다. 지금 시작해보세요 → [링크]"

  Module 1 수강 완료 시:
    "Module 1 완료! 잘하고 계세요 🎉
     Module 2에서 실전 자동화를 직접 만들어봅니다 → [링크]"

[D+7: 첫 주 요약] (자동)
  "첫 주 동안 Module [X]까지 진행하셨네요!
   진도: [X/6] → [프로그레스 바]
   이 속도면 [날짜]에 수료 예정입니다."
```

### 2.3 템플릿/디지털 프로덕트 구매자 온보딩 (즉시)

```
[구매 직후] (자동)
  ✉️ 즉시 이메일:
    "구매 감사합니다! 바로 시작하세요."
    → 다운로드 링크
    → 15분 설치 영상 가이드
    → "설치 중 막히면 이 이메일에 답장하세요"

[D+3] (자동)
  "템플릿 잘 설치하셨나요?
   혹시 어려움이 있으셨다면:
   → FAQ: [링크]
   → 1:1 도움: [카카오톡/이메일]

   더 많은 자동화를 원하시면:
   → 셀프페이스 코스: [링크] (10% 할인 쿠폰 동봉)
   → 무료 AI 오딧: [링크]"
```

## 3. Claude Code 온보딩 자동화 워크플로우

### WF-Onboard: 통합 온보딩

```
[트리거] 결제 완료 (Stripe/토스 Webhook)
    → 상품 유형 판별 (컨설팅/코스/템플릿)
    → 해당 온보딩 시퀀스 자동 시작:

[컨설팅]
    D+0: 웰컴 이메일 (스티비) + 카톡 알림 (Solapi)
    D+1: 설문 리마인더 (미작성 시만)
    D+7: 체크인 이메일

[코스]
    D+0: 웰컴 이메일
    D+1: 시작 독려
    D+3: 진도 체크 (코스 플랫폼 API 확인)
    D+7: 주간 요약

[템플릿]
    D+0: 다운로드 + 가이드 즉시 발송
    D+3: 팔로업 + 업셀 CTA

→ 모든 이메일 → Notion CRM에 발송 이력 자동 기록
```

| 서비스 | Aha Moment | 도달 목표 |
|--------|-----------|----------|
| **컨설팅** | "첫 자동화가 실제로 작동하는 순간" | D+5 이내 |

| **템플릿** | "템플릿 설치 후 첫 자동화가 돌아가는 순간" | 구매 후 30분 이내 |

**원칙: Aha Moment까지의 시간을 최소화**

## 5. 온보딩 → 만족 → 추천 루프

```
[온보딩 완료]
    → Aha Moment 달성
    → "이거 진짜 되네!"
    ↓
[만족]
    → NPS 9~10
    ↓
[자동 추천 트리거] (P3-레퍼럴 연동)
    → "만족하셨다면 동료에게 추천해주세요"
    → 추천 링크 + 보상 안내
    ↓
[추천 → 신규 고객 → 온보딩 → 만족 → 추천]
    = 자기 강화 루프
```
