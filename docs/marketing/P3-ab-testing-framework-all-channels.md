## 핵심: "이게 더 나을 것 같다"가 아니라 "데이터가 이게 낫다고 한다"

## 전 채널 테스트 매트릭스

| 채널 | 테스트 대상 | 측정 지표 | 주기 |
|------|-----------|----------|------|
| **유튜브 롱폼** | 썸네일 A/B (YouTube 기능) | CTR | 매 영상 |
| **숏폼** | 훅 문구 A/B (같은 내용, 다른 첫 3초) | 완시청률 | 주 1회 |
| **이메일** | 제목 A/B (스티비 기능) | 오픈율 | 매 발송 |
| **랜딩페이지** | 헤드라인/CTA A/B (VWO) | 전환율 | 격주 |
| **광고** | 크리에이티브 A/B (Meta) | CPA | 주 2개 |
| **가격** | 표시 방식 (월 vs 총액) | 상담 신청율 | 월 1회 |
| **CTA** | 문구 A/B ("무료 오딧" vs "30분 진단") | 클릭율 | 격주 |
| **캐러셀** | 첫 슬라이드 A/B | 저장율 | 주 1회 |

## 테스트 프로세스 (IDEA 사이클)

```
I — Identify: "이 지표를 개선하고 싶다" (예: 이메일 오픈율)
D — Design: "A 제목 vs B 제목" 설계
E — Execute: 50:50 분할 발송 + 최소 100명/그룹
A — Analyze: 승자 결정 + "왜?"

→ 격주 1개 테스트 = 연 26개 = 지속 개선 엔진
```
