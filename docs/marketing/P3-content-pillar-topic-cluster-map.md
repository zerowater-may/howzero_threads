## 핵심: 모든 콘텐츠가 유기적으로 연결되는 토픽 맵

## 5대 콘텐츠 필러 (Pillar)

```
[Pillar 1] 비개발자 AI 자동화 입문
  ├── AI 자동화란?
  ├── 비개발자가 AI를 쓸 수 있는 이유
  ├── AI vs 단순 자동화 차이
  ├── 시작하기 전 준비할 것
  └── 30일 로드맵

[Pillar 2] AI 도구 가이드
  ├── Claude Code 완전 가이드
  ├── Make.com 가이드
  ├── Zapier 가이드
  ├── ChatGPT/Claude API 활용
  ├── Claude Code vs Make vs Zapier 비교
  ├── 업무별 추천 도구 맵
  └── 무료 vs 유료 도구 비교

[Pillar 3] 업종별 AI 자동화 사례
  ├── 이커머스 자동화 가이드
  ├── 법률사무소 자동화
  ├── 부동산 중개 자동화
  ├── 병원/클리닉 자동화
  ├── 교육/코칭 자동화
  └── 1인 기업 자동화

[Pillar 4] AI 오딧 + 프로세스 최적화
  ├── AI 오딧이란?
  ├── AI 준비도 5차원 프레임워크
  ├── Low-hanging fruit 찾는 법
  ├── ROI 계산법
  ├── AI 도입 실패 사례
  └── 프로세스 맵핑 가이드

[Pillar 5] 하우제로 스토리 + Build in Public
  ├── GPT-3로 10억 SaaS 만든 과정
  ├── 실패 스토리 3가지
  ├── 주간 성과 리포트
  ├── 고객 사례 (케이스 스터디)
  └── AI 트렌드 역발상 해설
```

## 내부 링크 규칙

```
모든 콘텐츠는:
  1. 자신의 Pillar 페이지에서 링크 받음
  2. 같은 클러스터 내 다른 콘텐츠와 상호 링크
  3. 관련 Pillar 간 교차 링크
  4. CTA로 AI 진단 or 오딧 페이지 링크

→ 구글/네이버/AI: "이 사이트는 [비개발자 AI 자동화]에 대해
   체계적이고 권위 있는 정보를 가지고 있다" → 검색 순위 상승
```

## Notion에서 관리

```
[Notion "토픽 맵" DB]

필드:
  - Pillar (1~5)
  - 토픽명
  - URL (발행 시)
  - 상태 (기획/작성/발행)
  - 내부 링크 연결 (Relation)
  - 키워드
  - 월 검색량
  - 현재 순위

→ 어떤 토픽이 빈칸인지 한눈에 파악
→ 빈칸 = 다음 콘텐츠 주제
```
