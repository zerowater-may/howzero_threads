# P3: 콘텐츠 신디케이션 확장 — 외부 플랫폼 재발행으로 노출 면적 극대화

> **우선순위**: P3 | **예산**: ₩0 | **담당**: 대표

---

## 핵심: 1개 블로그를 5~8개 플랫폼에 동시 재발행 (canonical 설정)

## 재발행 플랫폼 목록

| 플랫폼 | 한국/글로벌 | DA | 특징 |
|--------|:---:|:---:|------|
| howzero.com/blog | 메인 | 자체 | canonical 원본 |
| 네이버 블로그 | 한국 | 높음 | 네이버 검색 1위 |
| 브런치 | 한국 | 높음 | 비즈니스 독자 |
| Medium | 글로벌 | 95 | AEO 최강 |
| Dev.to | 글로벌 | 높음 | 개발/테크 커뮤니티 |
| Hashnode | 글로벌 | 높음 | 테크 블로그 |
| LinkedIn Articles | 글로벌 | 99 | B2B 의사결정자 |
| 벤처스퀘어 기고 | 한국 | 높음 | 스타트업 업계 |

## 재발행 규칙

```
1. 원본: howzero.com/blog (항상 먼저 발행)
2. 24시간 후: 네이버 블로그 + 브런치
3. 48시간 후: Medium + Dev.to (영문)
4. 72시간 후: LinkedIn Articles
5. canonical URL: 항상 howzero.com 원본 지정
6. 각 플랫폼 맨 하단: "원본: howzero.com/blog/[slug]"
```

## Claude Code 자동화

```
howzero.com 블로그 발행
  → Claude Code 트리거
  → Claude API: 플랫폼별 포맷 변환 (네이버/브런치/Medium/Dev.to)
  → 24시간 딜레이 → 한국 플랫폼 자동 발행
  → 48시간 딜레이 → 글로벌 플랫폼 자동 발행
  → 각 발행 URL → Notion "신디케이션 DB" 기록
```

## 효과: 1개 블로그 → 7개 플랫폼 = 노출 면적 7배 + DA 백링크 + AEO 인용 확률↑
## KPI: 블로그당 7개 플랫폼 동시 발행, 신디케이션 경유 유입 전체의 30%
## 예산: ₩0 (모든 플랫폼 무료 + Claude Code 기존)
