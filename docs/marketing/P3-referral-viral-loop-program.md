# P3: 레퍼럴(추천) + 바이럴 루프 전략 — 고객이 고객을 데려오는 자동 성장 엔진

> **작성일**: 2026-03-29 | **우선순위**: P3 (중간 — 기존 고객/구독자 확보 후 가동)
> **기간**: 2026 Q3 (7월 설계, 8월 런칭, 9월~ 최적화)
> **예산**: 월 20만원 (보상 비용 포함)
> **담당**: 하우제로 대표 (설계) + Claude Code 자동화 (추적/보상)

## 1. 전략 배경 & 리서치 인사이트

### 왜 레퍼럴인가?

P1~P3 전략이 **새 리드를 획득**한다면, 레퍼럴은 **기존 만족 고객이 새 리드를 무료로 데려온다.** CAC를 0에 가깝게 만드는 유일한 채널.

| 채널 | CAC | 신뢰도 | 스케일 |
|------|-----|--------|--------|
| 숏폼/유튜브 | 중 | 중 | 높음 |
| 유료 광고 | 높음 | 낮음 | 높음 |
| SEO | 낮음 (장기) | 중 | 높음 |
| 파트너/콜라보 | 중 | 높음 | 중 |
| **레퍼럴 (추천)** | **최저** | **최고** | **자기증식** |

### 핵심 데이터

- **추천받은 고객은 비추천 고객보다 30~57% 더 많은 추가 추천을 생성** (Extole, 2026)
- **Dropbox: 양면 보상 레퍼럴로 15개월 만에 3,900% 성장** (4M 사용자)
- **Morning Brew: 레퍼럴 프로그램으로 수백만 뉴스레터 구독자 달성**
- **B2B SaaS 바이럴 계수(K-factor) 0.5 이상이면 강한 수준, 1.0 이상이면 자생적 성장** (IdeaPlan)
- **티어드 보상(단계별 보상)이 단일 보상보다 반복 추천율 높음** (Bloop.plus)

### 왜 Q3인가?

레퍼럴은 **추천할 만한 가치를 경험한 사람**이 있어야 작동한다.

```
Q2: 콘텐츠 시작 → 구독자 확보 → 이메일 리스트 구축
    → 첫 오딧 진행 → 첫 유료 고객 확보
    → "만족한 고객 10명+"이 생긴 시점

Q3: ← 이때 레퍼럴 프로그램 런칭
    만족 고객 + 구독자가 추천할 준비 완료
```

## 2. 레퍼럴 프로그램 설계

### 2.1 양면(Double-Sided) 보상 구조

Dropbox/PayPal이 증명한 원칙: **추천인과 피추천인 모두에게 보상해야 바이럴 루프가 돈다.**

| 서비스 | 추천인 보상 | 피추천인 보상 |
|--------|-----------|-------------|
| **뉴스레터 추천** | 추천 1명당 → 비공개 AI 자동화 가이드 1개 | 가입 시 → Claude Code 템플릿 1개 무료 |
| **AI 준비도 진단 추천** | 추천 3명 완료 → 무료 AI 오딧 30분 | 진단 시 → 맞춤 리포트 |
| **유료 서비스 추천** | 추천인 계약 성사 → 계약금의 **10%** 현금 or 크레딧 | 피추천인 → 첫 달 **20% 할인** |

### 2.2 티어드 보상 (Morning Brew式 마일스톤)

뉴스레터/구독자 추천 기반 — 추천 수에 따라 등급별 보상 해제:

| 추천 수 | 보상 | 심리 트리거 |
|---------|------|-----------|
| **1명** | 비공개 AI 자동화 가이드 PDF (10페이지) | "하나만 추천하면 되네" (낮은 진입 장벽) |
| **3명** | Claude Code 프리미엄 템플릿 팩 (29,000원 상당) | "3명이면 할 만하다" |
| **5명** | 무료 AI 오딧 30분 (대표 직접 진행) | "오딧이 무료? 해볼 만하다" |
| **10명** | 코호트 부트캠프 50% 할인 (245,000원 상당) | "진짜 열심히 추천하면 이걸 받네" |
| **20명** | 1:1 컨설팅 1시간 무료 + 하우제로 앰배서더 인증 | "앰배서더로 소개되다니" (소셜 화폐) |

### 2.3 앰배서더 프로그램

20명 이상 추천한 슈퍼 추천인 = **하우제로 앰배서더**

| 혜택 | 설명 |
|------|------|
| **앰배서더 인증 배지** | 링크드인/유튜브 프로필용 디지털 배지 |
| **앰배서더 전용 채널** | 비공개 디스코드/카카오톡 — 대표 직접 소통 |
| **콘텐츠 공동 제작** | 앰배서더 사례를 유튜브/뉴스레터에 피처 |
| **모든 신규 서비스 Early Access** | 코스, 템플릿, 이벤트 24시간 선공개 |
| **영구 10% 커미션** | 유료 추천 건에 대해 지속 수익 |

## 3. 바이럴 루프 메카니즘

### 3.1 3가지 바이럴 루프

#### Loop 1: 뉴스레터 추천 루프

```
[구독자 A]
    ↓ 뉴스레터 하단 "친구에게 공유" CTA
[추천 링크 공유] (카카오톡/링크드인/이메일)
    ↓
[친구 B가 구독]
    ↓
[A에게 보상 자동 지급] (가이드 PDF)
    ↓
[B도 뉴스레터 받기 시작]
    ↓ 다음 호에서 B에게도 "친구에게 공유" CTA
[B가 C를 추천]
    ↓ 루프 반복
```

#### Loop 2: AI 준비도 진단 바이럴 루프

```
[사용자 A가 진단 완료]
    ↓ 결과 카드 (Spotify Wrapped式) 자동 생성
[SNS에 결과 카드 공유] (링크드인/인스타/카카오톡)
    ↓ "나의 AI 준비도 47점... 😅"
[B가 보고 "나도 해봐야겠다"]
    ↓
[B가 진단 완료 → 결과 카드 공유]
    ↓ 루프 반복

+ 추가 부스터: "친구 3명에게 진단 공유하면 → 무료 오딧"
```

#### Loop 3: 고객 성공 사례 루프

```
[고객 A: 컨설팅/코스 완료]
    ↓ "월 200시간 절감했습니다" (성과)
[하우제로가 사례 콘텐츠로 제작]
    ↓ "A 고객 사례" 유튜브/뉴스레터 게시
[A가 자기 사례 콘텐츠를 자발적으로 공유]
    ↓ "내가 나왔다!" (사회적 증거 + 자발적 바이럴)
[A의 네트워크에서 B, C, D가 유입]
    ↓
[B, C, D가 오딧 신청]
    ↓ 루프 반복
```

### 3.2 바이럴 계수 (K-factor) 목표

```
K = i × c

i = 사용자 1인당 평균 추천 수
c = 추천받은 사람의 전환율

목표 시나리오:
  i = 2 (구독자 1명이 평균 2명에게 추천)
  c = 0.3 (추천받은 사람 30%가 실제 구독/가입)
  K = 2 × 0.3 = 0.6

B2B 기준 K = 0.6은 강한 수준 (벤치마크: 0.5+)
K = 1.0 이상이면 자생적 성장 (유료 마케팅 없이 성장)
```

## 4. 기술 구현

### 4.1 추천 추적 시스템

```
[추천 링크 구조]
  howzero.com/ref/{unique-code}

  예: howzero.com/ref/kim123
  → 쿠키 or UTM으로 추천인 식별
  → 가입/진단/구매 시 추천인 자동 매칭
```

### 4.2 Claude Code 자동화 워크플로우

#### WF1: 추천 링크 자동 생성 & 발급

```
[트리거] 뉴스레터 신규 구독
    → 고유 추천 코드 생성 (이메일 해시 기반)
    → Notion CRM에 추천 코드 저장
    → 웰컴 이메일에 개인 추천 링크 포함
```

#### WF2: 추천 추적 & 보상 자동 지급

```
[트리거] 추천 링크 경유 신규 가입 감지 (Webhook)
    → 추천인 Notion DB에 추천 수 +1 업데이트
    → 마일스톤 체크 (1명/3명/5명/10명/20명)
    → 마일스톤 도달 시 → 자동 보상 이메일 발송
      "축하합니다! 3명 추천 달성 → Claude Code 프리미엄 템플릿 팩 다운로드 링크"
    → Slack 알림: "[김OO] 5명 추천 달성! 무료 오딧 자동 제안됨"
```

#### WF3: 월간 추천 리더보드

```
[트리거] 매월 1일 09:00
    → Notion DB에서 월간 추천 수 TOP 10 추출
    → 리더보드 이미지 자동 생성
    → 뉴스레터에 "이번 달 추천왕" 코너로 삽입
    → TOP 3에게 특별 감사 이메일 자동 발송
```

#### WF4: 이탈 추천인 재활성화

```
[트리거] 매주 월요일 09:00
    → 추천 활동이 30일+ 없는 추천인 식별
    → "새로운 보상이 추가되었습니다!" 재참여 이메일
    → 현재 추천 수 + 다음 마일스톤까지 남은 수 알림
```

### 4.3 기술 스택

| 도구 | 역할 | 비용 |
|------|------|------|
| Claude Code (셀프호스팅) | 전체 자동화 | 무료 |
| Notion | 추천 데이터 + CRM 연동 | 무료 |
| 스티비 | 추천 보상 이메일 발송 | 기존 구독 |
| Vercel | 추천 랜딩페이지 + 리다이렉트 | 무료 |
| Google Sheets | 리더보드 + 통계 | 무료 |

## 5. 추천 촉진 터치포인트

### 5.1 어디에서 추천을 유도하는가?

| 터치포인트 | CTA 카피 | 타이밍 |
|-----------|---------|--------|
| **뉴스레터 하단** | "이 뉴스레터가 도움 됐다면 → 친구에게 공유하기 [추천 링크]" | 매주 |
| **AI 준비도 결과 화면** | "결과를 공유하고 동료와 비교해보세요 [공유 버튼]" | 진단 직후 |
| **오딧 완료 후** | "이 오딧이 도움 됐다면, 같은 고민 하는 분에게 소개해주세요 [추천 링크]" | 오딧 D+1 |
| **코스 수료 후** | "수료를 축하합니다! 동료 추천 시 다음 코스 20% 할인 [추천 링크]" | 수료 즉시 |
| **프로젝트 완료 후** | "만족하셨다면, 추천 1건당 10% 크레딧을 드립니다 [추천 링크]" | 완료 D+3 |
| **NPS 9~10 응답 시** | "높은 점수 감사합니다! 주변에 추천하시면 [보상] 드립니다" | NPS 제출 직후 (자동) |
| **유튜브 영상 마지막** | "이 영상이 도움 됐다면, 코딩 모르는 친구에게 공유해주세요" | 매 영상 |

### 5.2 추천 메시지 템플릿 (공유용)

추천인이 복사해서 보낼 수 있는 사전 작성 메시지:

```
[카카오톡/문자용]
"나 요즘 AI 자동화 뉴스레터 보는데 진짜 도움 돼.
코딩 모르는 사람도 바로 적용할 수 있는 팁들이야.
여기서 구독해봐 → {추천 링크}"

[링크드인용]
"비개발자를 위한 AI 자동화 실전 뉴스레터를 발견했습니다.
GPT-3로 연매출 10억 달성한 하우제로 대표가 직접 씁니다.
매주 비개발자도 바로 쓸 수 있는 자동화 팁 1개씩.
구독 → {추천 링크}"

[이메일용]
"제가 구독하는 AI 자동화 뉴스레터인데 추천드립니다.
비개발자 기준으로 쉽게 풀어서 실전에 바로 적용 가능해요.
구독하시면 Claude Code 자동화 템플릿도 무료로 받을 수 있습니다.
→ {추천 링크}"
```

### 7.4 일반 CAC vs 레퍼럴 CAC 비교

```
일반 채널 평균 CAC: 200,000원 (광고+콘텐츠+시간)
레퍼럴 CAC: 67,000원 (보상 비용만)
→ 레퍼럴이 66% 저렴
→ 게다가 추천 고객의 LTV가 30~57% 더 높음 (Extole)
→ 레퍼럴 LTV:CAC = 일반 대비 3~5배 효율
```
