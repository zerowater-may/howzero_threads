## 1. 전략 배경 & 리서치 인사이트

### 왜 가격/패키징 전략이 P2인가?

12개 기획서가 리드를 만들고 있다. 그런데 **리드가 돈을 내는 순간의 설계**가 없으면 매출은 0이다.

```
[현재 퍼널의 마지막 1마일]

숏폼 → 구독 → 이메일 → 진단 → 오딧 → ???
                                        ↑
                              "얼마예요?"
                              "뭘 받는 건데요?"
                              "왜 이 가격인데요?"

→ 이 질문에 대한 답이 가격/패키징 전략
```

### 핵심 데이터

- **앵커링 효과**: 최고가 티어를 먼저 제시하면 중간 티어가 합리적으로 느껴짐 — 컨설팅펌이 $120K 리테이너를 먼저 보여준 후 $40K 프로젝트 제안 시 카운터가 더 높아짐 (BrodNeil, 2026)
- **3-티어 가격 구조가 가장 높은 전환율** — 중간 티어를 "추천"으로 표시하면 60~70%가 선택 (Simon-Kucher)
- **Aha Moment까지 빨리 도달시키고 그 다음 페이월** — 너무 많이 무료로 주면 전환 안 됨 (Growth Unhinged, 2026)
- **프리미엄 전환율 벤치마크: B2B SaaS 2~5% 일반, 5~7% 강함** (IdeaPlan)
- **프리미엄 전환은 장기 게임 — 6개월 이상 무료 사용자도 전환** (Stripe, RevenueCat)
- **컨설팅 레이트 카드 2026: Professional $12K, Enterprise $25K+** (Data-Mania)

## 2. 하우제로 서비스 가격 체계 (Price Architecture)

### 2.1 가격 사다리 (Price Ladder)

비개발자 고객이 "무료"에서 시작해 자연스럽게 올라가는 구조:

```
┌───────────────────────────────────────────────────┐
│  $0                    무료 콘텐츠 + 리드마그넷    │
│  ──────────────────────────────────────────────── │
│  $0                    무료 AI 오딧 30분           │ ← Aha Moment
│  ──────────────────────────────────────────────── │
│  ₩9,000~49,000         디지털 프로덕트             │ ← 소액 첫 결제
│  ──────────────────────────────────────────────── │
│  ₩99,000              셀프페이스 코스              │ ← 자기주도 학습
│  ──────────────────────────────────────────────── │
│  ₩490,000             코호트 부트캠프              │ ← 라이브 + 커뮤니티
│  ──────────────────────────────────────────────── │
│  ₩1,000,000~3,000,000  유료 컨설팅/구축            │ ← 맞춤 솔루션
│  ──────────────────────────────────────────────── │
│  ₩3,000,000~           전환 패키지 (풀서비스)      │ ← 최고가
└───────────────────────────────────────────────────┘
```

**핵심 원리**: 각 단계에서 **다음 단계의 가치를 미리 보여준다.** "이거 좋았으면 → 다음 단계는 이것"

### 2.2 3-티어 컨설팅 패키지 (메인 수익원)

가격 심리학의 핵심: **3개 옵션을 제시하면 중간 것을 선택한다** (골디락스 효과)

| | 🥉 라이트 | 🥈 스탠다드 ⭐추천 | 🥇 프리미엄 |
|---|---|---|---|
| **가격** | ₩1,000,000 | ₩2,000,000 | ₩5,000,000 |
| **AI 오딧** | 심화 진단 (2시간) | 심화 진단 (2시간) | 심화 진단 (4시간) |
| **자동화 로드맵** | 3개월 | 6개월 | 12개월 |
| **자동화 구축** | 워크플로우 1개 | 워크플로우 3개 | 워크플로우 5개+ |
| **교육/이관** | 매뉴얼 제공 | 매뉴얼 + 1회 교육 | 매뉴얼 + 3회 교육 + 유지보수 1개월 |
| **후속 지원** | 이메일 2주 | 이메일+화상 1개월 | 전담 지원 3개월 |
| **보너스** | - | 코스 무료 수강권 | 코스 + 부트캠프 + 커뮤니티 VIP |

**앵커링 포인트**: 프리미엄(₩5,000,000)을 먼저 보여줌 → 스탠다드(₩2,000,000)가 합리적으로 느껴짐

**추천 태그**: 스탠다드에 "⭐ 가장 인기" 뱃지 → 60~70% 선택 유도

### 2.3 가격 심리학 적용 7가지

| 기법 | 적용 |
|------|------|
| **1. 앵커링** | 프리미엄(₩5M) 먼저 노출 → 스탠다드(₩2M) 합리적으로 인식 |
| **2. 골디락스** | 3티어 중 중간이 "추천" → 대부분 중간 선택 |
| **3. 프레이밍** | "월 ₩167,000" (₩2M÷12) vs "₩2,000,000" — 월 기준으로 표시 |
| **4. 손실 회피** | "지금 수동으로 쓰는 시간 × 시급 = 월 ₩300만 낭비 중" — 안 사면 손해 |
| **5. 사회적 증거** | "A 기업은 이 패키지로 월 200시간 절감" — 패키지 옆 사례 배치 |
| **6. 희소성** | "이번 달 스탠다드 3슬롯만 가능" — 한정 수량 |
| **7. ROI 프레이밍** | "₩2M 투자 → 연간 ₩3,600만 절감 = ROI 1,700%" — 비용이 아니라 투자 |

## 3. 무료→유료 전환 퍼널 설계

### 3.1 Aha Moment 정의

**Aha Moment**: 리드가 "이건 진짜 가치 있다"고 느끼는 순간. 이 순간 직후에 유료 제안을 해야 전환이 일어난다.

| 서비스 | Aha Moment | 타이밍 |
|--------|-----------|--------|
| **무료 AI 오딧 30분** | "우리 회사에 이런 비효율이 있었구나!" | 오딧 리포트 수령 시 |
| **AI 준비도 진단** | "내 점수가 이렇게 낮다고?!" | 결과 화면 즉시 |
| **무료 Claude Code 템플릿** | "이걸로 진짜 자동화가 됐다!" | 첫 워크플로우 실행 성공 시 |
| **웨비나 라이브 데모** | "와, 진짜 코딩 없이 되네" | 데모 시연 중 |
| **셀프페이스 코스 Module 1** | "이 사람 진짜 실력자네" | 첫 모듈 수료 시 |

### 3.2 Aha Moment → 유료 전환 시퀀스

#### 시퀀스 A: 무료 오딧 → 유료 컨설팅

```
[무료 AI 오딧 30분 완료]
    ↓ Aha Moment: "비효율 발견!"
    ↓
[D+0] 오딧 리포트 PDF 이메일
    "진단 결과: 5개 비효율 포인트 중 3개는 즉시 개선 가능합니다."
    → 리포트 마지막 페이지: 3-티어 패키지 안내
    ↓
[D+1] "Quick Win 1개를 직접 해볼 수 있는 가이드"
    → 무료 가치 제공 (신뢰 강화)
    ↓
[D+3] "이런 기업도 같은 점수에서 시작해서 이렇게 바뀌었습니다"
    → 사례 기반 (사회적 증거)
    → CTA: "3-티어 패키지 중 어떤 게 맞는지 15분 상담"
    ↓
[D+5] "이번 달 스탠다드 패키지 잔여 2슬롯"
    → 희소성 + 손실 회피
    ↓
[D+7] 미전환 시:
    → "직접 해보고 싶으시면 → 셀프페이스 코스 (₩99,000)"
    → 다운셀 (작은 결제부터 시작하게)
```

#### 시퀀스 B: 디지털 프로덕트 → 코스 → 컨설팅 (사다리 업셀)

```
[Claude Code 템플릿 구매 ₩29,000]
    ↓ Aha Moment: "자동화 맛 봤다!"
    ↓
[D+3] "템플릿 잘 쓰고 계신가요? 더 깊이 배우려면 → 코스"
    → 셀프페이스 코스 (₩99,000) 10% 할인 쿠폰
    ↓
[코스 수료]
    ↓ Aha Moment: "직접 해봤는데 한계가 있네"
    ↓
[수료 D+3] "혼자 하기 어려운 부분, 저희가 직접 해드립니다"
    → 컨설팅 패키지 안내 + Alumni 20% 할인
    ↓
[컨설팅 완료]
    ↓
[완료 D+7] "추가 자동화가 필요하시면 → 유지보수 패키지"
    → 월 정기 구독 모델 (₩300,000/월)
```

### 3.3 전환 트리거 체크리스트

오딧/상담 중 이 질문에 "YES"가 나오면 즉시 패키지 제안:

| 트리거 질문 | YES일 때 제안 |
|-----------|-------------|
| "이 자동화를 당장 쓰고 싶다" | 라이트 패키지 |
| "여러 프로세스를 한 번에 바꾸고 싶다" | 스탠다드 패키지 ⭐ |
| "팀 전체가 쓸 수 있게 해주세요" | 프리미엄 패키지 |
| "직접 배워서 하고 싶다" | 코스 + 템플릿 번들 |
| "예산이 부족하다" | 다운셀: 코스(₩99K) or 템플릿(₩29K) |

## 6. 가격 실험 로드맵

### 6.1 A/B 테스트 계획

| 실험 | 가설 | 측정 | 시기 |
|------|------|------|------|
| **3티어 vs 2티어** | 3티어가 중간 선택율을 높인다 | 패키지별 선택 비율 | Q2 |
| **월 표시 vs 총액 표시** | "월 ₩167K"이 "₩2M"보다 전환 높다 | 가격 페이지 CTA 클릭율 | Q2 |
| **ROI 계산기 유무** | 계산기가 있으면 상담 신청 증가 | 계산기 사용 → 상담 전환율 | Q3 |
| **희소성 유무** | "잔여 3슬롯"이 전환을 높인다 | 오딧→유료 전환율 | Q3 |
| **환불 보장 유무** | "30일 환불 보장"이 전환을 높인다 | 패키지 계약율 + 실제 환불율 | Q3 |

```

  - 수요가 공급(대표 시간)을 초과할 때

[신규 티어 추가 시점]
  - "프리미엄 이상"을 원하는 고객이 반복될 때 → 엔터프라이즈 티어
```
