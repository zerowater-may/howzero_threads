## 1. 전략 배경

### 왜 이것이 P1(최우선)인가?

9개 기획서가 생겼다. 그런데 **실행 관제탑이 없으면 아무것도 안 된다.**

```
[현재 상황]
  P1: 숏폼 블리츠 — 주 5개 숏폼 + 주 1개 롱폼
  P1: AI 준비도 진단 — 바이럴 리드마그넷
  P2: 이메일/커뮤니티 — 주 1회 뉴스레터
  P2: B2B 파트너십 — 월 2회 웨비나
  P2: SEO/AEO — 주 2회 블로그
  P2: 코스/디지털 프로덕트 — Q3 런칭
  P2: CRM 자동화 — 리드 생애주기 관리
  P3: 유료 광고 — Q2 후반 시작
  P3: PR/퍼스널 브랜딩 — 팟캐스트/언론

  = 총 10개+ 채널에서 주 15~30개 작업
  = 1명이 운영
  = 관제탑 없으면 확실히 터진다
```

### 핵심 원칙

- **Notion으로 계획, Google Sheets로 측정** — 2개만 쓴다
- **1소스 멀티유즈**: 롱폼 1개 → 숏폼/블로그/뉴스레터/링크드인 자동 파생
- **주 1회 리뷰, 월 1회 전략 조정** — 과도한 분석 금지
- **인풋(게시 수) 추적 우선, 아웃풋(조회수/전환) 추적은 월 단위**

## 2. 통합 콘텐츠 캘린더 설계 (Notion)

### 2.1 Notion 마스터 캘린더 DB 스키마

| 필드 | 타입 | 설명 |
|------|------|------|
| 콘텐츠 제목 | Title | 제목/주제 |
| **상태** | Select | 아이디어 / 기획 / 제작중 / 완료 / 게시됨 |
| **채널** | Multi-select | 유튜브 롱폼 / 쇼츠 / 틱톡 / 릴스 / 링크드인 / 뉴스레터 / 블로그(네이버) / 블로그(웹) |
| **기획서 연결** | Select | P1-숏폼 / P1-진단 / P2-이메일 / P2-파트너 / P2-SEO / P2-코스 / P2-CRM / P3-광고 / P3-PR |
| 콘텐츠 유형 | Select | 롱폼원본 / 숏폼클립 / 재가공 / 오리지널 |
| **원본 콘텐츠** | Relation | 롱폼 → 파생 숏폼/블로그 연결 (1소스 멀티유즈 추적) |
| 예정일 | Date | 게시 예정일 |
| 게시일 | Date | 실제 게시일 |
| CTA | Select | 뉴스레터 구독 / AI 진단 / 오딧 신청 / 코스 / 없음 |
| 훅(Hook) | Text | 첫 5초 대본 |
| URL | URL | 게시된 링크 |
| 조회수 | Number | (월 1회 업데이트) |
| 참여율 | Number | (월 1회 업데이트) |

### 2.2 캘린더 뷰 (5가지)

| 뷰 | 용도 |
|-----|------|
| **📅 주간 캘린더** | 이번 주 게시할 콘텐츠 한눈에 보기 |
| **📊 채널별 보드** | 채널별 파이프라인 (아이디어→기획→제작→게시) |
| **🔗 원본→파생 맵** | 롱폼 1개에서 파생된 콘텐츠 트리 |
| **📋 기획서별 필터** | P1~P3 기획서별 콘텐츠 진행 현황 |
| **✅ 인풋 트래커** | 주간 게시 목표 대비 실적 |

### 2.3 주간 인풋 목표 (전 기획서 통합)

| 채널 | 주간 목표 | 연결 기획서 | 비고 |

### 3.2 LTV:CAC 측정 프레임워크

```
[CAC 계산]
  월 마케팅 총 비용 (광고+도구+시간)
  ÷ 월 신규 유료 고객 수
  = CAC

  예시: 200만원 ÷ 10명 = 200,000원/고객

[LTV 계산]
  평균 계약 금액 × 평균 거래 횟수/년 × 평균 고객 유지 기간(년)
  = LTV

  예시: 200만원 × 2회 × 1.5년 = 600만원

[LTV:CAC]
  600만원 ÷ 200,000원 = 30:1 → 매우 건강

  벤치마크: 최소 3:1, 이상적 5:1+
  (컨설팅은 고단가+저CAC이므로 높은 비율 기대)
```

