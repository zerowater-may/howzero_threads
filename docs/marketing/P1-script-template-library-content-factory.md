## 1. 전략 배경

### 문제: "뭘 찍어야 하지?"

콘텐츠 제작에서 가장 큰 시간 낭비는 **촬영이 아니라 기획**이다.

```
[현재 병목]
  "이번 주 영상 주제 뭘로 하지?" (30분 고민)
  "훅을 어떻게 시작하지?" (20분 고민)
  "대본 구조는?" (40분 작성)
  "CTA를 어떻게 넣지?" (10분 고민)
  = 촬영 전 기획에 1.5시간+

[템플릿 라이브러리 후]
  "이번 주 주제: AI 도구 비교" → 템플릿 #7 선택
  → 빈칸 채우기 5분 → 대본 완성
  → 촬영 시작
  = 기획 5분
```

### 해결: 검증된 대본 구조를 템플릿화

MrBeast가 바이럴 패턴을 스프레드시트로 분석한 것처럼, 하우제로의 모든 콘텐츠 포맷을 **재사용 가능한 템플릿**으로 만든다.

## 2. 대본 템플릿 라이브러리 (20개)

### 2.1 유튜브 롱폼 템플릿 (5개)

#### LF-01: "How To" 튜토리얼

```
[0:00~0:10] 훅: "오늘 {결과}를 {시간} 만에 만드는 법을 보여드리겠습니다"
[0:10~1:00] 문제: "{타겟}이 {고통} 때문에 {현재 상태}입니다"
[1:00~2:00] 약속: "이 영상이 끝나면 {구체적 결과}를 얻습니다"
[2:00~15:00] 스텝: Step 1~5 (각 스텝 2~3분, 화면 녹화)
[15:00~17:00] 결과: Before/After 비교 + 수치
[17:00~18:00] CTA: "{다음 단계}는 {링크}에서"

변수: {결과}, {시간}, {타겟}, {고통}, {현재 상태}, {구체적 결과}
예시: "오늘 CS 자동 분류를 15분 만에 만드는 법을 보여드리겠습니다"
```

#### LF-02: "vs 비교" 리뷰

```
[0:00] 훅: "{도구A} vs {도구B} vs {도구C}, 비개발자라면 뭘 써야 할까?"
[0:30] 비교 기준: "5가지 기준으로 직접 써보고 비교했습니다"
[1:00~5:00] 도구A 실제 화면 시연 + 점수
[5:00~9:00] 도구B 실제 화면 시연 + 점수
[9:00~12:00] 도구C 실제 화면 시연 + 점수
[12:00~14:00] 비교표 정리 + 상황별 추천
[14:00~15:00] CTA: "뭘 골라야 할지 모르겠으면 → {CTA}"
```

#### LF-03: "스토리" 여정 공유

```
[0:00] 훅: "저는 {과거 상태}였습니다. 그런데 {전환점}이 모든 걸 바꿨습니다"
[0:30~3:00] 배경: 과거 상태 + 고통 + 시도한 것들
[3:00~5:00] 전환점: 무엇이 바뀌었는가
[5:00~12:00] 과정: 타임라인 + 시행착오 + 구체적 수치
[12:00~14:00] 현재: 달라진 결과 + 배운 교훈 3가지
[14:00~15:00] CTA: "당신도 시작하려면 → {CTA}"
```

#### LF-04: "역발상" Contrarian

```
[0:00] 훅: "모두가 {통념}이라고 하지만, 그건 틀렸습니다"
[0:30] 통념 설명: "왜 사람들이 이렇게 생각하는지"
[1:30] 반론: "실제로는 {반대 사실}입니다"
[3:00] 증거: 데이터 + 사례 + 경험
[8:00] 대안: "대신 이렇게 해야 합니다"
[12:00] 정리: 핵심 3줄 요약
[13:00] CTA
```

#### LF-05: "리스트" TOP N

```
[0:00] 훅: "{타겟}을 위한 최고의 {카테고리} {N}가지"
[0:30] 선정 기준 설명
[1:00~] 각 아이템 2~3분씩:
  #N: 이름 → 뭔지 → 왜 좋은지 → 누구에게 맞는지 → 화면 시연
[마지막] 1위 발표 + 요약표
CTA
```

### 2.2 숏폼 템플릿 (5개)

#### SF-01: "30초 팁"

```
[0~3초] 텍스트 오버레이: "{충격적 사실/질문}"
[3~20초] 팁 설명 (화면 녹화 or 말하기)
[20~28초] 결과/Before-After
[28~30초] "더 자세한 건 프로필 링크"
```

#### SF-02: "Before/After"

```
[0~3초] "Before:" {비효율 화면}
[3~5초] 전환 효과
[5~20초] "After:" {자동화 화면 + 수치}
[20~25초] "어떻게?" 핵심 1문장
[25~30초] CTA
```

#### SF-03: "반박/역발상"

```
[0~3초] "{통념}" → 큰 X 표시
[3~5초] "실제로는..."
[5~25초] 반론 + 증거 (빠른 템포)
[25~30초] CTA
```

#### SF-04: "이거 아세요?"

```
[0~3초] "이거 아세요?" (호기심)
[3~20초] 놀라운 사실 + 시연
[20~30초] "해보고 싶으면 → 프로필"
```

#### SF-05: "하루 루틴/시스템"

```
[0~5초] "제가 매일 AI로 {X}하는 시스템"
[5~25초] 각 단계 빠른 화면 전환 (3~4단계)
[25~30초] "이 시스템 받으려면 → 프로필"
```

### 2.3 링크드인 포스트 템플릿 (5개)

#### LI-01: "인사이트 + 교훈"

```
오늘 {경험}을 하며 깨달은 것:

{핵심 인사이트 1문장}

{배경 설명 2~3문장}

{구체적 사례 or 데이터}

{결론/교훈}

{질문으로 끝} — "여러분은 어떠세요?"

---
📌 저는 {자기소개 1줄}
🔗 {CTA 링크}
```

#### LI-02: "역발상 핫테이크"

```
{도발적 한 문장}.

(그리고 아마 맞습니다.)

왜냐하면:
1. {이유 1}
2. {이유 2}
3. {이유 3}

대신 이렇게 하세요:
→ {대안}

동의하시나요? 댓글로 알려주세요.
```

#### LI-03: "리스트 + 체크리스트"

```
{타겟}이 반드시 알아야 할 {N}가지:

1. {항목 1} — {한줄 설명}
2. {항목 2} — {한줄 설명}
...
N. {항목 N} — {한줄 설명}

📌 저장해두고 나중에 확인하세요.

---
{자기소개} | {CTA}
```

#### LI-04: "여정 공유 (Build in Public)"

```
{기간} 동안 {한 일} 결과:

📊 숫자:
- {지표 1}: {수치}
- {지표 2}: {수치}
- {지표 3}: {수치}

✅ 잘 된 것: {1가지}
❌ 실패한 것: {1가지}
💡 배운 것: {1가지}

다음 주 계획: {1가지}

{질문}
```

#### LI-05: "사례 스토리"

```
{고객 업종}의 {고객명} 대표님 이야기:

문제: {고통}
해결: {하우제로 서비스}
결과: {수치}

{인용} — {고객}

{교훈}

---
{CTA}
```

## 4. 템플릿 저장/관리

```
[Notion "대본 템플릿 라이브러리"]

  DB 필드:
  - 템플릿 ID (LF-01, SF-03 등)
  - 포맷 (롱폼/숏폼/링크드인/뉴스레터/캐러셀)
  - 이름 ("How To 튜토리얼")
  - 구조 (전체 대본 구조)
  - 변수 목록
  - 사용 횟수 (몇 번 사용했는지)
  - 평균 성과 (조회수/참여율 — 사용 후 기록)
  - 태그 (교육/역발상/비교/스토리/리스트)

→ "이번 주 숏폼 5개" → 템플릿 5개 선택 → 변수 채우기 → 완성
→ 성과 데이터 축적 → "가장 잘 되는 템플릿" 파악 → 반복
```
