# P1: 콘텐츠 리퍼포징(재가공) 자동화 시스템 — 1소스 멀티유즈 Claude Code 파이프라인

> **작성일**: 2026-03-29 | **우선순위**: P1 (최우선 — 전체 콘텐츠 효율의 핵심)
> **기간**: 2026 Q2 즉시 (4월 구축)
> **예산**: 월 10만원
> **담당**: 하우제로 대표 + Claude Code 자동화

## 1. 전략 배경

### 문제: 1인이 10개 채널을 운영해야 한다

하우제로의 콘텐츠 채널:
유튜브 롱폼 / 쇼츠 / 틱톡 / 릴스 / 링크드인 / 뉴스레터 / 네이버 블로그 / 웹 블로그 / 브런치 / Medium / X / Threads

**12개 채널 × 주 2~5개 = 주 30~60개 콘텐츠**

수동으로는 불가능. **자동화 파이프라인이 있어야만 실행 가능.**

### 해결: 1 → N 자동 변환

```
[롱폼 1개 촬영 (30분)]
    ↓ Claude Code 자동 파이프라인
[12개 채널 × 2~5개씩 = 30~60개 콘텐츠 자동 생산]

대표 투입: 촬영 30분 + 검토 30분 = 총 1시간
나머지: 전자동
```

## 2. 리퍼포징 자동화 아키텍처

### 2.1 메인 파이프라인

```
[원본: 유튜브 롱폼 1개]
    │
    ├── [Claude Code WF1] 자동 트랜스크립트
    │   Clova Note or Whisper API → 텍스트 추출
    │
    ├── [Claude Code WF2] 숏폼 자동 클리핑
    │   OpusClip API → 핵심 5~10개 클립 자동 생성
    │   → 유튜브 쇼츠 / 틱톡 / 릴스 각각 포맷팅
    │
    ├── [Claude Code WF3] 텍스트 콘텐츠 자동 변환
    │   Claude API:
    │   ├── 트랜스크립트 → 네이버 블로그 (한국어, 2,000~3,000자)
    │   ├── 트랜스크립트 → 웹 블로그 (SEO 최적화, 영문 옵션)
    │   ├── 트랜스크립트 → 브런치 (에세이 톤)
    │   ├── 트랜스크립트 → Medium (영문)
    │   ├── 트랜스크립트 → 뉴스레터 소재 (핵심 인사이트 500자)
    │   ├── 트랜스크립트 → 링크드인 포스트 2개 (인사이트 + 여정)
    │   ├── 트랜스크립트 → X 스레드 (7트윗)
    │   └── 트랜스크립트 → Threads 포스트 2개
    │
    ├── [Claude Code WF4] 캐러셀 자동 생성
    │   Claude API → 핵심 포인트 7개 추출
    │   → Canva API or 템플릿 → 인스타/링크드인 캐러셀 PDF
    │
    ├── [Claude Code WF5] 예약 게시
    │   Buffer API:
    │   ├── 쇼츠 → 유튜브 예약
    │   ├── 틱톡 → 틱톡 예약
    │   ├── 릴스 → 인스타 예약
    │   ├── 링크드인 → 링크드인 예약
    │   └── X/Threads → 예약
    │
    └── [Claude Code WF6] Notion 캘린더 자동 업데이트
        모든 콘텐츠 → Notion 마스터 캘린더에 자동 기록
        상태: "제작완료" → "예약됨" → "게시됨"
```

## 3. Claude Code 워크플로우 6개 상세

### WF1: 자동 트랜스크립트

```
[트리거] 유튜브 새 영상 발행 감지 (YouTube API)
    → 오디오 추출 (yt-dlp)
    → Whisper API or Clova Note → 텍스트 변환
    → 타임스탬프 포함 트랜스크립트 → Notion DB 저장
    → 다음 워크플로우 트리거
```

### WF2: 숏폼 자동 클리핑

```
[트리거] WF1 완료
    → OpusClip API: 영상 URL 전송
    → AI가 바이럴 가능성 높은 구간 5~10개 자동 선택
    → 각 클립에 자동 자막 생성
    → 쇼츠(9:16) / 틱톡 / 릴스 포맷별 출력
    → 임시 저장 → 대표 5분 검토 → 승인
```

### WF3: 텍스트 콘텐츠 변환

```
[트리거] WF1 완료 (트랜스크립트 준비)
    → Claude API 병렬 호출 (8개 변환 동시):

    프롬프트 예시 (네이버 블로그):
    "이 유튜브 대본을 네이버 블로그 포스트로 변환해줘.
     규칙:
     - 2,000~3,000자
     - 소제목 3~5개 (H2)
     - 이미지 삽입 위치 표시
     - SEO 키워드: [키워드] 자연스럽게 3~5회
     - 마지막에 CTA: AI 준비도 진단 or 오딧 신청
     - 톤: 직설적, 실용적, 과장 없이"

    → 8개 변환 결과 → 각각 Notion DB에 초안 저장
    → 대표에게 Slack 알림: "8개 초안 검토 대기"
```

### WF4: 캐러셀 자동 생성

```
[트리거] WF1 완료
    → Claude API: "이 대본에서 핵심 포인트 7개를 추출해서
       캐러셀 슬라이드 텍스트로 만들어줘"
    → 슬라이드 1: 훅 (질문형)
    → 슬라이드 2~6: 포인트 1개씩
    → 슬라이드 7: CTA + 요약
    → Canva 템플릿에 텍스트 자동 삽입 (Canva API)
    → PDF 출력 → 링크드인/인스타 업로드 대기
```

### WF5: 예약 게시

```
[트리거] 대표가 Notion에서 "승인" 체크
    → Buffer API:
       쇼츠 → 유튜브 예약 (화 10시, 목 10시, 토 10시...)
       틱톡 → 틱톡 예약 (화~토 12시)
       릴스 → 인스타 예약 (화/목/토 18시)
       링크드인 → 링크드인 예약 (화 8시, 목 8시, 토 10시)
    → 스티비 API: 뉴스레터 큐잉 (다음 화요일 08:00)
    → 네이버 블로그: 수동 발행 (자동화 제한) → 알림만
```

### WF6: 캘린더 자동 업데이트

```
[트리거] WF5 완료 (게시 예약)
    → Notion 마스터 캘린더 자동 업데이트:
       - 콘텐츠 제목
       - 채널
       - 예정일/게시일
       - 원본 롱폼 연결 (Relation)
       - 상태: "예약됨"
    → 게시 확인 후 → 상태: "게시됨"
    → 주간 인풋 트래커 자동 카운트
```

## 4. 주간 운영 루틴

```
[화요일: 콘텐츠 배칭 데이]

09:00 롱폼 촬영 (30분)
    ↓
09:30 유튜브 업로드 → Claude Code WF1~4 자동 트리거
    ↓
10:00 (자동 진행 중) 대표는 다른 업무
    ↓
11:00 Slack 알림: "8개 텍스트 초안 + 숏폼 8개 + 캐러셀 2개 준비"
    ↓
11:00~11:30 대표 검토 (30분)
    → 초안 읽고 수정/승인 → Notion에서 "승인" 체크
    ↓
11:30 WF5~6 자동 실행 → 전 채널 예약 게시 완료
    ↓
12:00 완료! 이번 주 콘텐츠 30개 셋업 끝.

총 대표 투입: 촬영 30분 + 검토 30분 = 1시간
자동화 비율: 87%
```
