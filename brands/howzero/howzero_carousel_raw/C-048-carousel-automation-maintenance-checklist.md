> **유형**: 인스타 캐러셀 | **슬라이드**: 8장 | **톤**: 실용+체크리스트

---

## 슬라이드 1 (표지)
```
자동화 유지보수 체크리스트

"세팅했으니 끝"이라고 생각하셨죠?

⚠️ 방치하면 자동화도 고장 납니다
```

---

## 슬라이드 2
```
자동화가 멈추는 3가지 이유

1. API 변경
   → 연동 서비스가 업데이트되면 깨짐

2. 데이터 형식 변경
   → 입력 데이터가 바뀌면 오류 발생

3. 사용량 초과
   → 무료 플랜 한도 도달 시 중단

세팅보다 유지보수가 진짜 실력입니다.
```

---

## 슬라이드 3
```
📋 주간 체크리스트

□ 자동화 실행 로그 확인
□ 에러 발생 건수 확인
□ 처리 건수가 예상 범위인지 확인
□ 알림이 정상 발송되었는지 확인
□ 출력 데이터 품질 샘플 체크

소요 시간: 10~15분
매주 월요일 아침 추천.
```

---

## 슬라이드 4
```
📋 월간 체크리스트

□ 연동 서비스 업데이트 확인
□ API 키 만료 여부 확인
□ 사용량/비용 확인 (한도 체크)
□ 워크플로우 효율 점검 (불필요한 단계 제거)
□ 백업 확인 (데이터 유실 방지)

소요 시간: 30분
매월 첫째 주 추천.
```

---

## 슬라이드 5
```
📋 분기별 체크리스트

□ 자동화 ROI 재계산
□ 새로운 자동화 기회 탐색
□ 기존 워크플로우 최적화
□ 팀 피드백 수집 (불편한 점, 개선 요청)
□ 도구 대안 검토 (더 나은 옵션 등장?)

소요 시간: 1~2시간
분기 첫 주 추천.
```

---

## 슬라이드 6
```
에러 대응 프로세스

에러 발생 시:

1️⃣ 에러 로그 확인 (어디서 멈췄는지)
2️⃣ 입력 데이터 확인 (형식 변경?)
3️⃣ 연동 서비스 상태 확인 (장애?)
4️⃣ 수동 재실행 테스트
5️⃣ 수정 후 모니터링

대부분의 에러는 "데이터 형식 변경"입니다.
```

---

## 슬라이드 7
```
유지보수를 자동화하는 방법

□ 에러 발생 시 슬랙/이메일 자동 알림
□ 실행 성공률 대시보드 세팅
□ 사용량 임계치 알림 설정
□ 주간 리포트 자동 생성

유지보수까지 자동화하면
진짜 "손 안 대는 자동화"가 됩니다.
```
