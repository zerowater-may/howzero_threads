```
팀장이 알아야 할
AI 자동화

직접 코딩할 필요 없습니다
판단할 수 있으면 됩니다

🎯 팀 생산성 2배 만드는 의사결정 가이드
```

---

## 슬라이드 2
```
팀장의 역할은 "도입"이 아니라 "판단"

❌ 팀장이 직접 자동화 세팅
✅ 팀장이 판단하고 지시

판단해야 할 것:
1. 어떤 업무를 자동화할지
2. 어떤 도구를 쓸지
3. 누가 담당할지
4. 성과를 어떻게 측정할지

이 4가지만 결정하면 됩니다.
```

---

## 슬라이드 3
```
팀에서 자동화할 업무 찾는 법

팀원에게 물어보세요:

"지금 하는 업무 중에
가장 반복적이고 지루한 게 뭐야?"

그 답이 자동화 1순위입니다.

조건:
□ 주 2회 이상 반복
□ 규칙이 명확함
□ 사람의 창의적 판단이 불필요
```

---

## 슬라이드 4
```
팀원 설득하는 법

팀원의 걱정: "AI가 내 자리를 뺏는 거 아닌가?"

이렇게 말하세요:
"반복 업무를 AI에 맡기고
너는 더 중요한 일에 집중해"

실제로:
자동화 도입 후 해고된 사례보다
업무 범위가 넓어진 사례가 압도적.
```

---

## 슬라이드 5
```
팀 자동화 도입 순서

1주차: 반복 업무 리스트 작성
2주차: ROI 높은 1개 선정
3주차: 도구 선택 + 담당자 지정
4주차: 세팅 + 테스트
5주차~: 운영 + 모니터링

한 번에 여러 개 하지 마세요.
1개 성공 경험이 팀 전체를 바꿉니다.
```

---

## 슬라이드 6
```
성과 측정 지표

자동화 전/후 비교 지표:

⏱️ 처리 시간 (Before → After)
💰 비용 절감액 (인건비 기준)
📊 처리 건수 (생산성 증가)
❌ 에러율 (실수 감소)
😊 팀원 만족도 (업무 부담 변화)

숫자로 보고해야 윗선도 납득합니다.
```

## 슬라이드 7
```
팀장이 하면 안 되는 3가지

❌ 1. 유행하는 도구부터 사기
→ 문제 정의가 먼저

❌ 2. 전 업무를 한꺼번에 자동화 시도
→ 1개씩 순차적으로

❌ 3. 세팅만 하고 방치
→ 주간 점검 필수

이 3가지만 안 해도
실패 확률이 80% 줄어듭니다.
```
