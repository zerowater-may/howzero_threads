# C-061: 인스타/링크드인 캐러셀 — "자동화 우선순위 매트릭스"

> **유형**: 캐러셀 (9장) | **채널**: 인스타 + 링크드인 | **목적**: 저장 유도 + 실용 프레임워크 제공

---

## 슬라이드 1 (표지)
```
자동화 우선순위 매트릭스

뭐부터 자동화해야 할지 모르겠다면
이 매트릭스 하나로 끝납니다

→ 스와이프
```

---

## 슬라이드 2
```
왜 매트릭스가 필요한가?

자동화를 시작하면
"다 해야 할 것 같은" 충동이 옵니다.

그리고 3개월 뒤
아무것도 완성하지 못합니다.

우선순위 없는 자동화 = 실패.
하우제로가 10번 넘게 겪은 일입니다.
```

---

## 슬라이드 3
```
매트릭스 구조

        효과 큼
          │
  ②       │     ①
  해볼만함  │  지금 당장
          │
─────────┼─────────
          │
  ④       │     ③
  안 함    │  나중에
          │
        효과 작음

가로축: 난이도 (쉬움 → 어려움)
세로축: 효과 (작음 → 큼)
```

---

## 슬라이드 4
```
① 지금 당장 (쉬움 + 효과 큼)

= Low-hanging Fruit

예시:
• 주문 알림 자동화 (30분)
• FAQ 자동 응답 (2시간)
• 리포트 자동 발송 (1시간)
• 리뷰 요청 자동 발송 (1시간)

여기서 시작하세요.
ROI 300%+ 보장되는 영역입니다.
```

---

## 슬라이드 5
```
② 해볼만함 (어려움 + 효과 큼)

= 전략적 투자

예시:
• 전체 워크플로우 자동화 (2주)
• AI 고객 상담 시스템 (1개월)
• 맞춤형 CRM 연동 (2주)

효과는 크지만 시간이 걸립니다.
①을 먼저 끝내고 착수하세요.
```

---

## 슬라이드 6
```
③ 나중에 (쉬움 + 효과 작음)

= 있으면 좋지만 급하지 않음

예시:
• 내부 알림 커스텀
• 파일 정리 자동화
• 내부 캘린더 동기화

한가할 때 하세요.
지금 시간 쓸 곳이 아닙니다.
```

---

## 슬라이드 7
```
④ 안 함 (어려움 + 효과 작음)

= 시간 낭비

예시:
• 사용빈도 낮은 리포트 자동화
• 월 1회 하는 작업 자동화
• 과도한 커스텀 개발

"할 수 있다"와 "해야 한다"는 다릅니다.
이 영역은 과감하게 버리세요.
```

---

## 슬라이드 8 (실전 적용법)
```
실전 적용법

1. 반복 업무 10개 적기
2. 각각 난이도/효과 점수 매기기 (1~5점)
3. 매트릭스에 배치
4. ① 영역에서 딱 1개 선택
5. 이번 주에 완료

10개 중 ①에 해당하는 건
보통 2~3개입니다.
그 2~3개가 전체 효과의 80%입니다.
```
