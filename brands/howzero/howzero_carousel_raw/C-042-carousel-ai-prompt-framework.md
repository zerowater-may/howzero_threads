---

## 슬라이드 2
```
왜 프롬프트가 중요한가?

같은 AI에게:
❌ "보고서 만들어줘" → 엉뚱한 결과
✅ 체계적 프롬프트 → 완벽한 결과

AI는 똑같습니다.
프롬프트가 다를 뿐입니다.
```

---

## 슬라이드 3
```
RACE 프레임워크

R — Role (역할)
A — Action (행동)
C — Context (맥락)
E — Example (예시)

이 4가지만 넣으면 결과가 극적으로 좋아집니다.
```

---

## 슬라이드 4
```
R — Role (역할 지정)

"당신은 10년 경력의 마케팅 전문가입니다"
"당신은 이커머스 CS 매니저입니다"
"당신은 세무사입니다"

역할을 주면 AI가 그 관점에서 답합니다.
전문성 있는 답변이 나옵니다.
```

---

## 슬라이드 5
```
A — Action (구체적 행동)

❌ "보고서 만들어줘"
✅ "이 매출 데이터를 분석해서
    주간 보고서를 만들어줘.
    핵심 요약 3줄 + 지표 표 + 인사이트 3가지 포함."

행동을 구체적으로 = 결과도 구체적으로.
```

---

## 슬라이드 6
```
C — Context (맥락 제공)

"이 보고서는 대표님에게 보고하는 용도입니다"
"우리 회사는 이커머스 기업이고 직원 10명입니다"
"지난주 매출이 20% 떨어졌습니다"

맥락이 있으면 AI가 상황에 맞춰 답합니다.
```

---

## 슬라이드 7
```
E — Example (예시 제공)

"이전에 이렇게 만들었으니 같은 형식으로"
"톤은 이 샘플처럼"
"표는 이런 컬럼으로"

예시를 주면 AI가 정확히 맞춥니다.
예시 1개 > 설명 100줄.
```
