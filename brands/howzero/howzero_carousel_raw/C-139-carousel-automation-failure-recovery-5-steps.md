# C-139: 인스타 캐러셀 — "자동화 실패 후 복구 5단계"
> **유형**: 인스타 캐러셀 | **슬라이드**: 8장
## 슬라이드 1: 자동화 실패 후 복구 5단계 🔧 (저장 필수)
## 슬라이드 2: 자동화는 실패한다 — 100% 성공하는 자동화는 없다. 중요한 건 실패 후 얼마나 빨리 복구하느냐
## 슬라이드 3: Step 1: 영향 범위 파악 — 어디서 멈췄는지, 어떤 데이터가 영향받았는지 즉시 확인. 로그 확인 필수
## 슬라이드 4: Step 2: 수동 백업 전환 — 자동화 중지 → 수동 프로세스로 즉시 전환. 업무 공백 최소화
## 슬라이드 5: Step 3: 원인 분석 — API 변경? 데이터 형식 변경? 도구 업데이트? 근본 원인 1개를 특정하라
## 슬라이드 6: Step 4: 수정 및 테스트 — 원인 제거 후 테스트 데이터로 3회 이상 검증. 실환경 투입 전 반드시
## 슬라이드 7: Step 5: 재발 방지 — 모니터링 알림 추가 + SOP에 장애 사례 기록 + 유사 자동화 점검
