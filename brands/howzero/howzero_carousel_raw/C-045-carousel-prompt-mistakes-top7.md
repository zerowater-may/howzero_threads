# C-045: 인스타 캐러셀 — "프롬프트 실수 TOP 7"

> **유형**: 인스타 캐러셀 | **슬라이드**: 9장 | **톤**: 직설+실용

---

## 슬라이드 1 (표지)
```
프롬프트 실수 TOP 7

당신이 AI를 못 쓰는 이유는
AI가 아니라 질문 방식입니다

⚠️ 7번은 거의 모든 사람이 합니다
```

---

## 슬라이드 2
```
실수 #1. 역할을 안 준다

❌ "마케팅 전략 짜줘"
✅ "당신은 이커머스 10년 경력 CMO입니다.
    월 매출 5억 규모 브랜드의 마케팅 전략을 짜줘."

역할이 없으면 AI는 범용 답변을 합니다.
범용 = 쓸모없는 답변.
```

---

## 슬라이드 3
```
실수 #2. 한 번에 다 시킨다

❌ "사업계획서, 재무모델, 마케팅전략 다 만들어줘"
✅ 1단계: 시장 분석 → 2단계: 전략 → 3단계: 재무

AI도 단계별로 해야 퀄리티가 나옵니다.
한 번에 몰아치면 전부 얕아집니다.
```

---

## 슬라이드 4
```
실수 #3. 맥락을 안 준다

❌ "이메일 써줘"
✅ "B2B SaaS 회사 대표에게 보내는 콜드 이메일.
    상대방은 50인 규모 이커머스 대표.
    우리 제품은 CS 자동화 솔루션."

맥락이 없으면 AI는 추측합니다.
추측 = 틀린 답변.
```

---

## 슬라이드 5
```
실수 #4. 출력 형식을 지정 안 한다

❌ "분석해줘"
✅ "분석 결과를 다음 형식으로 줘:
    1) 핵심 요약 3줄
    2) 데이터 표
    3) 액션 아이템 3가지"

형식을 정해야 바로 쓸 수 있는 결과가 나옵니다.
```

---

## 슬라이드 6
```
실수 #5. 예시를 안 보여준다

❌ "톤을 전문적으로 해줘"
✅ "아래 예시처럼 톤을 맞춰줘:
    [실제 샘플 붙여넣기]"

AI에게 "전문적"은 100가지 해석이 가능합니다.
예시 1개가 설명 10줄보다 정확합니다.
```

---

## 슬라이드 7
```
실수 #6. 피드백 없이 1회성으로 끝낸다

❌ 결과 불만족 → "AI는 쓸모없네"
✅ "이 부분을 이렇게 수정해줘"
   "톤을 좀 더 직설적으로"
   "데이터를 추가해서 다시"

대화를 이어가면 결과가 좋아집니다.
1회차 결과로 판단하지 마세요.
```

---

## 슬라이드 8
```
실수 #7. 검증 없이 그대로 쓴다

⚠️ 가장 위험한 실수

AI 결과물을 100% 신뢰하면 안 됩니다.
수치, 팩트, 고유명사는 반드시 확인.

AI는 "그럴듯하게" 만드는 데 최적화되어 있습니다.
"정확하게"가 아닙니다.

결과물 = 초안. 최종본은 사람이 확인.
```
