# C-030: 인스타 캐러셀 — "자동화 안전 체크리스트 — 사고 방지 7가지"

> **유형**: 인스타 캐러셀 | **슬라이드**: 9장 | **톤**: 실용+주의

---

## 슬라이드 1 (표지)
```
자동화 안전 체크리스트

사고 나기 전에 확인하세요

⚠️ 저장 필수
```

---

## 슬라이드 2
```
✅ #1 테스트 먼저

자동화 → 바로 전체 적용? ❌
자동화 → 테스트 1건 → 확인 → 전체 적용 ✅

실제 사고:
테스트 없이 자동 이메일 발송
→ {{고객명}}이 그대로 나감
→ 고객 100명에게 실수 이메일
```

---

## 슬라이드 3
```
✅ #2 사람 연결 장치

AI가 모든 걸 처리해서는 안 됩니다.

필수 규칙:
· AI가 3번 못 풀면 → 사람에게 넘기기
· 금액 관련 문의 → 사람 확인
· 불만/클레임 → 사람 직접 대응

"자동이지만 사람이 있다"는 신뢰가 중요
```

---

## 슬라이드 4
```
✅ #3 알림 빈도 설정

실제 사고:
재고 부족 알림을 1분마다 설정
→ 새벽 3시에 200개 알림 폭발

올바른 설정:
· 일반 알림: 하루 1회
· 긴급 알림: 즉시 (하루 최대 3회)
· 주간 리포트: 주 1회
```

---

## 슬라이드 5
```
✅ #4 민감 정보 체크

AI에 넣으면 안 되는 것:
· 주민등록번호
· 신용카드 전체 번호
· 비밀번호
· 의료 기록

넣어도 되는 것:
· 매출 데이터
· 이메일 내용
· 리뷰 텍스트
· 일반 업무 데이터
```

---

## 슬라이드 6
```
✅ #5 95% 규칙

AI 정확도: 약 95%
100% 완벽을 기대하면 실패합니다.

올바른 접근:
· AI가 95% 처리
· 사람이 5% 검토
· 이게 최적 조합

"AI가 실수하면 어쩌지?" → 사람도 실수합니다
```

---

## 슬라이드 7
```
✅ #6 월 1회 점검

자동화는 만들고 끝이 아닙니다.

월 1회 확인할 것 (10분):
· 자동화가 정상 작동하는지
· 정확도가 떨어지지 않았는지
· 업무 프로세스가 변경되지 않았는지
· 새로 자동화할 업무가 있는지
```

---

## 슬라이드 8
```
✅ #7 롤백 계획

만약 자동화가 문제를 일으키면?

계획:
· 자동화 즉시 중단 (스위치 off)
· 수동으로 전환 (이전 방식)
· 문제 원인 파악
· 수정 후 재시작

걱정하지 마세요.
자동화는 끄면 됩니다. 되돌릴 수 없는 게 아닙니다.
```
