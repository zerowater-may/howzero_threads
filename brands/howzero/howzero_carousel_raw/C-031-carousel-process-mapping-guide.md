# C-031: 인스타 캐러셀 — "프로세스 매핑 — 자동화 전에 반드시 해야 할 것"

> **유형**: 인스타 캐러셀 | **슬라이드**: 8장 | **톤**: 방법론

---

## 슬라이드 1 (표지)
```
자동화 전에
반드시 해야 할 것

프로세스 매핑 가이드

💡 이걸 안 하면 자동화 실패합니다
```

## 슬라이드 2
```
왜 프로세스 매핑이 먼저인가?

"어떤 AI 도구를 써야 하나요?" ← 잘못된 첫 질문
"지금 이 업무의 단계가 뭔가요?" ← 올바른 첫 질문

프로세스를 모르면:
도구를 써도 효과 없음
자동화해도 느려지기만 함
돈만 쓰고 결과 없음
```

## 슬라이드 3
```
Step 1: 종이 한 장 준비

업무 이름을 적으세요.
예: "고객 문의 처리"

그 아래에 단계를 적으세요:
1. 문의 접수 (카톡/이메일)
2. 내용 확인
3. 답변 작성
4. 답변 발송
5. 해결 확인
6. 기록
```

## 슬라이드 4
```
Step 2: 각 단계에 시간 기록

1. 문의 접수 → 1분
2. 내용 확인 → 2분
3. 답변 작성 → 5분
4. 답변 발송 → 1분
5. 해결 확인 → 3분
6. 기록 → 2분
= 건당 14분

하루 20건이면: 4시간 40분
```

## 슬라이드 5
```
Step 3: "이건 AI가 할 수 있나?" Y/N

1. 문의 접수 → Y (자동 수집)
2. 내용 확인 → Y (자동 분류)
3. 답변 작성 → Y (자동 초안)
4. 답변 발송 → Y (자동 발송)
5. 해결 확인 → N (사람 판단)
6. 기록 → Y (자동 기록)

6단계 중 5단계 = AI 가능
```

## 슬라이드 6
```
Step 4: 자동화 순서 정하기

난이도 vs 효과로 우선순위:

🟢 쉽고 효과 큼 → 먼저 (문의 분류, 기록)
🟡 어렵지만 효과 큼 → 두 번째 (답변 자동화)
🔴 효과 적음 → 나중에 또는 안 함

한 번에 다 하지 마세요.
가장 쉬운 것부터.
```

## 슬라이드 7
```
예시: 프로세스 맵 완성본

[고객 문의 처리]
문의 접수 → AI 자동 수집
↓
내용 확인 → AI 자동 분류 (긴급/일반)
↓
답변 작성 → AI 초안 (80% 자동)
↓
답변 발송 → 사람 검토 후 발송
↓
해결 확인 → 사람 판단
↓
기록 → AI 자동 기록

Before: 건당 14분 × 20건 = 280분
After: 건당 3분 × 20건 = 60분
```
