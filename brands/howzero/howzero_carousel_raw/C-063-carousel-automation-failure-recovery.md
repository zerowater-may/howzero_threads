# C-063: 인스타/링크드인 캐러셀 — "자동화 실패 복구 가이드"

> **유형**: 캐러셀 (9장) | **채널**: 인스타 + 링크드인 | **목적**: 저장 유도 + 실패 대응 매뉴얼

---

## 슬라이드 1 (표지)
```
자동화가 멈췄다.
고객 데이터가 꼬였다.
API가 터졌다.

자동화 실패 복구 가이드

→ 스와이프
```

---

## 슬라이드 2
```
자동화는 반드시 실패합니다

실패하지 않는 자동화는 없습니다.
문제는 "실패할 때 얼마나 빨리
복구하느냐"입니다.

하우제로도 자동화 장애를
월 평균 2~3회 겪습니다.

차이는 복구 시간.
10분 vs 10시간.
```

---

## 슬라이드 3
```
실패 유형 1: API 연결 끊김

증상: 데이터가 안 넘어간다
원인: API 키 만료, 서버 점검, 요금 초과

복구법:
① API 키 상태 확인
② 서비스 상태 페이지 확인
③ 키 갱신 또는 대체 경로 설정
④ 밀린 데이터 수동 재실행

예방: 만료일 알림 자동화 (아이러니하죠)
```

---

## 슬라이드 4
```
실패 유형 2: 데이터 중복/누락

증상: 같은 메일 2번, 주문 누락
원인: 트리거 중복 실행, 필터 오류

복구법:
① 중복 데이터 식별 (타임스탬프 기준)
② 중복 제거 후 정상 데이터만 남기기
③ 필터 조건 재점검
④ 테스트 데이터로 재실행

예방: 고유 ID 기반 중복 체크 로직 추가
```

---

## 슬라이드 5
```
실패 유형 3: 워크플로우 무한 루프

증상: 같은 작업이 끝없이 반복
원인: 트리거와 액션이 서로를 호출

복구법:
① 즉시 워크플로우 비활성화
② 루프 원인 지점 확인
③ 조건문으로 루프 차단
④ 실행 횟수 제한 설정

예방: 모든 워크플로우에 실행 횟수 상한 설정
```

---

## 슬라이드 6
```
실패 유형 4: 형식 변환 오류

증상: 날짜가 깨짐, 숫자가 문자로 바뀜
원인: 시스템 간 데이터 형식 불일치

복구법:
① 오류 데이터 범위 특정
② 형식 변환 로직 수정
③ 영향받은 데이터 일괄 재변환
④ 변환 테스트 케이스 추가

예방: 데이터 전달 시 형식 명시적 지정
```

---

## 슬라이드 7
```
복구 속도를 높이는 3가지 습관

1. 에러 알림 세팅
   실패 즉시 슬랙/이메일로 알림.
   모르고 3일 지나면 복구 비용 10배.

2. 실행 로그 보관
   최소 30일치 로그 보관.
   "언제, 어디서, 왜" 추적 가능.

3. 수동 복구 매뉴얼 문서화
   복구 절차를 미리 적어두면
   담당자 누구든 10분 안에 해결.
```

---

## 슬라이드 8 (체크리스트)
```
자동화 운영 안전장치 체크리스트

□ 에러 발생 시 즉시 알림 설정됨
□ 실행 로그 30일 이상 보관
□ 각 워크플로우 수동 복구 매뉴얼 있음
□ 월 1회 장애 시뮬레이션 실행
□ API 키 만료일 알림 설정됨
□ 데이터 백업 주기 설정됨

4개 이상 체크 → 안전합니다.
3개 이하 → 지금 바로 세팅하세요.
```
