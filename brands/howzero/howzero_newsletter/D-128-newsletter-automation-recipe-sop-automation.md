안녕하세요, 하우제로입니다.

SOP(Standard Operating Procedure). 표준 운영 절차서.

대부분의 회사에서 SOP는 이렇습니다: Notion에 잘 정리해놓고, 아무도 안 봅니다. 새 직원이 들어오면 "이거 읽어봐"라고 링크 던져주고, 실제로는 옆 사람한테 물어봐서 배웁니다.

SOP는 문서가 아니라 시스템이어야 합니다. 사람이 읽고 따라하는 게 아니라, 시스템이 자동으로 실행하는 거예요.

---

🍳 **이번 주 레시피: SOP 자동화**

**난이도**: ★★★☆☆ (중급)
**세팅 시간**: 프로세스당 1~2시간
**효과**: 실행 오류 90% 감소, 신규 인력 적응 시간 60% 단축

---

🔧 **3단계 프레임워크**

**Step 1: SOP 디지털화**

```
기존 SOP를 "트리거 → 액션 → 체크" 구조로 분해:

예시: 신규 고객 온보딩 SOP
━━━━━━━━━━━━━━━━━━━━━━
트리거: 결제 완료
→ 액션 1: 환영 이메일 발송 (템플릿 A)
→ 액션 2: Slack #cs에 알림
→ 액션 3: 온보딩 자료 패키지 전달
→ 체크: 24시간 내 로그인 여부 확인
→ 미로그인 시: 팔로업 이메일 발송
```

**Step 2: 자동 실행 파이프라인**

```
Make/Zapier로 자동화:
→ 트리거 감지 시 자동 시작
→ 각 액션 순차 실행
→ 체크포인트에서 조건 분기
→ 예외 발생 시 담당자에게 에스컬레이션
→ 전체 과정 로그 자동 기록
```

**Step 3: SOP 챗봇**

```
GPT-4 기반 내부 챗봇:
→ 전체 SOP 문서를 학습시킨 RAG 시스템
→ 직원이 "환불 처리 어떻게 해?" 질문
→ 해당 SOP를 찾아서 단계별로 안내
→ 필요한 양식/템플릿 자동 제공
→ "이 프로세스 시작하기" 버튼 → 자동화 트리거
```

---

📊 **하우제로 SOP 자동화 현황**

| 프로세스 | 자동화율 | 월 실행 횟수 |
|---------|---------|------------|
| 신규 고객 온보딩 | 95% | 25건 |
| 환불 처리 | 90% | 8건 |
| 콘텐츠 발행 | 80% | 12건 |
| 주간 리포팅 | 100% | 4건 |
| 강사 섭외/계약 | 70% | 3건 |

자동화율 100%가 아닌 것들이 있습니다. 의도적입니다. "판단이 필요한 지점"은 사람이 개입합니다. 환불 사유가 복잡한 경우, 콘텐츠 최종 검수, 강사 역량 평가 같은 것들이요.

---

⚠️ **주의사항**

- 모든 SOP를 한 번에 자동화하지 마세요. 실행 빈도가 높은 것부터 하나씩
- 자동화 전에 프로세스 자체를 먼저 개선하세요. 엉망인 SOP를 자동화하면 엉망인 결과가 빨라질 뿐입니다
- SOP 챗봇은 정기적으로 SOP 문서를 업데이트해야 합니다. 낡은 SOP를 답변하면 더 위험해요

SOP가 "아 그거 Notion에 있어"에서 "시스템이 알아서 해"로 바뀌면, 비즈니스의 수준이 달라집니다.
