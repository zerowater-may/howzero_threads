팀이 20명이 되면 조직도가 필요합니다.

5명일 때: 모두가 모든 걸 합니다.
10명일 때: 역할이 생깁니다.
20명일 때: 구조가 필요합니다.

하우제로가 정확히 이 단계입니다.
1명에서 20명까지, 조직 구조를 3번 바꿨습니다.

---

**레시피 #86 구성:**

1. **업무 흐름 자동 매핑**
   - 팀원별 업무 로그 자동 수집 (Jira/Linear/슬랙)
   - 누가 누구와 자주 협업하는지 자동 분석
   - 실제 업무 흐름 기반 조직도 자동 생성
   - 공식 조직도 vs 실제 협업 패턴 비교

2. **병목 자동 감지**
   - 특정 인원에게 업무 집중? → "병목 리스크" 알림
   - 팀 간 핸드오프 지연? → "프로세스 갭" 플래그
   - 한 사람이 빠지면 멈추는 업무? → "단일 장애점" 경고

3. **조직 시뮬레이션**
   - "팀을 이렇게 재편하면?" 시나리오 시뮬레이션
   - 팀 A와 팀 B를 합치면 협업 효율이 올라갈까?
   - 신규 채용 1명 → 어느 팀에 배치하면 최대 효과?
   - 데이터 기반 의사결정 지원

4. **역할/책임 자동 문서화**
   - 각 포지션의 실제 업무 내용 자동 정리
   - 역할 중복 자동 감지
   - 미정의 영역(누구의 책임인지 불명확한 업무) 자동 식별
   - R&R 매트릭스 자동 생성 및 업데이트

---

하우제로 조직 변천사:

1~5명 (1~2년차): 플랫 구조. 대표가 모든 결정.
5~15명 (2~4년차): 기능별 구분 (개발, 콘텐츠, 운영). 리드 3명.
15~20명 (4~5년차): 지역별 + 기능별 매트릭스.
- 한국팀, 일본팀, 동남아팀
- 각 팀에 개발/콘텐츠/영업 기능 내재화
