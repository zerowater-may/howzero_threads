안녕하세요, 하우제로입니다.

새 직원 한 명 들어오면, 기존 직원 한 명이 일주일간 마비됩니다.

계정 만들어주고, 시스템 설명하고, 매뉴얼 찾아서 보내주고, 질문에 답하고. 이걸 매번 사람이 합니다. 3명 뽑으면 3번 반복.

5인 이하 소규모 팀이라면 더 심합니다. 온보딩 담당자가 따로 없으니까 대표가 직접 하죠. 대표의 시간은 가장 비싼 시간인데요.

---

🍳 **이번 주 레시피: 직원 온보딩 자동화**

**난이도**: ★★☆☆☆ (초급)
**세팅 시간**: 15분
**효과**: 온보딩 기간 5일 → 2일

---

📋 **재료**

- Claude Code (월 $20)
- Notion 또는 Google Docs (무료)
- Slack 또는 카카오톡 (알림용)

---

🔧 **만드는 법**

**Step 1: 온보딩 체크리스트 자동 생성 (5분)**

```
신규 직원 입사 시 다음 온보딩 체크리스트를 자동 생성해줘:

Day 1: 계정 셋업
- [ ] 이메일 계정 생성
- [ ] Slack 초대
- [ ] 프로젝트 관리 도구 접근 권한
- [ ] 공유 드라이브 접근 권한

Day 2: 업무 이해
- [ ] 회사 소개 문서 읽기
- [ ] 팀 구조 및 역할 파악
- [ ] 주요 프로세스 매뉴얼 학습

Day 3-5: 실무 적응
- [ ] 첫 과제 수행
- [ ] 멘토와 1:1 미팅
- [ ] 질문 목록 정리 및 공유

각 항목 완료 시 자동으로 다음 단계 안내.
```

**Step 2: AI 온보딩 어시스턴트 (5분)**

```
신규 직원이 질문하면 다음 자료에서 답변을 찾아서 알려줘:

1. 회사 위키/매뉴얼
2. FAQ 문서
3. 과거 온보딩 질문 모음

못 찾으면 "이 질문은 [담당자]에게 확인이 필요합니다"로 안내.
매일 밤 "오늘 해결 못한 질문" 리스트를 팀장에게 전송.
```

**Step 3: 진행률 추적 자동화 (5분)**

```
온보딩 체크리스트 진행률을 매일 추적해서:
- 신규 직원에게: "오늘 남은 할 일 3개" 아침 알림
- 팀장에게: 주간 온보딩 진행률 리포트
- 지연 항목 있으면 자동 리마인더
```
