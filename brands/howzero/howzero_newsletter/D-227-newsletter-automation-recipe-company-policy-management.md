"경비 처리 기준이 뭐였죠?"
"출장비는 얼마까지 되나요?"
"재택근무 규정이 바뀌었다고요? 언제?"

팀이 커지면 규정이 생기고,
규정이 생기면 아무도 읽지 않습니다.

하우제로도 마찬가지였습니다.
15명 팀에 규정 문서 32개.
실제로 읽은 사람: 신규 입사자 (그것도 첫 주만).

---

**레시피 #84 구성:**

1. **규정 챗봇**
   - "출장비 한도가 얼마야?" → 즉시 답변
   - 자연어 질문 → 관련 규정 조항 자동 검색 + 인용
   - 슬랙에서 바로 사용 가능 (/policy 명령어)

2. **규정 변경 자동 알림**
   - 규정 문서가 수정되면 → 변경 내용 요약 자동 생성
   - 관련 팀원에게만 타겟 알림 (경비 규정 → 전원, HR 규정 → 해당자만)
   - "이해했습니다" 확인 버튼 → 열람 추적

3. **자동 준수 모니터링**
   - 경비 청구 → 규정 한도 자동 체크
   - 초과 시 자동 알림 + 사유 입력 요청
   - 반복 위반 패턴 → 규정 자체가 현실적인지 검토 제안

4. **규정 자동 리뷰 사이클**
   - 6개월마다 각 규정의 "활용 빈도" 자동 분석
   - 한 번도 조회되지 않은 규정 → "폐지 검토" 플래그
   - 질문이 많은 영역 → "규정 보완 필요" 플래그
   - 규정 버전 관리 자동화 (누가, 언제, 무엇을 바꿨는지)

---

결과:
- 규정 관련 질문: 주 15건 → 주 2건 (챗봇이 해결)
- 경비 규정 위반: 월 8건 → 월 1건
- 규정 업데이트 인지율: 30% → 95%
- HR 담당자 규정 관리 시간: 주 6시간 → 주 1시간

규정은 만드는 게 중요한 게 아닙니다.
지켜지는 게 중요합니다.
지켜지려면 쉬워야 합니다.
