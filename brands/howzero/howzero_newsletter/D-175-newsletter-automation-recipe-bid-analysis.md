안녕하세요, 하우제로입니다.

고객사 하나가 연간 입찰 50건에 참여합니다. 낙찰률 18%. 입찰서 작성에 건당 평균 3일.

문제는 입찰가 산정이 "경험과 감"에 의존한다는 겁니다. 과거 낙찰 데이터를 분석해보니, 너무 높게 써서 떨어진 게 35%, 너무 낮게 써서 이기고도 손해 본 게 12%.

데이터 기반으로 입찰가를 분석하는 시스템을 만들었더니 낙찰률이 18%에서 32%로 올랐습니다.

---

🍳 **이번 주 레시피: 입찰가 분석 자동화**

**난이도**: ★★★★☆ (중상급)
**세팅 시간**: 6시간
**효과**: 낙찰률 향상, 입찰 준비 시간 60% 절감

---

🔧 **자동화 구조**

**1. 과거 데이터 수집 & 분석**

```
입찰 이력 자동 DB화:
→ 과거 입찰 건별: 입찰가, 낙찰가, 경쟁사 수, 발주처
→ 나라장터/조달청 공개 데이터 자동 크롤링
→ 유사 프로젝트 낙찰가 범위 자동 산출
→ 발주처별 낙찰 패턴 분석 (최저가? 적격심사?)
→ 경쟁사별 입찰 패턴 추정
```

**2. 최적 입찰가 추천**

```
입찰가 시뮬레이션:
→ 프로젝트 유형 + 예정가격 입력
→ 유사 과거 건 자동 매칭 (10~50건)
→ 낙찰 확률 구간별 시뮬레이션
→ "적정 입찰가 범위" 추천 (하한~상한)
→ 마진율별 시나리오 3개 자동 생성
→ 리스크 점수 표시 (낮을수록 안전)
```

**3. 입찰서 자동 초안**

```
문서 자동화:
→ 발주 공고문 분석 → 평가 기준 자동 추출
→ 평가 항목별 가중치 파악
→ 과거 수주 건에서 유사 실적 자동 매칭
→ 입찰 제안서 구조 자동 생성
→ 필수 첨부 서류 체크리스트 자동 생성
```
