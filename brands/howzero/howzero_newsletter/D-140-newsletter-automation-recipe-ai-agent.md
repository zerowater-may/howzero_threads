지금까지의 자동화는 "규칙 기반"이었습니다. "A가 발생하면 B를 실행해라." 정해진 대로 움직이는 거죠. AI 에이전트는 다릅니다. "목표를 주면, 스스로 판단하고 실행합니다."

이게 자동화의 다음 단계입니다.

---

🍳 **이번 주 레시피: AI 에이전트 구축**

**난이도**: ★★★★★ (고급)
**세팅 시간**: 4~6시간
**효과**: 복잡한 의사결정 업무의 자동화

---

🔧 **AI 에이전트 vs 기존 자동화**

```
기존 자동화 (Rule-based):
→ "이메일이 오면 분류해라"
→ 정해진 규칙대로만 동작
→ 예외 상황 → 멈춤

AI 에이전트 (Agent-based):
→ "고객 문의를 해결해라"
→ 문의 내용을 이해하고, 적절한 도구를 선택하고, 답변을 생성
→ 예외 상황 → 판단 + 에스컬레이션
```

---

**Step 1: 에이전트 설계**

```
목적 정의: "고객 문의 1차 대응 에이전트"

역할:
→ 고객 이메일/채팅 문의 수신
→ 의도 파악 (환불, 기술 지원, 일반 질문, 불만)
→ 적절한 액션 실행 (FAQ 답변, 티켓 생성, 환불 처리)
→ 복잡한 건은 사람에게 넘김

도구 (Tools):
→ FAQ 검색 (RAG)
→ 주문 정보 조회 (DB 연동)
→ 환불 실행 (Stripe API)
→ 티켓 생성 (Linear/Jira)
→ 이메일 발송 (SendGrid)
```

**Step 2: 에이전트 구현**

```
기술 스택:
→ LLM: GPT-4 / Claude API
→ 오케스트레이션: LangChain 또는 CrewAI
→ 벡터 DB: Pinecone (FAQ/문서 검색)
→ 외부 도구: Make/Zapier 연동

핵심 프롬프트 구조:
"당신은 하우제로의 CS 에이전트입니다.
 고객 문의를 분석하고, 적절한 도구를 사용하여 해결하세요.
 확신이 없으면 반드시 사람에게 에스컬레이션하세요.
 절대 추측으로 답변하지 마세요."
```

**Step 3: 안전장치 (Guardrails)**

```
필수 안전장치:
→ 환불 금액 ₩10만 이상 → 사람 승인 필요
→ 고객 감정 "분노" 감지 → 즉시 사람 전달
→ 동일 고객 3회 이상 문의 → 사람 개입
→ 에이전트 확신도 70% 미만 → 답변 보류 + 사람 검토
→ 모든 에이전트 액션 로그 기록
```

⚠️ **주의사항**

- **에이전트에게 "절대 하면 안 되는 것"을 명확하게 정의하세요.** 환불, 계정 삭제, 개인정보 제공 등

- **에이전트는 만능이 아닙니다.** 단순 반복이 많은 업무부터 시작하세요
