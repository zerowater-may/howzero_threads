안녕하세요, 하우제로입니다.

앱을 만들기로 했습니다.

"SaaS 있는데 왜 앱까지?" 이 질문을 저도 6개월 동안 했습니다. 그리고 데이터가 답을 줬어요.

고객의 68%가 모바일로 대시보드를 확인합니다. 그런데 웹 SaaS는 모바일에서 불편합니다. 이탈 로그를 보면 모바일 접속 후 30초 이내 이탈이 42%. 반응형 웹으로는 한계가 있었습니다.

앱을 만들겠다는 건, 새 사업이 아니라 기존 고객의 불편을 해결하는 겁니다.

---

📱 **앱 기획 방향**

**핵심 원칙: "대시보드 + 알림"**

앱에서 모든 걸 할 필요 없습니다. 복잡한 세팅은 웹에서. 앱은 두 가지만 잘하면 됩니다.

```
앱 핵심 기능:
1. 실시간 대시보드 (매출, KPI, 자동화 상태)
2. 푸시 알림 (이상 감지, 주요 지표 변화)
3. 빠른 승인/결재 (터치 한 번으로)
4. 주간 리포트 모바일 뷰
```

---

💡 **개발 전략: 빌드 vs 노코드**

```
검토한 옵션:
→ 네이티브 개발 (Swift + Kotlin): 6개월, ₩1.5억
→ React Native: 4개월, ₩8,000만
→ Flutter: 4개월, ₩7,000만
→ 노코드 (FlutterFlow): 2개월, ₩2,000만

선택: FlutterFlow → 네이티브 전환 (2단계)
```

1단계는 FlutterFlow로 MVP를 2개월 안에 출시합니다. 핵심 기능만 넣고, 실제 사용 데이터를 모읍니다. 2단계에서 검증된 기능만 네이티브로 다시 만듭니다.

완벽한 앱을 6개월 뒤에 내는 것보다, 80점짜리 앱을 2개월 뒤에 내는 게 낫습니다. 나머지 20점은 고객이 알려줍니다.

🔥 **불안한 것들**

- 앱 유지보수 비용. 웹만 관리하다가 앱까지 관리하면 리소스가 분산됩니다
- 앱스토어 정책 변화 리스크. 남의 플랫폼에 올리는 건 항상 불안합니다
- 팀에 모바일 전문가가 없습니다. 외주 or 채용 결정을 해야 합니다

전부 공개하면서 가겠습니다. 다음 뉴스레터에서 와이어프레임 공유합니다.
