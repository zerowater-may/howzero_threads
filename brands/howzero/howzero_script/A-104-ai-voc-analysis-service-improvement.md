# A-104: 유튜브 롱폼 — "AI로 고객 VOC 분석 → 서비스 개선"
> **유형**: 유튜브 롱폼 | **길이**: 17분
## 메타데이터
- **제목**: AI로 고객 VOC 분석 → 서비스 개선까지 자동화
- **썸네일 텍스트**: VOC 분석 AI 자동화 | 고객 목소리→개선
- **태그**: VOC, 고객 의견, 감성 분석, 서비스 개선, 품질 관리, AI, 하우제로
## 대본

### 훅 (0:00–0:40)
고객이 뭘 원하는지 알면 매출이 올라갑니다. 당연한 이야기인데, 대부분의 회사가 고객 목소리(VOC)를 제대로 분석하지 않습니다. CS 문의, 리뷰, SNS 댓글, 설문 — 데이터는 넘치는데 정리가 안 됩니다. AI로 전 채널 VOC를 자동 수집+분석해서 서비스 개선 포인트까지 자동 도출하는 시스템. NPS 25점 올린 사례 보여드리겠습니다.

### 문제 정의 (0:40–3:00)
VOC 관리의 현실. 첫째, 데이터 분산 — CS 상담 기록, 네이버 리뷰, 인스타 댓글, 카카오 상담, 설문 결과가 각각 따로. 둘째, 분석 불가 — 월 수백~수천 건의 VOC를 사람이 읽고 분류하는 건 불가능. 셋째, 액션 연결 부재 — 분석해도 "그래서 뭘 고치지?"의 답이 안 나옴.

### 자동화 설계 (3:00–8:30)
**VOC 자동 수집**: CS 티켓 시스템, 네이버 리뷰, 인스타 댓글, 카카오 상담 로그 → 하나의 구글 시트로 자동 통합.

**자동 분류+감성 분석**: AI가 VOC를 — 카테고리 분류(제품 품질, 배송, CS 태도, 가격, UI/UX 등) + 감성 분류(긍정/부정/중립) + 긴급도 판별(즉시 대응/일반/참고).

**트렌드 분석**: 주간/월간 — 카테고리별 VOC 추이 + 급증하는 불만 사항 자동 감지 + 경쟁사 리뷰 대비 분석.

**개선 제안 자동화**: 부정 VOC 상위 항목 → AI가 구체적 개선 액션 제안(예: "배송 지연 불만 40%↑ → 택배사 변경 또는 발송 기한 단축 필요") → 담당 부서에 자동 티켓 생성.

**효과 측정**: 개선 시행 후 해당 카테고리 VOC 변화 자동 추적 → 개선 효과 리포트.

### 데모/사례 (8:30–13:30)
이커머스 V사. 월 VOC 약 1,200건(CS 800건 + 리뷰 300건 + SNS 100건). 자동화 전: VOC 분석 없음. CS 팀장이 감으로 "요즘 배송 불만 많은 것 같다" 수준. 자동화 후: 전 채널 자동 수집+분류. "포장 불량" 불만 전월 대비 85%↑ 자동 감지 → 포장재 변경 → 다음 달 관련 불만 70%↓. NPS 32 → 57.

도구: 구글 시트 + Make + GPT-4 + Looker Studio + 슬랙. 월 운영비 약 8만 원.
