## 대본

### 훅 (0:00–0:40)
펜션 운영해본 분들 아실 겁니다. 예약 플랫폼 3개 관리하면서 중복 예약 막고, 정산 맞춰보고, 리뷰 하나하나 답글 달고… 체크인/체크아웃 관리까지 하면 잠잘 시간이 없습니다. 이걸 AI로 전부 자동화한 펜션 사장님, 지금은 주 3일만 일하십니다.

### 문제 정의 (0:40–3:00)
숙박업 운영의 3대 병목. 첫째, 멀티 채널 예약 관리 — 에어비앤비, 야놀자, 여기어때, 직접 예약… 채널별 캘린더 동기화가 안 되면 더블부킹. 둘째, 정산 — 채널별 수수료가 다르고, 부가세, 카드 수수료까지 따지면 실수 잦음. 셋째, 리뷰 관리 — 채널별로 리뷰가 분산. 답글 안 달면 평점 하락.

### 자동화 설계 (3:00–8:30)
**예약 통합**: 모든 채널 예약을 하나의 구글 시트/DB로 자동 동기화 → 잔여 객실 자동 업데이트 → 더블부킹 차단 → 체크인 전날 자동 안내 문자(위치, 비밀번호, 주의사항).

**정산 자동화**: 채널별 수수료율 DB → 예약 확정 시 순매출 자동 계산 → 월말 정산 리포트 자동 생성 → 세금계산서 발행 연동.

**리뷰 자동 관리**: 전 채널 리뷰 자동 수집 → 감성 분석(긍정/부정/중립) → 부정 리뷰 즉시 알림 → AI가 톤에 맞는 답글 초안 생성 → 사장님 확인 후 게시.

### 데모/사례 (8:30–13:30)
강원도 소재 펜션 G. 객실 8개. 에어비앤비+야놀자+직접예약 3채널 운영. 자동화 전: 더블부킹 월 2~3회(환불+리뷰 피해), 정산 오차 월 50만 원+, 리뷰 답글 평균 5일 소요. 자동화 후: 더블부킹 제로. 정산 오차 제로. 리뷰 답글 평균 3시간(AI 초안+사장님 확인). 체크인 안내 100% 자동. 운영 시간 주 7일 → 주 3일.

도구: 구글 시트 + Make + 채널 API + GPT-4 + 카카오 알림톡. 월 운영비 약 5만 원.
