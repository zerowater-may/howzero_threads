## 대본

### 훅 (0:00–0:40)
콘텐츠 만들고 올리면 끝인가요? 대부분 그렇게 합니다. 그런데 같은 콘텐츠도 썸네일 하나, 제목 한 글자 바꾸면 클릭률이 2~3배 차이납니다. 문제는 A/B 테스트를 수동으로 하면 너무 오래 걸린다는 거죠. AI가 변형안 자동 생성, 자동 배포, 결과 자동 분석, 승자 자동 확대까지. 콘텐츠 성과가 감이 아니라 데이터로 결정됩니다.

### 문제 정의 (0:40–3:00)
콘텐츠 A/B 테스트의 현실. 첫째, 변형안 만들기 귀찮음 — 제목 3개, 썸네일 3개, CTA 3개 만들려면 조합만 27개. 사람이 다 못 합니다. 둘째, 테스트 설계가 어려움 — 표본 크기, 테스트 기간, 통계적 유의성. 마케터가 통계를 모릅니다. 셋째, 결과 분석이 수동 — 플랫폼별로 데이터 뽑아서 비교합니다. 넷째, 학습이 축적 안 됨 — 이번 테스트 결과가 다음 콘텐츠에 반영 안 됩니다. 매번 제로에서 시작.

### 자동화 설계 (3:00–8:30)
**변형안 자동 생성**: 원본 콘텐츠 입력 → AI가 제목 5개 변형 + 썸네일 텍스트 5개 변형 + CTA 3개 변형 자동 생성 → 과거 성과 데이터 기반으로 승률 높은 조합 우선 추천.

**자동 배포+테스트 실행**: 변형안 자동 배포(이메일·SNS·광고 플랫폼) → 표본 크기 자동 계산 → 통계적 유의성 달성까지 자동 모니터링 → 중간 결과 리포트 자동 생성.

**승자 자동 확대**: 통계적 유의성 달성 시 자동 판정 → 승자 버전 전체 오디언스에 자동 확대 → 패자 버전 자동 중단 → 비용 낭비 최소화.

**학습 DB 자동 구축**: 모든 테스트 결과 자동 축적 → "우리 오디언스는 숫자가 포함된 제목에 CTR 1.8배" 같은 인사이트 자동 추출 → 다음 콘텐츠 생성 시 AI가 학습 DB 참조하여 초안 품질 향상.

### 데모/사례 (8:30–13:00)
D2C 브랜드 X. 주 3회 뉴스레터 + 주 5회 SNS 포스팅. 자동화 전: A/B 테스트 월 1~2회(수동), 이메일 오픈율 평균 18%, SNS 평균 CTR 1.2%, 콘텐츠 제작 주 15시간. 자동화 후: A/B 테스트 모든 콘텐츠 자동, 이메일 오픈율 평균 31%, SNS 평균 CTR 2.8%, 콘텐츠 제작 주 8시간(변형안 자동 생성).
