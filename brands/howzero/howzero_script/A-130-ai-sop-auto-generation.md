## 대본

### 훅 (0:00–0:40)
"이거 어떻게 하는 거예요?" 신입 직원이 물을 때마다 사장님이 직접 알려주고 계시죠. 그 설명을 녹음해서 AI에게 주면 5분 만에 SOP가 나옵니다. 표준작업절차서. 이름은 거창하지만, 결국 "이 업무는 이렇게 하세요"를 문서로 만든 겁니다. SOP가 없으면 사람이 바뀔 때마다 바퀴를 다시 발명합니다. AI로 SOP를 자동 생성하는 방법을 보여드리겠습니다.

### 문제 정의 (0:40–3:00)
SOP 부재의 3대 문제. 첫째, 인수인계 비용 — 직원 퇴사 시 업무 인수인계 평균 2~4주. SOP가 있으면 3일. 둘째, 품질 편차 — 같은 업무를 사람마다 다르게 처리. 고객 경험 불일치. 셋째, 사장님 의존 — "사장님한테 물어봐야 해요"가 모든 질문의 답. 사장님이 병목.

SOP를 못 만드는 이유도 명확합니다. 시간이 없어서. 일하면서 동시에 문서를 만들 여유가 없습니다. 하지만 AI가 있으면 말만 하면 됩니다.

### 자동화 설계 (3:00–8:30)
**음성→SOP 자동 변환**: 사장님이 업무를 설명하면서 녹음(5~10분) → Whisper API로 텍스트 변환 → GPT-4가 SOP 포맷으로 자동 정리(목적, 대상, 준비물, 단계별 절차, 주의사항, FAQ).

**기존 자료→SOP 변환**: 카톡 대화, 이메일, 메모에 흩어진 업무 지침 → AI가 수집·정리 → 중복 제거 → 체계적 SOP로 재구성.

**SOP 관리 시스템**: 구글 독스에 SOP 라이브러리 구축 → 업무별 검색 → 버전 관리(변경 이력 자동 기록) → 신입 온보딩 시 해당 업무 SOP 자동 안내.

**SOP 자동 업데이트**: 업무 프로세스가 변경되면 담당자가 변경 사항 음성 입력 → AI가 기존 SOP에 반영 → 변경 알림 관련 직원에게 자동 발송.

### 데모/사례 (8:30–13:30)
온라인 쇼핑몰 SP. 직원 8명, 주요 업무 프로세스 25개, SOP 0개(전부 구전). 자동화 프로젝트: 2주간 사장님+팀장이 핵심 업무 25개를 각 5~10분씩 녹음 → AI가 25개 SOP 자동 생성 → 검수·수정 1주 → 완성.

결과: 신입 온보딩 기간 3주→1주. "사장님한테 물어봐" 빈도 일 8건→1건. 업무 실수율 30%↓. 6개월 후 직원 퇴사 발생 — 인수인계 3일에 완료(이전 4주).

도구: 음성 녹음(스마트폰) + Whisper API + GPT-4 + 구글 독스 + Make. 초기 구축비 30만 원, 이후 유지비 월 3만 원.
