## 대본

### 훅 (0:00–0:40)
인력파견 업체 대표님, 가장 스트레스받는 게 뭐예요? 고객사가 "내일까지 3명 보내주세요" 하면 가용 인력 뒤져서 전화 돌리고, 안 되면 다른 사람 찾고… 매칭 하나에 반나절. 여기에 근태 관리, 급여 정산까지 하면 사무실에서 살아야 합니다. AI로 인력 매칭부터 정산까지 자동화한 업체 이야기입니다.

### 문제 정의 (0:40–3:00)
인력파견 업체의 3대 병목. 첫째, 인력 매칭 — 고객사 요구 조건(직종, 경력, 지역, 가용일)과 보유 인력을 매칭. 수동으로 하면 오래 걸리고 최적이 아님. 둘째, 근태 관리 — 파견 인력의 출퇴근, 초과근무, 결근 관리. 현장이 분산돼서 파악이 어려움. 셋째, 정산 — 인력별 시급, 야간/주말 수당, 4대 보험, 고객사별 청구까지 복잡.

### 자동화 설계 (3:00–8:30)
**인력 매칭 자동화**: 고객사 요청 접수(직종, 인원, 기간, 위치) → AI가 인력 DB에서 조건 매칭(경력, 평가, 거리, 가용일) → 매칭 스코어 순으로 추천 → 확정된 인력에게 자동 안내(일시, 장소, 복장 등).

**근태 자동화**: GPS 기반 출퇴근 앱 → 자동 기록 → 지각/조퇴/결근 자동 감지 → 관리자+고객사 알림 → 대체 인력 자동 제안.

**정산 자동화**: 근태 기록 기반 → 인력별 급여 자동 계산(기본급+수당+공제) → 고객사별 청구서 자동 생성 → 발송 → 입금 확인 자동 매칭.

**인력 평가**: 고객사 평가 자동 수집 → 인력별 평가 DB 누적 → 매칭 시 평가 반영 → 우수 인력 우선 배치.

### 데모/사례 (8:30–13:30)
인력파견 Y사. 등록 인력 500명, 거래 고객사 30개. 자동화 전: 매칭 건당 평균 4시간, 근태 확인 일 2시간, 정산 월 3일. 자동화 후: 매칭 건당 20분(AI 추천 확인만), 근태 자동(확인만 15분/일), 정산 자동(검토만 2시간/월). 고객사 만족도 향상(적합 인력 배치율 60%→85%).
