## 대본

### 훅 (0:00–0:40)
자동차 정비소 1개 운영할 때는 사장님 머릿속에 다 있습니다. 단골 차량 번호, 정비 이력, 부품 재고. 그런데 2호점, 3호점 내면? 전부 깨집니다. 정비소 체인이 성장하면서 무너지는 이유는 기술력이 아니라 관리 시스템입니다. 3개 지점 정비소를 AI로 통합 관리한 사례를 보여드리겠습니다.

### 문제 정의 (0:40–3:30)
정비소 체인의 3대 병목. 첫째, 예약 분산 — 지점별로 전화 예약, 예약 현황 공유 안 됨, 특정 지점만 과부하. 둘째, 정비 이력 파편화 — A 지점에서 엔진오일 교체한 고객이 B 지점 방문하면 이력을 모름. 처음부터 다시 확인. 셋째, 부품 재고 — 지점별 재고가 달라서 "이 부품 없으니 다음에 오세요"가 반복. 지점 간 부품 이동도 수동.

### 자동화 설계 (3:30–9:30)
**통합 예약 시스템**: 카카오 챗봇으로 예약 접수 → 차량 번호 입력하면 자동으로 정비 이력 조회 → 가장 가까운 지점 추천 + 잔여 시간대 표시 → 예약 확정 → 방문 전날 리마인드 + 예상 정비 항목 안내.

**통합 정비 이력 DB**: 모든 지점의 정비 기록이 하나의 구글 시트로 통합 → 차량 번호로 전체 이력 조회 → 주행거리 기반 다음 정비 항목 자동 추천 → 정비 완료 후 고객에게 결과 리포트 자동 발송.

**부품 재고 자동화**: 지점별 부품 사용량 실시간 집계 → 안전 재고 이하 시 자동 발주 → 지점 간 재고 차이 감지 시 이동 제안 → 월별 부품 사용량 분석으로 시즌별 수요 예측.

**고객 케어 자동화**: 엔진오일 교체 후 5,000km 예상 도달 시점에 안내 → 정기 점검 시기 자동 알림 → 계절별 점검 프로모션(여름 에어컨, 겨울 배터리) 자동 발송.

### 데모/사례 (9:30–14:30)
정비소 체인 AR. 3개 지점, 일 방문 차량 총 45대. 자동화 전: 지점별 예약 전화 각 20통+, 정비 이력 공유 안 됨, 부품 재고 파악 주 1회(수동), 고객 재방문율 35%. 자동화 후: 예약 전화 70%↓(챗봇 처리). 통합 이력으로 교차 지점 방문 시 즉시 이력 확인. 부품 재고 실시간 파악 + 발주 자동. 정비 리마인드로 고객 재방문율 35%→58%.

도구: 구글 시트 + Make + GPT-4 + 카카오 챗봇. 월 운영비 약 12만 원.
