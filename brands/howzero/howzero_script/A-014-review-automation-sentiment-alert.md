# A-014: 유튜브 롱폼 — "부정 리뷰 10초 만에 잡는 AI 시스템 만들기"

> **유형**: 유튜브 롱폼 | **길이**: 14분 | **시리즈**: 자동화 레시피 EP.03

---

## [0:00~0:08] 훅

"별점 1점 리뷰가 올라왔습니다. 여러분은 언제 확인하세요? 3일 후? 일주일 후? 그때는 이미 늦었습니다. 오늘 별점 3 이하 리뷰가 올라오면 10초 만에 알림이 오고, 답변 초안까지 만들어주는 시스템을 15분 만에 만들겠습니다."

## [0:08~2:00] 왜 리뷰 자동화가 중요한가

"리뷰를 3일 후에 확인하면 이미 매출이 떨어져 있습니다.

데이터:
- 부정 리뷰 1개가 고객 구매 결정에 미치는 영향: 평균 22%
- 부정 리뷰 후 24시간 내 대응 시 → 재구매율 33% 회복
- 부정 리뷰 후 72시간 이후 대응 시 → 회복률 7%

10분 내 대응 vs 3일 후 대응. 회복률 5배 차이.

근데 리뷰를 매일 확인하려면 하루 30분~1시간. 그 시간이 없으니까 3일 후에 보게 되는 거죠. 악순환.

AI가 해결합니다. 리뷰 올라오면 → 10초 내 알림 → 답변 초안까지. 확인 시간 0분."

## [2:00~4:00] 완성된 결과 먼저

**[화면: 슬랙 알림 스크린샷]**

"완성되면 이렇게 알림이 옵니다.

(화면: 슬랙 알림)
'⚠️ 부정 리뷰 감지
 플랫폼: 쿠팡
 상품: [상품명]
 별점: ★☆☆☆☆
 내용: 배송이 너무 느려요. 2주 걸림.
 감성: 부정 (confidence 94%)

 📝 추천 답변:
 고객님, 불편을 드려 진심으로 죄송합니다.
 해당 주문 건의 배송 지연 원인을 확인하고
 즉시 보상 조치를 진행하겠습니다...'

이게 리뷰 올라오고 10초 만에 옵니다. 답변 복붙해서 바로 달면 끝."

## [4:00~11:00] 라이브 빌드 (7분)

**[화면: 무편집 화면 녹화]**

"만들겠습니다.

**4:00~5:30 — 리뷰 소스 연결**
쿠팡/스마트스토어 리뷰 페이지를 크롤링하는 트리거.
(화면: 웹 크롤링 노드 or RSS 피드 설정)
'15분마다 새 리뷰 체크'로 설정.

**5:30~7:30 — AI 감성 분석**
새 리뷰가 감지되면 → AI에게 보냄.
프롬프트:
'이 리뷰의 감성을 분석해줘.
분류: 긍정/부정/중립
신뢰도: %
핵심 키워드: 1~3개
별점 3 이하이면 "긴급" 태그 추가'

(화면: AI 노드 + 프롬프트 입력)

**7:30~9:00 — 조건 분기: 부정만 알림**
IF 감성 = 부정 OR 별점 ≤ 3 → 알림 경로
ELSE → 일일 요약 리포트에 저장

(화면: 조건 분기 설정)

**9:00~10:30 — 알림 + 답변 초안**
슬랙으로 알림 보내기.
답변 초안 자동 생성:
프롬프트: '이 부정 리뷰에 대해 친절하고 공손한 사과 답변을 작성해줘. 구체적 보상 약속 포함.'

(화면: 알림 + 답변 생성 노드)

**10:30~11:00 — 테스트**
가상 부정 리뷰 입력 → 10초 후 슬랙 알림 도착 → 답변 초안 확인.
완벽."

## [11:00~12:30] 보너스: 일일 리뷰 리포트

'오늘의 리뷰 요약:

이 리포트를 매일 받으면 고객이 뭘 좋아하고 뭘 싫어하는지 데이터로 알 수 있습니다."
