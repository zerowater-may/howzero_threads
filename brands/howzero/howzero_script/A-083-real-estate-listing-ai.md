## 대본

### 훅 (0:00–0:40)
공인중개사분들, 매물 엑셀 몇 행이세요? 500개? 1,000개? 그걸 손으로 관리하면서 고객한테 매물 추천도 손으로 하고 계시죠. 매물 올리고, 사진 편집하고, 네이버부동산에 등록하고, 고객 문의 응대하고… 중개가 아니라 데이터 입력 알바를 하고 계신 겁니다. 이걸 AI로 바꾼 중개사무소 이야기를 해드리겠습니다.

### 문제 정의 (0:40–3:00)
중개사무소 업무의 핵심 병목 세 가지. 첫째, 매물 등록 — 사진 촬영 → 편집 → 설명 작성 → 플랫폼별 등록. 매물 1건당 최소 30분. 둘째, 고객-매물 매칭 — 고객 조건(예산, 지역, 평수, 층수 등)과 매물을 머리로 매칭. 매물이 많으면 기억에 의존. 셋째, 팔로업 — 관심 고객에게 신규 매물 안내. 수동으로 하면 빠뜨림.

### 자동화 설계 (3:00–8:30)
**매물 등록 자동화**: 사진 촬영 → AI가 자동 보정(밝기, 왜곡) + 방 구조 태깅 + 매물 설명 자동 생성(평수, 방향, 특장점 기반) + 네이버부동산/직방 동시 등록.

**고객-매물 매칭**: 고객 DB에 조건 입력 → 신규 매물 등록 시 AI가 자동 매칭 스코어 계산 → 80점 이상이면 자동 알림(카카오톡/문자).

**시세 분석**: 국토교통부 실거래가 API + 네이버부동산 호가 데이터 자동 수집 → 매물별 적정가 범위 자동 산출 → 매도인에게 근거 기반 가격 제안.

### 데모/사례 (8:30–13:30)
경기 소재 C공인중개사사무소. 대표 1명 + 직원 1명. 관리 매물 약 350건. 자동화 전: 매물 등록 1건 30분, 고객 매칭 감에 의존, 신규 매물 알림 수동. 자동화 후: 매물 등록 1건 8분(사진 촬영만 사람), 자동 매칭으로 고객 연락 적중률 35%→62%, 신규 매물 자동 알림으로 문의 40%↑.

도구: 구글 시트 + Make + GPT-4 Vision + 카카오 알림톡 API. 월 운영비 약 6만 원.
