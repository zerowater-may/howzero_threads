## 대본

### 훅 (0:00–0:40)
사업을 팔 생각이 있으신 분. 아니, 지금 없더라도 언젠가 팔거나 넘길 수 있잖아요. 근데 "팔려는 순간"에 준비하면 늦습니다. 매수자가 보는 건 시스템입니다. 사장님 머릿속에만 있는 사업은 가치가 낮습니다. AI로 사업을 시스템화하면, 기업 가치가 올라가고, Exit 준비가 자동으로 됩니다.

### 문제 정의 (0:40–3:00)
사업 매각의 병목. 첫째, 대표 의존도 — 매수자는 "이 사업이 대표 없이 돌아가나?"를 봅니다. 대표 의존도가 높으면 가치가 떨어집니다. 둘째, 데이터 정리 — 매출, 비용, 고객, 계약 데이터가 정리되어 있지 않으면 실사(Due Diligence)에서 걸립니다. 셋째, 프로세스 문서화 — "어떻게 운영되는 사업인가"를 설명할 수 없으면 매수자가 리스크로 봅니다.

### 자동화 설계 (3:00–8:30)
**재무 데이터 자동 정리**: 매출, 비용, 이익 데이터 자동 수집 → 월별/분기별 재무 리포트 자동 생성 → 추세 분석 자동 → 실사 요청 시 즉시 제출 가능한 형태 유지.

**KPI 대시보드 자동화**: 핵심 지표(MRR, 고객 이탈률, CAC, LTV, 마진율) 실시간 자동 추적 → 매수자가 한 눈에 사업 건전성 파악.

**프로세스 자동 문서화**: 모든 자동화 워크플로우가 곧 프로세스 문서. Make 시나리오 자체가 "이 사업이 이렇게 돌아갑니다"의 증거. 추가로 AI가 각 자동화의 목적, 입출력, 담당자를 정리한 문서 자동 생성.

**고객 데이터 자동 정리**: 고객 리스트, 세그먼트, 이탈률, LTV 자동 계산 → 고객 자산 가치 자동 산출 → 계약 현황(정기 구독, 연 계약) 자동 정리.

**리스크 자동 점검**: 법적 리스크(계약, 규제 준수), 기술 리스크(특정 도구 의존도), 인적 리스크(핵심 인재 의존도) → AI가 분기별 리스크 리포트 자동 생성.

### 데모/사례 (8:30–13:30)
SaaS 스타트업 E. ARR 8억, 직원 10명. 3년 후 Exit 목표. 자동화 전: 재무 데이터 엑셀 분산, 프로세스 문서 없음, 대표가 세일즈·CS·전략 전부, 기업 가치 평가 ARR 3배 = 24억. 2년간 시스템화 진행. 자동화 후: 재무·KPI 실시간 대시보드, 프로세스 100% 문서화+자동화, 대표 없이 운영 가능, 고객 이탈률 8% → 3%.

기업 가치 재평가: ARR 12억(성장) x 5배(시스템화 프리미엄) = 60억. 시스템화로 멀티플이 3배 → 5배로 상승. 실제 M&A 진행 중.

도구: 구글 시트 + Make + GPT-4 + 노션 + 슬랙. 월 운영비 약 10만 원.
