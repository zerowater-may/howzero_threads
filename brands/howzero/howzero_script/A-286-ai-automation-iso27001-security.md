## 대본

### 훅 (0:00–0:40)
AI 자동화 도입하고 싶은데, ISO 27001 인증 때문에 못 하겠다고요? "외부 API 쓰면 인증 날아가는 거 아니야?" 결론부터 말씀드립니다. AI 자동화와 ISO 27001은 충돌하지 않습니다. 오히려 AI로 보안 관리를 자동화하면 인증 유지가 더 쉬워집니다. 접근 권한 자동 점검, 로그 자동 분석, 취약점 자동 스캔. 심사 때 자료 정리하느라 야근하던 거, AI가 끝냅니다. 코딩 제로.

### 문제 정의 (0:40–3:00)
ISO 27001 유지의 병목. 첫째, 연간 심사 준비 — 증빙 자료 수집에 매번 2~3주. 셋째, 접근 권한 관리 — 퇴사자 계정 비활성화, 권한 변경 추적 수동. 넷째, 보안 로그 분석 — 수만 건의 로그를 사람이 봅니다. 이상 징후를 놓칩니다. 다섯째, AI 도입 시 데이터 흐름 — 외부 API로 데이터가 나가면 통제 범위 밖이 됩니다.

### 자동화 설계 (3:00–9:00)
**접근 권한 자동 점검**: HR 시스템 + 계정 관리 연동 → 퇴사·부서 이동 시 권한 자동 변경/회수 → 주간 권한 점검 리포트 자동 생성 → "미처리 퇴사자 계정 0건" 자동 검증.

**보안 로그 AI 분석**: 서버·앱·네트워크 로그 자동 수집 → AI가 이상 패턴 자동 감지 → "새벽 3시 대량 데이터 다운로드 시도" 즉시 알림 → 월간 보안 분석 리포트 자동 생성.

**심사 증빙 자동 수집**: 통제 항목별 증빙 자료 자동 수집+정리 → 연간 심사 시 폴더 구조 자동 생성 → "A.9 접근 통제 — 증빙 12건 자동 첨부" → 심사 준비 2주 → 2일.

**AI 데이터 흐름 통제**: AI 자동화 워크플로우 내 데이터 분류 자동화 → 민감 정보 자동 마스킹 → 외부 API 전송 전 PII 제거 → 데이터 흐름 맵 자동 생성+업데이트 → 심사관 질의 대응 자동화.
