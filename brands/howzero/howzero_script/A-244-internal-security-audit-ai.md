## 대본

### 훅 (0:00–0:40)
사내 보안 감사, 1년에 한 번 하시죠? 외부 감사 들어오기 전에 급하게 준비하고, 로그 뒤지고, 권한 정리하고. 2주는 걸립니다. 그런데 보안 사고는 감사와 감사 사이에 터집니다. AI가 매주 자동으로 접근 권한을 점검하고, 이상 행위를 실시간 감지하고, 감사 보고서를 자동 생성합니다. 감사 주기 12개월에서 1주. 보안은 빈도가 생명입니다.

### 문제 정의 (0:40–3:00)
사내 보안 감사의 현실. 첫째, 감사 주기가 너무 김 — 연 1~2회. 그 사이에 퇴사자 계정이 살아있고, 불필요한 권한이 쌓입니다. 둘째, 로그 분석이 수동 — 수천 건의 접근 로그를 사람이 확인합니다. 이상 패턴을 놓칩니다. 셋째, 보고서 작성이 고통 — 감사 결과를 문서화하는 데만 3~5일. 넷째, 후속 조치 추적 불가 — 감사에서 지적된 사항이 실제로 개선됐는지 추적이 안 됩니다.

### 자동화 설계 (3:00–8:00)
**접근 권한 자동 점검**: 전체 시스템 계정 목록 + HR 재직자 목록 자동 대조 → 퇴사자·휴직자 계정 자동 플래그 → 부서 이동 후 이전 부서 권한 유지자 감지 → 과도한 관리자 권한 보유자 리스트업 → 매주 자동 실행.

**이상 행위 실시간 감지**: 로그인 패턴 AI 학습(시간·위치·빈도) → 평소와 다른 접근 감지(새벽 접속, 해외 IP, 대량 다운로드) → 위험도 자동 스코어링 → 고위험 건 즉시 보안 담당자 알림.

**자동 감사 보고서**: 주간/월간 점검 결과 자동 집계 → 위험 수준별 분류(긴급·주의·참고) → 이전 감사 대비 변화 추이 자동 비교 → PDF 보고서 자동 생성.

**후속 조치 추적**: 지적 사항별 담당자 자동 배정 → 조치 기한 설정 → 미조치 건 자동 리마인더 → 완료율 자동 집계.

### 데모/사례 (8:00–12:30)
IT 서비스 기업 T. 직원 120명, 내부 시스템 15개. 자동화 전: 보안 감사 연 1회(2주 소요), 퇴사자 계정 방치 평균 45일, 이상 행위 감지 0건(아무도 안 봄), 감사 보고서 작성 5일. 자동화 후: 보안 점검 매주 자동, 퇴사자 계정 방치 0일(HR 연동 즉시 플래그), 이상 행위 감지 월 평균 3건, 감사 보고서 자동 생성(1시간).

실제 보안 사고 예방 사례: 퇴사 예정자가 대량 파일 다운로드 시도 → AI 즉시 감지 → 접근 차단 → 데이터 유출 방지. 이 한 건이 감사 자동화 투자비 3년치 이상의 가치.

도구: 구글 시트 + Make + GPT-4 + 각 시스템 API/로그 연동. 월 운영비 약 10만 원.
