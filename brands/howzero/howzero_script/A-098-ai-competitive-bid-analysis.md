# A-098: 유튜브 롱폼 — "AI로 경쟁 입찰가 분석 자동화"
> **유형**: 유튜브 롱폼 | **길이**: 17분
## 메타데이터
- **제목**: AI로 경쟁 입찰가 분석 자동화 — 낙찰률 높이는 데이터 전략
- **썸네일 텍스트**: 입찰가 AI 분석 | 낙찰률 UP
- **태그**: 입찰, 경쟁 분석, 낙찰률, 공공 입찰, 견적, AI, 하우제로
## 대본

### 훅 (0:00–0:40)
입찰 넣을 때 가격을 감으로 쓰고 있지 않습니까? 경쟁사가 얼마에 넣는지 모르니까 너무 높게 쓰면 탈락, 너무 낮게 쓰면 수익이 없습니다. AI로 과거 낙찰 데이터를 분석해서 최적 입찰가를 자동 산출하는 시스템. 낙찰률 22%에서 41%로 올린 실제 사례 보여드리겠습니다.

### 문제 정의 (0:40–3:00)
입찰의 핵심 어려움. 첫째, 정보 비대칭 — 경쟁사 입찰 패턴을 모름. 둘째, 데이터 방치 — 나라장터 등에 과거 낙찰 데이터가 공개되어 있는데 활용 안 함. 셋째, 수동 계산 — 예정가격, 복수예비가격, 투찰률 등을 수동 분석. 넷째, 경험 의존 — 10년 차 담당자 감에 의존.

### 자동화 설계 (3:00–8:30)
**데이터 수집**: 나라장터/조달청 과거 낙찰 데이터 자동 크롤링 → 업종별, 규모별, 지역별 DB 구축.

**패턴 분석**: AI가 과거 데이터에서 — 평균 낙찰률, 투찰 업체 수 패턴, 시즌별 변동, 예정가격 대비 낙찰가 비율 자동 분석.

**최적가 산출**: 신규 입찰 건 입력 → AI가 유사 과거 사례 매칭 → 최적 투찰률 범위 제안 → 예정가격 기준 투찰가 자동 계산.

**경쟁사 분석**: 자주 만나는 경쟁사별 투찰 패턴 분석 → 경쟁사가 참여할 확률 + 예상 투찰가 범위 예측.

**결과 트래킹**: 입찰 결과 자동 수집 → 예측 vs 실제 비교 → 모델 자동 개선.

### 데모/사례 (8:30–13:30)
건설업 Q사. 연간 입찰 참여 약 80건. 자동화 전: 낙찰률 22%. 입찰가 산출 담당자 감에 의존. 자동화 후: 과거 3년 낙찰 데이터 4만 건 분석. AI 추천 투찰률 적용. 낙찰률 22%→41%. 연간 추가 수주 약 12건, 매출 8억 원↑. 입찰가 분석 시간 건당 3시간 → 15분.

도구: 파이썬 크롤러(1회 구축) + 구글 시트 + GPT-4 + Looker Studio. 월 운영비 약 10만 원.

### 효과 정리 (13:30–15:30)
- 낙찰률 22% → 41%
- 입찰가 분석 시간 92% 절감
- 연간 추가 수주 매출 약 8억 원
- 경쟁사 패턴 데이터화
