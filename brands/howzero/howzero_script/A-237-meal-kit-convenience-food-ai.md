## 대본

### 훅 (0:00–0:40)
밀키트 브랜드 운영하시는 분, 이번 주 얼마나 만들어야 하는지 어떻게 결정하세요? 감으로? 지난주 판매량 기준? 유통기한 짧은 식품은 수요 예측 실패 = 폐기 또는 품절입니다. AI가 수요를 예측하고, 생산량을 자동 결정하고, 구독 관리까지 해주는 시스템. 폐기율 70% 감소.

### 문제 정의 (0:40–3:00)
밀키트·간편식의 병목. 첫째, 수요 예측 — 유통기한이 짧아서 과잉 생산은 폐기, 과소 생산은 기회 손실. 둘째, 구독 관리 — 구독 고객 배송 스케줄, 메뉴 변경, 해지 처리가 복잡. 셋째, 레시피·콘텐츠 — 신메뉴 개발, 조리법 카드, SNS 콘텐츠를 만들 시간이 없습니다.

### 자동화 설계 (3:00–8:00)
**수요 예측 자동화**: 과거 판매 데이터 + 구독 스케줄 + 날씨 + 요일 효과 + 프로모션 계획 → AI가 메뉴별 일일 수요 예측 → 생산량 자동 결정 → 식자재 발주 자동 생성.

**구독 관리 자동화**: 구독 결제 자동 → 배송 전 "이번 주 메뉴" 안내 + 변경 옵션 제공 → 고객별 알레르기/선호 자동 반영 → 해지 요청 시 리텐션(메뉴 변경 제안/쿠폰) 자동.

**레시피 콘텐츠 자동화**: 신메뉴 출시 → AI가 조리법 카드(단계별 사진+설명) 자동 생성 → 블로그·인스타용 콘텐츠 자동 변환 → 고객 리뷰 수집 → 인기 메뉴 자동 식별.

**품질·배송 자동 관리**: 제조 → 포장 → 발송 각 단계 체크리스트 자동화 → 배송 완료 후 "맛있게 드셨나요?" 자동 발송 → 불만 접수 시 즉시 CS팀 알림 + 자동 보상 제안.

밀키트 브랜드 Y. 메뉴 8종, 주 생산량 3,000팩, 구독 고객 500명. 자동화 전: 수요 예측 수동(폐기율 18%), 구독 관리 주 10시간, 신메뉴 콘텐츠 건당 4시간, 구독 해지율 월 12%. 자동화 후: AI 수요 예측(폐기율 5%), 구독 관리 자동(주 1시간 확인), 콘텐츠 건당 30분, 구독 해지율 월 6%.

폐기 비용 월 300만 원 → 80만 원. 구독 고객 500명 → 850명(6개월). 신메뉴 출시 주기 월 1회 → 격주 1회.
