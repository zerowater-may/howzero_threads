## 대본

### 훅 (0:00–0:40)
치과 원장님, 하루에 전화 몇 통 받으세요? 예약 잡고, 변경하고, 노쇼 관리하고, 차트 꺼내고… 진료 시간보다 행정 시간이 더 긴 병원이 태반입니다. 제가 실제로 자동화 컨설팅한 치과 한 곳은, 데스크 직원이 하루 평균 127통의 전화를 받고 있었습니다. 지금은 그 중 89통을 AI가 처리합니다. 오늘 그 시스템을 전부 공개합니다.

### 문제 정의 (0:40–3:00)
의원·치과의 3대 행정 병목: 첫째, 예약 관리. 전화 → 수기 기록 → 중복 예약 → 노쇼. 둘째, 차트·진료 기록 정리. 진료 끝나면 수기 입력, 빠뜨리면 보험 청구에 문제. 셋째, 환자 리마인드. 다음 내원일, 스케일링 주기, 보철 체크 시기… 일일이 기억 못 합니다. 이 세 가지를 각각 자동화하면, 데스크 인력 최소 1명 분량의 업무가 사라집니다. 과장이 아닙니다. 숫자로 증명하겠습니다.

### 자동화 설계 (3:00–8:00)
**1단계 — 예약 자동화**: 네이버 예약/카카오톡 챗봇 연동. 환자가 직접 빈 시간대 확인 + 예약. 구글 캘린더 또는 자체 DB와 실시간 동기화. 중복 예약 자동 차단. 예약 확정 시 카카오 알림톡 자동 발송. 예약 24시간 전 리마인드 자동 발송. 노쇼 시 대기 환자에게 자동 안내.

**2단계 — 차트 메모 자동화**: 진료 중 음성 녹음(동의 하에) → Whisper API로 텍스트 변환 → GPT가 SOAP 노트 형식으로 자동 정리 → 원장님 확인 후 저장. 키워드 태깅 자동화(크라운, 임플란트, 신경치료 등).

**3단계 — 리마인드 자동화**: 환자별 다음 내원 주기 DB 관리. 스케일링 6개월, 교정 4주, 보철 체크 1년 등. 주기 도래 3일 전 자동 문자/알림톡. 미내원 환자 1주·2주·4주 자동 팔로업.

### 데모/사례 (8:00–13:00)
실제 구축 사례: 서울 소재 치과 A. 원장 1명 + 위생사 2명 + 데스크 1명. 월 평균 내원 환자 480명. 자동화 전: 데스크 직원 하루 8시간 중 6시간이 전화+예약+차트 정리. 자동화 후: 전화 70% 감소(온라인 예약 전환). 차트 정리 시간 일 90분 → 20분. 리마인드 발송 수동 0건 → 자동 월 320건. 노쇼율 18% → 7%.

구축 도구: 구글 시트 + Make(Integromat) + 카카오 알림톡 API + Whisper API + GPT-4. 코딩 제로. 월 운영비 약 8만 원.
