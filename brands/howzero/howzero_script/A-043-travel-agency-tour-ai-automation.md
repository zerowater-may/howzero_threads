## [1:30~4:00] 문제 정의 — 여행사의 3대 반복 업무 (2분 30초)

**[화면: 프로세스 맵 시각화]**

**대본:**
"여행사의 반복 업무를 정리하면 크게 3가지입니다.

**1번: 견적 문의 응답**
카카오톡, 전화, 인스타 DM으로 오는 견적 문의. '일본 3박 4일 2인 얼마예요?' 이런 거요. 같은 질문을 하루에 20번 받습니다. 매번 수동으로 검색하고 견적서 만듭니다.

**2번: 맞춤 일정 설계**
고객마다 원하는 게 다릅니다. '아이가 있어서 이동 적게', '맛집 위주로', '쇼핑 시간 넉넉히'. 이걸 일일이 손으로 짜요. 건당 1시간.

**3번: 예약 확인 및 안내**
출발 전 안내 메시지, 현지 날씨, 환율 정보, 비자 확인. 고객 한 명당 5~10번의 연락. 빠뜨리면 클레임.

이 세 가지에 직원 시간의 70%가 들어갑니다. 정작 중요한 '신규 상품 개발'이나 '파트너 관리'는 못 해요."

## [4:00~7:30] 자동화 설계 — 3단계 시스템 (3분 30초)

**[화면: 자동화 아키텍처 다이어그램]**

**대본:**
"제가 설계한 여행사 AI 자동화 시스템은 3단계입니다.

**1단계: AI 견적 봇**
카카오톡이나 웹사이트에 견적 봇을 연결합니다. 고객이 '목적지, 인원, 날짜, 예산'을 입력하면 AI가 즉시 견적 초안을 만들어줍니다.

어떻게? 기존에 만들었던 견적서 100~200개를 AI에 학습시킵니다. 항공권 가격 범위, 숙소 등급별 가격, 현지 투어 비용 — 이런 데이터를 구조화해서 넣으면 AI가 패턴을 파악합니다.

'발리 4박 5일 2인 200만 원 예산' 이라고 입력하면 3초 만에 견적 초안이 나옵니다. 직원은 검토만 하면 돼요.

**2단계: AI 일정 설계기**
이게 핵심입니다. 고객 요구사항을 입력하면 AI가 맞춤 일정을 자동 생성합니다.

비결은 '일정 템플릿 데이터베이스'입니다. 지역별, 테마별로 기본 일정 블록을 만들어 놓고, AI가 고객 조건에 맞게 블록을 조합합니다.

'오사카 3박 4일, 아이 동반, 맛집 위주' → AI가 키즈 프렌들리 식당 + 적절한 이동 동선 + 체크인 시간 고려해서 일정을 짜줍니다.

**3단계: 자동 안내 시스템**
출발 7일 전, 3일 전, 1일 전, 당일 — 각 시점에 맞는 안내 메시지를 자동 발송합니다. 날씨, 환율, 준비물, 긴급 연락처까지 자동으로 포함됩니다.

이 세 단계를 연결하면 견적 문의부터 출발 안내까지 전 과정이 반자동화됩니다."

## [7:30~11:00] 데모 — 실제 구축 과정 (3분 30초)

**[화면: 화면 녹화 — Make/Zapier + GPT API + 구글 시트]**

**대본:**
"직접 보여드리겠습니다. 코딩 없이 만드는 과정입니다.

(화면 녹화 시작)

먼저 구글 시트에 견적 데이터베이스를 만듭니다. 칼럼은 '목적지, 기간, 인원, 예산 범위, 항공 예상가, 숙소 예상가, 현지비용 예상가, 총견적'. 기존 견적서 데이터를 여기 정리합니다.

다음, Make 자동화를 설정합니다.

시나리오 1: 카카오톡 메시지 수신 → GPT API로 고객 의도 파싱 → 구글 시트에서 유사 견적 검색 → GPT가 맞춤 견적서 초안 생성 → 직원에게 슬랙 알림

이거 설정하는 데 30분이면 됩니다.

시나리오 2: 예약 확정 시 → 구글 캘린더에 일정 등록 → 출발 7일 전, 3일 전, 1일 전 자동 안내 메시지 발송

이것도 20분이면 세팅 끝납니다.

(화면 녹화: 실제 테스트)

테스트해 볼게요. '도쿄 3박 4일 2인 150만 원'이라고 입력합니다.

... 3초 후 견적 초안이 나옵니다. 항공 40만 원, 숙소 45만 원, 현지비용 35만 원, 여행자보험 3만 원. 총 123만 원. 예산 내입니다.

직원은 이걸 확인하고 '전송' 버튼만 누르면 됩니다. 30분 걸리던 게 3분으로 줄었습니다."
