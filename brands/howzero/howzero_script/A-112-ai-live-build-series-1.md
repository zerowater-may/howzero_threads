- **썸네일 텍스트**: 라이브 빌드 #1 | 10분 구축
- **태그**: 라이브 빌드, 실전, 고객 문의 분류, 자동화 구축, Make, GPT, 하우제로
## 대본

### 훅 (0:00–0:50)
오늘부터 새로운 시리즈를 시작합니다 — "10분 라이브 빌드." 말이 아니라 실제로 만듭니다. 화면 켜고, 10분 타이머 켜고, 처음부터 끝까지 AI 자동화 시스템을 구축하는 모습을 보여드립니다. 오늘의 미션: 고객 문의 이메일이 들어오면 AI가 자동으로 분류(결제/배송/반품/기술/일반)하고, 담당자에게 자동 배정하는 시스템. 시작합니다.

### 시작 전 설명 (0:50–3:00)
오늘 만들 시스템의 구조. 이메일 수신(Gmail) → AI 분류(GPT-4) → 카테고리별 담당자에게 슬랙 알림 + 구글 시트 로그. 사용 도구: Gmail + Make + GPT-4 API + 슬랙 + 구글 시트. 비용: 월 약 3만 원. 자, 10분 타이머 시작합니다.

### 라이브 빌드 (3:00–13:00)
**[0:00–2:00] Make 시나리오 생성**: Gmail 트리거 설정 → 새 이메일 수신 시 실행. 제목+본문 추출.

**[2:00–5:00] GPT-4 분류 모듈**: OpenAI 모듈 추가 → 프롬프트 작성: "다음 고객 문의를 분류하세요. 카테고리: 결제, 배송, 반품, 기술문의, 일반. JSON으로 응답: {category, urgency, summary}" → 테스트 실행 → 분류 결과 확인.

**[5:00–7:00] 슬랙 알림**: 카테고리별 분기(Router 모듈) → 결제 → #payment-team, 배송 → #shipping-team... 각 채널에 자동 메시지(카테고리, 긴급도, 요약, 원문 링크).

**[7:00–9:00] 구글 시트 로그**: 날짜, 발신자, 카테고리, 긴급도, 요약, 처리 상태를 자동 기록.

**[9:00–10:00] 최종 테스트**: 실제 테스트 이메일 발송 → Gmail 수신 → Make 실행 → GPT-4 분류 → 슬랙 알림 → 구글 시트 기록. 10분 완료.

### 결과 리뷰 (13:00–16:00)
구축 완료. 실제 작동하는 모습을 한 번 더 시연. 이 시스템이 적용되면 — CS 담당자가 이메일을 하나하나 읽고 분류하는 시간 제로. 긴급 문의 자동 에스컬레이션. 처리 현황 자동 대시보드. 확장 가능: 카카오톡, 네이버 톡톡 등 다른 채널 추가도 동일한 방식.

지금 보신 것처럼, AI 자동화는 복잡하지 않습니다. 10분이면 실제 작동하는 시스템을 만들 수 있습니다. 코딩 한 줄도 없었습니다.
