# A-086: 유튜브 롱폼 — "AI로 입찰/제안서 자동 작성"
> **유형**: 유튜브 롱폼 | **길이**: 18분
## 메타데이터
- **제목**: AI로 입찰·제안서 자동 작성 — 수주율 높이는 실전 시스템
- **썸네일 텍스트**: 제안서 AI 자동화 | 수주율 2배
- **태그**: 입찰, 제안서, RFP, AI 자동 작성, 수주, 하우제로
## 대본

### 훅 (0:00–0:40)
제안서 하나 쓰는 데 며칠 걸리세요? 3일? 5일? RFP 분석하고, 과거 제안서 뒤지고, 팀원 분담하고, 합치고, 디자인하고… 그런데 수주율이 20%면 5건 쓸 때 4건은 버리는 겁니다. AI로 제안서 초안을 4시간 만에 만들고, 수주율을 2배로 올린 방법을 보여드리겠습니다.

### 문제 정의 (0:40–3:00)
제안서 작성의 3대 병목. 첫째, RFP 분석 — 수십 페이지의 요구사항을 읽고 핵심을 추출하는 데 반나절. 둘째, 콘텐츠 작성 — 회사 소개, 수행 실적, 기술 방안, 일정, 비용 산출. 매번 처음부터 쓰거나 과거 제안서를 짜깁기. 셋째, 품질 편차 — 담당자 역량에 따라 제안서 품질이 들쑥날쑥.

### 자동화 설계 (3:00–9:00)
**RFP 자동 분석**: RFP PDF 업로드 → AI가 핵심 요구사항 추출 + 평가 배점 분석 + 필수 제출 서류 체크리스트 + 경쟁 포인트 도출.

**제안서 초안 생성**: 과거 수주 제안서 DB 구축(벡터화) → RFP 요구사항에 맞는 과거 콘텐츠 자동 매칭 + 신규 내용 생성 → 구조화된 제안서 초안 자동 생성.

**품질 체크**: AI가 RFP 요구사항 vs 제안서 내용 1:1 대조 → 누락 항목 알림 + 평가 배점 기준으로 보완 제안.

**자동 포맷팅**: 회사 CI에 맞춰 자동 디자인 + 목차·페이지 번호·헤더·푸터 자동 생성.

### 데모/사례 (9:00–14:00)
IT 서비스 기업 F사. 월 평균 제안서 6건 작성. 자동화 전: 제안서 1건당 평균 5일(40시간), 수주율 22%. 자동화 후: RFP 분석 4시간→30분. 초안 생성 20시간→4시간. 품질 체크 자동. 전체 소요 5일→1.5일. 수주율 22%→38%(AI 품질 체크로 누락 항목 제로).

도구: 노션 + GPT-4 + Pinecone + Make + 구글 슬라이드 API. 월 운영비 약 15만 원.
