### [1:30~4:00] 문제 정의 — 물류 업무의 현실

"중소 이커머스의 물류 프로세스:

1. **주문 확인** — 플랫폼별로 로그인해서 확인 (스마트스토어, 쿠팡, 자사몰)
2. **송장 번호 생성** — 배송사 시스템에서 출력
3. **고객 알림** — '배송이 시작되었습니다' 문자/카톡
4. **배송 추적** — 문제 발생 시 확인 (지연, 분실)
5. **반품/교환** — 접수 → 회수 → 재발송

하루 100건 기준:
1~3번에 3시간. 4~5번에 1시간. = 매일 4시간.

직원 1명이 온종일 물류만.
이 직원이 다른 일을 할 수 있다면?"

### [4:00~8:00] 자동화 설계 — 4개 시스템

"**시스템 1: 다채널 주문 통합**
- 스마트스토어 + 쿠팡 + 자사몰 주문을 하나의 시트로 자동 통합
- 중복 확인, 주소 정리, 상품 매핑 자동

**시스템 2: 송장+배송 알림 자동화**
- 통합 주문 데이터 → 배송사 API 연동 → 송장 자동 생성
- 고객에게 배송 시작 알림 자동 발송
- 배송 완료 시 자동 알림 + 리뷰 요청

**시스템 3: 배송 이상 감지**
- 출고 후 3일 이내 배송 완료 안 된 건 자동 감지
- 지연 건 → 자동 추적 + 고객에게 안내 메시지
- 분실 의심 건 → 배송사에 자동 문의

**시스템 4: 반품/교환 자동 관리**
- 반품 신청 접수 → 자동 분류 (고객 변심/상품 하자)
- 반품 배송비 자동 계산
- 회수 완료 → 환불 처리 → 고객 알림 자동"

### [8:00~14:00] 라이브 데모

"[시스템 1 — 주문 통합]
(화면: 3개 플랫폼 주문 데이터)

'이 3개 플랫폼 주문 데이터를 하나의 통합 시트로 만들어줘.
컬럼: 주문번호, 고객명, 연락처, 주소, 상품, 수량, 결제금액
중복 주문은 표시해줘.'

(실행 — 150건 주문이 하나의 깔끔한 시트로)

수동이면 1시간. AI는 5초.

[시스템 2 — 배송 알림]
'오늘 출고 완료된 80건에 대해 고객별 배송 알림 메시지를 만들어줘.
포함: 고객명, 상품명, 송장번호, 예상 도착일
톤: 친근하고 정보 전달 중심'

(실행 — 80개 개인화된 알림 메시지)

[시스템 3 — 이상 감지]
'현재 배송 중인 건 중에서 출고 후 3일 이상 경과 + 배송 미완료 건을 찾아줘.
각 건에 배송사 문의 메시지 초안도 만들어줘.'

7건 감지. 각각에 문의 메시지까지 자동.

[시스템 4 — 반품]
'이번 주 반품 신청 12건을 분류해줘.
고객 변심 / 상품 하자 / 오배송으로 나누고,
각각 반품 배송비와 처리 안내 메시지를 만들어줘.'

12건 분류 + 안내 메시지 자동.
수동이면 30분. AI는 10초."
