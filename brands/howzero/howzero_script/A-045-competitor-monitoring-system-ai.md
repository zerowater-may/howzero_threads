## [1:30~4:00] 문제 정의 — 수동 모니터링의 한계 (2분 30초)

**[화면: 프로세스 비교 다이어그램]**

**대본:**
"수동 모니터링의 문제는 3가지입니다.

**1번: 속도**
경쟁사가 변화를 줬을 때, 내가 확인할 때까지의 시간 갭. 이게 보통 1~3일입니다. 이커머스에서 하루는 매출에 직결됩니다.

**2번: 범위**
사람이 볼 수 있는 범위에 한계가 있습니다. 경쟁사 웹사이트, SNS, 리뷰 사이트, 뉴스, 채용 공고 — 전부 다 확인할 수 없어요. 채용 공고를 왜 확인하냐고요? 경쟁사가 'AI 엔지니어 급구'를 올렸으면 AI 신제품 준비 중이라는 뜻입니다.

**3번: 분석**
데이터를 모아도 분석이 안 됩니다. '경쟁사 A가 3개월간 가격을 5% 내렸는데, 리뷰 감성이 부정에서 긍정으로 바뀌고 있다' — 이런 트렌드를 사람이 잡아내기 어렵습니다.

AI는 이 세 가지를 전부 해결합니다. 실시간 + 넓은 범위 + 자동 분석."

## [4:00~7:30] 자동화 설계 — 4단계 시스템 (3분 30초)

**[화면: 시스템 아키텍처]**

**대본:**
"경쟁사 모니터링 AI 시스템은 4단계입니다.

**1단계: 데이터 수집**
경쟁사 웹사이트, SNS, 리뷰 플랫폼, 뉴스에서 데이터를 자동 수집합니다.

웹 스크래핑이라고 하는데요, 코딩 없이 가능합니다. Browse AI, Apify 같은 도구가 있어요. URL 넣고 '어떤 데이터를 가져올지' 클릭으로 선택하면 됩니다.

예를 들어 경쟁사 쇼핑몰의 가격표를 매일 1회 자동 수집. 가격 변동이 생기면 즉시 알림.

**2단계: 변화 감지**
수집한 데이터를 이전 데이터와 비교합니다. 가격 변동, 신규 상품 등록, 리뷰 급증 — 이런 '변화 이벤트'를 자동 감지합니다.

구글 시트에 데이터를 쌓고, Make 자동화가 '이전 값과 현재 값'을 비교합니다. 차이가 있으면 다음 단계로 넘깁니다.

**3단계: AI 분석**
GPT API가 변화의 의미를 분석합니다.

'경쟁사 A의 프리미엄 라인 가격이 15% 하락. 동시에 신규 브랜드 라인 출시. 기존 재고 소진을 위한 가격 조정으로 판단됨. 우리의 동일 카테고리 가격 대응 필요성: 중간. 이유: 타겟 고객층이 상이함.'

이런 분석이 자동으로 나옵니다.

**4단계: 리포트 + 알림**
주간 경쟁사 리포트를 자동 생성해서 이메일로 보내줍니다. 긴급 변화는 슬랙이나 카카오톡으로 즉시 알림."

**[화면: 화면 녹화]**

(화면 녹화 시작)

시나리오: 이커머스 경쟁사 3곳의 가격과 신상품을 모니터링합니다.

1단계: Browse AI에서 경쟁사 상품 페이지를 설정합니다. '상품명, 가격, 할인율' 데이터를 선택합니다. 수집 주기는 매일 오전 9시.

(설정 과정 화면 녹화)

2단계: 수집된 데이터가 구글 시트로 자동 전송됩니다. 시트에 '이전 가격' 열이 있어서, 변동이 있으면 자동 표시됩니다.

3단계: Make 시나리오를 만듭니다.
트리거: 구글 시트에 '가격 변동' 행이 추가될 때
액션 1: 변동 데이터를 GPT API로 보냄 → AI가 분석
액션 2: 분석 결과를 슬랙 채널에 전송

(테스트 실행)

테스트합니다. 경쟁사 B의 인기 상품이 29,900원에서 24,900원으로 변경됐다고 가정하겠습니다.

AI 분석 결과: '경쟁사 B 인기상품 16.7% 가격 인하. 해당 상품 카테고리에서 최저가 포지션 확보 시도. 우리 동일 카테고리 상품(27,900원) 대비 3,000원 저렴. 대응 옵션: 1) 가격 매칭, 2) 번들 프로모션, 3) 무대응(차별화 포인트 강조).'

이게 자동으로 나옵니다. 의사결정만 하면 됩니다."
