## 대본

### 훅 (0:00–0:40)
NPS 점수, 측정해보신 적 있습니까. "0~10점 중 추천 의향은?" 이 질문 하나로 고객 충성도를 측정하는 지표입니다. 문제는 점수만 알고 이유를 모른다는 겁니다. NPS 50점인데 왜 50점인지, 어떻게 60점으로 올릴지 모릅니다. AI로 NPS 자동 측정부터 이유 분석, 개선 액션까지 자동화한 시스템. 코딩 제로입니다.

### 문제 정의 (0:40–3:30)
NPS 관리의 3대 병목. 첫째, 측정 빈도 — 분기 1회, 연 1회 측정. 문제가 생겨도 3개월 후에나 파악. 둘째, 원인 분석 부재 — 점수는 나오는데 "왜?"를 모름. 주관식 답변이 있어도 수백 개를 사람이 읽고 분류 불가. 셋째, 개선 연결 안 됨 — NPS 결과를 보고서로 만들고 끝. 구체적 개선 액션으로 연결 안 됨.

### 자동화 설계 (3:30–10:00)
**NPS 자동 측정**: 주요 터치포인트(구매 후, 서비스 이용 후, CS 상담 후)에서 자동 NPS 설문 발송 → 0~10점 + 주관식 이유("왜 이 점수를 주셨나요?") → 응답 자동 수집 → 실시간 NPS 대시보드(추천자/중립자/비추천자 비율).

**AI 원인 분석**: 주관식 응답 수백~수천 건 → AI가 자동 분류(배송, 품질, 가격, CS, UI/UX 등 카테고리별) → 카테고리별 감성 분석(긍정/부정/중립) → 핵심 키워드 자동 추출 → 주간 인사이트 리포트("이번 주 NPS 하락 주 원인: 배송 지연 언급 45%↑").

**개선 액션 자동 연결**: AI가 NPS 하락 원인별 개선 방안 자동 제안("배송 지연 → 물류사 SLA 재검토 권장") → 담당 부서에 자동 할당 → 개선 실행 후 해당 카테고리 NPS 변화 자동 추적 → 개선 효과 자동 리포트.

**비추천자 자동 케어**: NPS 0~6점(비추천자) 즉시 알림 → AI가 응답 내용 분석 후 케어 방안 자동 제안 → CS팀에 자동 할당("고객 A: 품질 불만 — 교환+사과 쿠폰 권장") → 케어 후 재설문 자동 발송 → 전환율 추적(비추천자→중립/추천자).
