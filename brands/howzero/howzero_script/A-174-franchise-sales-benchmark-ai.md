## 대본

### 훅 (0:00–0:40)
프랜차이즈 본사 운영하시는 분들, 상위 10% 점포와 하위 10% 점포의 매출 차이가 보통 3~5배입니다. 같은 브랜드, 같은 메뉴, 같은 교육인데. 그 차이가 어디서 오는지 데이터로 분석하고 계십니까. 대부분 "위치가 좋아서"로 끝냅니다. AI로 점포별 매출 벤치마크, 성과 격차 원인 분석, 하위 점포 개선 방안까지 자동화한 시스템. 코딩 제로입니다.

### 문제 정의 (0:40–3:30)
프랜차이즈 본사의 3대 병목. 첫째, 데이터 분산 — 점포별 매출·비용·고객 데이터가 각각 다른 POS, 다른 형식. 통합 비교 불가. 둘째, 분석 부재 — "이번 달 전체 매출"만 봄. 점포별 상세 비교, 원인 분석 없음. 셋째, 개선 실행 — 하위 점포에 "매출 올려라"만 하고 구체적 방법 안 줌. 슈퍼바이저 역량 의존.

### 자동화 설계 (3:30–10:00)
**데이터 통합**: 전 점포 POS 데이터 자동 수집(일별 매출, 객단가, 주문 건수, 시간대별 매출, 메뉴별 매출) → 통합 대시보드 자동 생성 → 점포 규모·상권 유형별 그룹핑.

**벤치마크 자동 분석**: 같은 그룹 내 점포별 KPI 자동 비교(매출, 객단가, 회전율, 폐기율, 인건비율) → 상위 20% 점포의 공통 패턴 자동 추출("상위 점포 공통: 세트 메뉴 비율 45%↑, 피크 시간 회전율 1.3배↑") → 하위 점포별 개선 포인트 자동 식별.

**점포별 개선 리포트 자동 생성**: AI가 하위 점포별 맞춤 개선 방안 자동 작성("A점포: 세트 메뉴 추천 스크립트 도입 권장 — 상위 점포 대비 세트 비율 20%↓") → 슈퍼바이저에게 자동 배포 → 실행 후 성과 자동 추적(4주 단위).

**트렌드 예측**: 점포별 매출 추세 AI 예측(다음 달 예상 매출) → 하락 예상 점포 사전 경고 → 시즌 영향 자동 분석("작년 동기 대비 상권 X에서 매출 하락 트렌드 — 경쟁 점포 진입 가능성").

### 데모/사례 (10:00–14:30)
치킨 프랜차이즈 K. 가맹점 45개. 자동화 전: 매출 월 1회 엑셀 수집, 점포 비교 안 함, 하위 점포 관리 SV 감(感) 의존. 자동화 후: 전 점포 실시간 대시보드. 상위 점포 패턴 분석 → "배달 앱 리뷰 답변율"이 매출과 0.78 상관관계 발견. 하위 10개 점포에 맞춤 개선 리포트 배포 → 3개월 후 하위 점포 평균 매출 22%↑. 전체 프랜차이즈 매출 9%↑.
