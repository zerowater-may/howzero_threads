## 대본

### 훅 (0:00–0:40)
스터디카페 운영, 사실상 무인인데 완전 무인은 아니죠. 좌석 관리, 결제 오류, 시설 민원, 소음 관리… 사장님이 CCTV 보면서 원격으로 대응하고 있습니다. AI로 이 마지막 10%까지 자동화하면, 진짜 100% 무인 운영이 가능합니다.

### 문제 정의 (0:40–2:30)
스터디카페의 잔여 수동 업무. 첫째, 좌석 이탈 — 결제 후 자리 안 앉거나 초과 이용. 둘째, 시설 민원 — 에어컨, 조명, 프린터 고장 문의. 셋째, 소음 민원 — 통화, 음식 냄새 등 이용자 간 마찰. 넷째, 마케팅 — 비수기 좌석 가동률 40%까지 떨어짐.

### 자동화 설계 (2:30–7:00)
**좌석 관리 자동화**: 좌석 센서(또는 키오스크 연동) → 실시간 좌석 현황 대시보드 → 이용 시간 만료 10분 전 알림 → 초과 시 자동 추가 결제 안내.

**민원 AI 챗봇**: 카카오톡 챗봇으로 시설 민원 접수 → AI가 분류(시설/소음/결제/기타) → 간단 문제는 자동 안내(프린터 리셋 방법 등) → 복잡한 건 관리자 알림.

**소음 감지**: 데시벨 센서 → 기준치 초과 시 해당 구역 안내 방송 자동 + 반복 시 관리자 알림.

**비수기 마케팅**: 좌석 가동률 데이터 분석 → 비수기 자동 프로모션(시간대별 할인, 이벤트) → SNS+알림톡 자동 발송.

### 데모/사례 (7:00–11:30)
수도권 스터디카페 W. 좌석 80석. 자동화 전: 사장님 원격 대응 일 평균 2시간, 좌석 가동률 비수기 42%, 민원 응대 시간 평균 30분. 자동화 후: 사장님 대응 시간 일 15분, 좌석 가동률 비수기 65%(자동 프로모션), 민원 AI 자동 처리 75%.

도구: 키오스크 API + 구글 시트 + Make + GPT-4 + 카카오 챗봇. 월 운영비 약 6만 원.
