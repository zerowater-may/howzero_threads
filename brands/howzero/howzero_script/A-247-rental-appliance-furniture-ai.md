### 문제 정의 (0:40–3:00)
렌탈 사업의 병목. 첫째, 연체 관리 — 수백~수천 계약의 납부 상태를 추적하고, 연체 시 단계별(알림→독촉→경고→회수) 대응이 수동입니다. 둘째, 계약 갱신/해지 — 만기 도래 고객 파악, 갱신 제안, 해지 절차가 복잡합니다. 셋째, 기사 배정 — 설치·점검·회수 기사 스케줄을 수동으로 관리합니다. 넷째, 반환 제품 관리 — 회수된 제품의 상태 평가, 재렌탈 가능 여부 판단, 수리/폐기 결정이 사람 의존입니다.

### 자동화 설계 (3:00–8:00)
**연체 예측+자동 관리**: 고객별 납부 이력 AI 분석 → 연체 확률 예측(결제일 전 고위험 고객 사전 알림) → 연체 발생 시 자동 단계별 대응(D+1 알림톡 → D+7 전화 안내 예약 → D+30 경고 발송 → D+60 회수 프로세스 자동 시작).

**계약 라이프사이클 자동화**: 만기 90일 전 자동 안내 → 갱신/업그레이드/해지 옵션 제시 → 고객 선택에 따라 자동 처리 → 해지 시 회수 프로세스 자동 트리거.

**기사 자동 배정**: 설치·점검·회수 건 자동 집계 → 기사 위치·역량·일정 기반 최적 배정 → 고객에게 방문 일정 자동 안내 → 방문 결과 자동 기록.

**반환 제품 AI 상태 평가**: 기사가 회수 시 사진 촬영 → AI가 외관 상태 자동 등급 평가(A/B/C/D) → A·B등급 재렌탈 자동 등록 → C등급 수리 자동 접수 → D등급 폐기 처리.
