## 대본

### 훅 (0:00–0:40)
고객 클레임 하나 처리하는 데 평균 시간이 얼마인지 아십니까. 47분입니다. 접수, 확인, 내부 보고, 보상 결정, 통보. 하루에 클레임 10건이면 직원 하나가 클레임에만 매달립니다. AI로 클레임 접수부터 유형 분류, 보상 기준 자동 적용, 고객 통보까지 한 번에 처리하는 시스템. 코딩 제로입니다.

### 문제 정의 (0:40–3:30)
클레임 처리의 3대 병목. 첫째, 접수 채널 분산 — 전화, 이메일, 카카오, 인스타 DM. 어디서 뭐가 들어왔는지 추적 불가. 누락 발생. 둘째, 처리 기준 불일치 — 같은 유형 클레임인데 담당자마다 보상이 다름. "이전에는 환불해줬는데요" 2차 클레임 유발. 셋째, 처리 지연 — 상급자 승인 대기, 재고 확인, 배송 확인 등 내부 프로세스가 느림. 고객 분노 증폭.

### 자동화 설계 (3:30–10:00)
**통합 접수 자동화**: 전화(STT 전환), 이메일, 카카오, 인스타 DM → 모든 클레임을 하나의 구글 시트로 통합 접수 → 접수 즉시 고객에게 접수 확인+처리 예상 시간 자동 안내.

**AI 유형 분류·심각도 판정**: 클레임 내용을 GPT-4가 분석 → 유형 자동 분류(배송 지연, 품질 불량, 오배송, 서비스 불만, 환불 요청) → 심각도 자동 판정(1~5등급) → 긴급(등급 4~5)은 즉시 관리자 알림.

**보상 기준 자동 적용**: 유형별·심각도별 보상 매트릭스 사전 설정(배송 지연 3일 이내→5% 쿠폰, 품질 불량→전액 환불+재발송) → AI가 해당 건에 맞는 보상 자동 산출 → 보상 기준 내 건은 자동 승인 → 기준 초과 건만 관리자 승인 요청.

**고객 통보 자동화**: 보상 결정 즉시 고객에게 안내("품질 불량 확인 → 전액 환불+재발송 처리 완료") → 처리 만족도 자동 설문 → 불만족 시 2차 대응 트리거.

**월간 클레임 분석**: 유형별·제품별·기간별 클레임 트렌드 자동 분석 → 반복 클레임 상위 3개 자동 경고("이번 달 품질 불량 클레임 40%↑ — 제품 X 집중") → 품질팀/운영팀에 자동 리포트.
