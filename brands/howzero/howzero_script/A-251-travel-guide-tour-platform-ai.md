### 문제 정의 (0:40–3:00)
투어/여행 플랫폼의 병목. 첫째, 문의 응대 — 시차 때문에 해외 고객 응대가 늦어집니다. 느린 응답 = 이탈. 둘째, 투어 추천 — 수십 개 상품 중 고객에게 맞는 걸 추천하려면 취향·일정·인원·예산을 다 물어봐야 합니다. 셋째, 예약 관리 — 가이드 스케줄, 차량 배정, 인원 관리가 수동입니다. 넷째, 후기 관리 — 투어 후 리뷰 요청을 까먹거나, 부정 후기 대응이 늦습니다.

### 자동화 설계 (3:00–8:00)
**AI 다국어 문의 응대**: 한/영/중/일 24시간 자동 응답 → FAQ(일정·가격·포함사항·취소정책) 즉시 응답 → 복잡한 문의만 담당자 연결 → 문의 내용 기반 투어 자동 추천.

**맞춤 투어 추천**: 고객 정보(여행 일정·인원·예산·관심사) 수집 → AI가 최적 투어 조합 추천 → "2박3일 부산 여행이시면, 첫날 해운대 서핑 투어 + 둘째날 감천문화마을 워킹투어 추천" → 원클릭 패키지 예약.

**예약+운영 자동화**: 예약 확정 → 가이드 자동 배정(언어·지역·일정 매칭) → 차량/장비 자동 예약 → 전일 리마인더 자동 발송(고객+가이드) → 기상 악화 시 자동 알림 + 대안 제안.

**후기 자동 관리**: 투어 종료 24시간 후 리뷰 요청 자동 발송 → 별점 4점 이상 시 SNS 공유 유도 → 3점 이하 시 즉시 CS 담당자 알림 → 플랫폼별(구글·트립어드바이저·네이버) 리뷰 현황 자동 모니터링.
