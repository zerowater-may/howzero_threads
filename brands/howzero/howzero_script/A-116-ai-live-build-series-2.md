## 대본

### 훅 (0:00–0:50)
라이브 빌드 시리즈 #2입니다. 지난 #1에서 고객 문의 자동 분류 시스템을 10분 만에 만들었습니다. 오늘의 미션: 매일 아침 9시, 어제 매출 데이터를 자동으로 분석하고, 핵심 인사이트가 포함된 리포트를 슬랙에 자동 발송하는 시스템. 10분 타이머, 시작합니다.

### 시작 전 설명 (0:50–3:00)
구조: 매일 아침 9시 스케줄 트리거 → 구글 시트에서 어제 매출 데이터 가져오기 → GPT-4가 분석(전일 대비, 전주 대비, 이상치 감지, 인사이트) → 슬랙 #daily-report 채널에 보기 좋은 형태로 전송. 도구: 구글 시트 + Make + GPT-4 + 슬랙. 월 운영비 약 3만 원.

### 라이브 빌드 (3:00–13:00)
**[0:00–1:30] 스케줄 트리거**: Make에서 새 시나리오 생성 → Schedule 트리거 → 매일 오전 9시 KST 설정.

**[1:30–3:30] 데이터 수집**: 구글 시트 모듈 → "일별 매출" 시트에서 어제 날짜 행 조회. 총매출, 주문 수, 객단가, 채널별 매출, 환불액 가져오기. + 전일, 전주 동일 요일 데이터도 함께 가져오기(비교용).

**[3:30–6:30] AI 분석**: OpenAI 모듈 → 프롬프트: "다음 매출 데이터를 분석해주세요. 1) 전일 대비 변화 2) 전주 동일 요일 대비 변화 3) 이상치 감지 4) 핵심 인사이트 1가지 5) 추천 액션 1가지. 한국어, 간결체. 이모지 사용 금지." → 테스트 실행 → 분석 결과 확인.

**[6:30–8:30] 슬랙 전송**: 슬랙 모듈 → #daily-report 채널 → 메시지 포맷팅: 날짜, 핵심 수치(총매출, 전일비, 전주비), AI 인사이트, 추천 액션. Block Kit으로 보기 좋게.

**[8:30–10:00] 테스트+마무리**: 수동 실행 → 데이터 수집 → AI 분석 → 슬랙 발송 확인. 10분 완료.

### 결과 리뷰 (13:00–16:00)
실제 슬랙에 도착한 리포트를 보면서 리뷰. 이 시스템의 가치: 대표님이 매일 아침 출근하면 슬랙에 어제 매출 분석이 와 있음. 숫자만이 아니라 인사이트와 액션까지. "어제 매출 15% 하락 — 원인 추정: 메인 배너 변경 후 클릭률 하락 — 추천: 이전 배너로 롤백 테스트."

확장 가능: 주간 리포트, 월간 리포트, 채널별 상세 리포트 추가 가능. 이메일 발송, PDF 자동 생성도 동일 구조.
