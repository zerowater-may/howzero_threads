### 문제 정의 (0:40–3:00)
도시락/케이터링의 병목. 첫째, 수요 예측 — 날씨·요일·시즌·주변 행사에 따라 수요가 급변합니다. 감으로 대응하면 폐기 아니면 품절. 둘째, 대량 주문 관리 — 기업 케이터링은 인원수·메뉴·알레르기·배송 시간이 다 다릅니다. 하나만 틀려도 클레임. 셋째, 배송 효율 — 점심시간 전에 수십 곳에 동시 배송해야 합니다. 루트 최적화가 안 되면 지각 배송. 넷째, 위생 관리 — 식품 안전 기록, 온도 로그, 유통기한 관리가 법적 의무입니다.

### 자동화 설계 (3:00–8:00)
**AI 수요 예측**: 과거 판매 데이터 + 요일 패턴 + 기상 예보 + 주변 행사 캘린더 + 계절 변수 → AI가 일별 메뉴별 수요 예측 → 식자재 발주량 자동 산출 → 전일 저녁 최종 수량 확정.

**대량 주문 자동 관리**: 기업 고객 주문 접수(웹폼/카카오톡) → AI가 주문 내용 자동 구조화(인원·메뉴·알레르기·배송 시간·장소) → 주방에 생산 지시서 자동 생성 → 고객에게 주문 확인+메뉴 상세 자동 발송.

**배송 루트 자동 최적화**: 당일 배송 건 집계 → 배송 위치·시간·수량 기반 최적 루트 자동 산출 → 기사별 배차 자동 → 고객에게 배송 예상 시간 자동 안내 → 지연 시 즉시 알림.

**위생 기록 자동화**: 식자재 입고 시 유통기한 자동 스캔+기록 → 조리 온도 자동 로깅 → 배송 차량 온도 자동 모니터링 → HACCP 기록 자동 생성 → 점검 시 원클릭 리포트.
