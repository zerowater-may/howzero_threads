# A-084: 유튜브 롱폼 — "AI로 사내 위키/매뉴얼 자동 구축"
> **유형**: 유튜브 롱폼 | **길이**: 16분
## 메타데이터
- **제목**: AI로 사내 위키·매뉴얼 자동 구축 — 신입이 질문 안 하는 회사 만들기
- **썸네일 텍스트**: 사내 위키 AI 자동화 | 질문 제로
- **태그**: 사내 위키, 매뉴얼 자동화, 온보딩, 지식 관리, AI, 하우제로
## 대본

### 훅 (0:00–0:40)
"이거 어떻게 해요?" — 사내에서 하루에 몇 번 듣는 질문입니까? 매뉴얼 있다고요? 아무도 안 읽습니다. 왜냐면 200페이지짜리 워드 파일에서 원하는 정보를 찾을 수가 없으니까요. AI로 사내 지식 베이스를 구축하면, 직원이 질문하면 AI가 3초 만에 정확한 답을 줍니다. 선배 시간 안 뺏습니다.

### 문제 정의 (0:40–3:00)
사내 지식 관리의 현실. 첫째, 매뉴얼이 없거나, 있어도 최신이 아님. 둘째, 지식이 특정 직원 머릿속에만 존재 — 그 사람 퇴사하면 끝. 셋째, 검색이 안 됨 — 구글 드라이브에 파일 수백 개가 있는데 원하는 답을 못 찾음. 결과: 신입 온보딩에 평균 3개월, 반복 질문에 선배 시간 하루 1.5시간 소모.

### 자동화 설계 (3:00–8:00)
**1단계 — 기존 문서 수집**: 구글 드라이브, 노션, 슬랙 메시지, 이메일 등에서 업무 관련 문서·대화 자동 수집. **2단계 — AI 정리**: GPT-4가 문서를 주제별로 분류 + 중복 제거 + 최신 버전만 유지 + Q&A 형태로 재구성. **3단계 — AI 검색 챗봇**: 슬랙에 사내 AI 봇 배치. 직원이 자연어로 질문 → RAG(검색 증강 생성)로 사내 문서에서 답변 + 출처 링크. **4단계 — 자동 업데이트**: 새 프로세스 도입 시 슬랙 공지 → AI가 자동으로 위키에 반영.

### 데모/사례 (8:00–12:30)
IT 스타트업 D사. 직원 25명. 슬랙에 AI 봇 "제로봇" 배치. 자동화 전: 신입 온보딩 질문 일 평균 12건, 선배 대응 시간 일 1.5시간. 자동화 후: 제로봇이 질문의 85% 자동 응답. 선배 대응 시간 일 15분. 신입 온보딩 기간 3개월 → 3주. 퇴사자 지식 유실 제로.

도구: 노션 + 슬랙 + Make + GPT-4 + 벡터 DB(Pinecone). 월 운영비 약 12만 원.

### 효과 정리 (12:30–14:30)
