### [1:30~4:00] 문제 정의

"중소기업 인사 업무를 단계별로 나누면:

1. **채용 공고 작성** — 매번 새로 쓰기 귀찮음
2. **이력서 스크리닝** — 100개 중 10개 골라내기

5. **합격/불합격 통보** — 하나하나 이메일
6. **온보딩** — 매번 같은 설명 반복

이 중 사람이 꼭 해야 하는 건?
4번 면접 평가, 그리고 최종 판단.

### [4:00~8:00] 자동화 설계

"오늘 만들 시스템 4개:

**시스템 1: 채용 공고 자동 생성**
- 직무, 요구 조건, 회사 정보 입력
- AI가 플랫폼별 최적화된 공고 작성
- 사람인/잡코리아/원티드 각각 톤 맞춤

**시스템 2: 이력서 자동 스크리닝**
- 이력서 파일을 업로드하면
- AI가 직무 매칭 점수 산출 (0~100)
- 상위 20%만 추천 + 각 이력서 요약

**시스템 3: 면접 일정 자동 조율**
- 후보자에게 가능 시간 자동 문의 이메일
- 캘린더와 연동해서 빈 시간 매칭
- 확정되면 양쪽에 자동 알림

**시스템 4: 온보딩 자동화**
- 합격 통보 → 필요 서류 안내 자동 발송
- 첫 출근 전 체크리스트 자동 공유
- 1주차/2주차/4주차 체크인 이메일 자동 전송"

### [8:00~13:30] 라이브 데모

"[시스템 1 — 채용 공고]
(화면: Claude Code)

'프론트엔드 개발자 채용 공고를 작성해줘.
경력 3~5년, React 필수, TypeScript 우대.
회사: 이커머스 스타트업, 15인, 성수동.
톤: 원티드 스타일로.'

(실행 — 완성된 공고)

2분 걸리던 게 5초입니다.
사람인용으로 바꿔달라고 하면 톤도 맞춰줍니다.

[시스템 2 — 이력서 스크리닝]
(화면: 이력서 PDF 5개 업로드)

'이 이력서들을 다음 기준으로 평가해줘:
1. React 경력 3년 이상
2. TypeScript 사용 경험
3. 이커머스 도메인 경험
각 이력서에 점수(0~100)와 한 줄 요약.'

(실행)

100개 이력서를 수동으로 읽으면 5시간.
AI로 하면 3분.
그리고 '왜 이 점수인지' 이유도 나옵니다.

[시스템 3 — 면접 일정]
'상위 5명에게 면접 일정 조율 이메일을 작성해줘.
가능 시간: 다음 주 화~목 10시~17시.
형식: 정중하지만 간결하게.'

[시스템 4 — 온보딩]
'신규 입사자 온보딩 체크리스트를 만들어줘.
포함 항목: 서류, 장비, 계정, 팀 소개, 1주차 목표'

깔끔한 온보딩 문서가 나옵니다.
이걸 매번 새로 만들 필요가 없습니다."
