# A-220: 유튜브 롱폼 — "AI로 팀 회고(레트로) 자동화"
> **유형**: 유튜브 롱폼 | **길이**: 16분
## 메타데이터
- **제목**: AI로 팀 회고(레트로) 자동화 — 형식적 회고를 진짜 개선으로
- **썸네일 텍스트**: 팀 회고 AI | 형식 → 실행
- **태그**: 팀 회고, 레트로스펙티브, 개선, 팀 문화, AI 자동화, 하우제로
## 대본

### 훅 (0:00–0:40)
스프린트 회고, 하고 계세요? "잘한 점: 열심히 했다. 개선할 점: 소통 더 하자." — 매번 같은 이야기. 이런 회고는 안 하느니만 못합니다. AI가 실제 업무 데이터를 분석해서 진짜 문제를 찾아주고, 개선 액션까지 추적해주는 시스템. 형식적 회고를 진짜 개선으로 바꾼 팀 이야기입니다.

### 문제 정의 (0:40–3:00)
팀 회고의 병목. 첫째, 기억 의존 — 2주 전 일을 기억에 의존해서 이야기합니다. 주관적이고 편향됩니다. 둘째, 반복 패턴 — 같은 문제가 매 회고마다 나옵니다. 액션 아이템이 실행되지 않으니까. 셋째, 시간 비효율 — 1시간 회고 중 30분이 "뭐가 있었지?" 회상에 쓰입니다.

### 자동화 설계 (3:00–8:00)
**데이터 기반 회고 자료 자동 생성**: 스프린트 기간 중 → 완료 태스크, 지연 태스크, PR 리뷰 시간, 배포 횟수, 에러 발생 건수 자동 집계 → AI가 "이번 스프린트 팩트 시트" 자동 생성.

**감정 데이터 자동 수집**: 스프린트 종료 시 → 팀원별 익명 감정 체크(에너지 레벨, 만족도, 블로커) 슬랙 봇으로 수집 → AI가 팀 분위기 요약.

**AI 인사이트 도출**: 팩트 시트 + 감정 데이터 + 이전 회고 액션 아이템 → AI가 "이번 스프린트 핵심 이슈 3가지" 도출 → 회고 시작 전 팀에게 공유(회상 시간 절약).

**액션 아이템 자동 추적**: 회고에서 나온 개선 사항 → 태스크로 자동 전환 → 담당자 배정 → 다음 회고 시 "이전 액션 아이템 실행 여부" 자동 리포트.

### 데모/사례 (8:00–12:30)
개발팀 8명, 2주 스프린트. 자동화 전: 회고 1시간(대부분 회상), 액션 아이템 실행률 20%, "소통 더 하자" 3연속 등장, 팀원 참여도 저조. 자동화 후: 회고 30분(팩트 시트 기반 토론), 액션 아이템 실행률 85%, 구체적 이슈 기반 개선.

3개월 후: 스프린트 완료율 65% → 82%. 배포 후 핫픽스 40% 감소. 팀원 회고 만족도 "유익하다" 90%.

도구: 지라 API + 슬랙 + 구글 시트 + Make + GPT-4. 월 운영비 약 5만 원.
