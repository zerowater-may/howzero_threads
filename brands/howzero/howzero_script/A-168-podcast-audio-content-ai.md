## 대본

### 훅 (0:00–0:40)
팟캐스트 하나 만드는 데 걸리는 시간. 주제 기획 2시간, 대본 3시간, 녹음 1시간, 편집 3시간, 쇼노트 작성 1시간, 배포 30분. 총 10시간 반입니다. 주 1회 발행이면 한 주의 25%가 팟캐스트에 갑니다. AI로 이 10시간을 3시간으로 줄인 시스템. 코딩 제로입니다.

### 문제 정의 (0:40–3:30)
팟캐스트 제작의 3대 병목. 첫째, 주제 기획 — 매주 "이번 주 뭐 하지?" 고민. 트렌드 리서치에 시간 소모. 둘째, 대본·쇼노트 — 대본 없이 하면 횡설수설, 대본 쓰면 시간 폭발. 쇼노트는 SEO 중요한데 귀찮아서 대충. 셋째, 편집·배포 — 음성 편집(어, 음 제거, 볼륨 정규화), 각 플랫폼별 업로드, 홍보 클립 제작. 반복 작업의 연속.

### 자동화 설계 (3:30–10:00)
**주제 기획 자동화**: 업종·관심 키워드 설정 → AI가 매주 트렌드 리서치(구글 트렌드, 네이버 데이터랩, 업계 뉴스) → 에피소드 주제 후보 5개 자동 생성(제목+핵심 포인트 3개+예상 청취자 관심도 점수) → 호스트가 선택.

**대본 자동 생성**: 선택된 주제 → AI가 구조화된 대본 초안 생성(인트로·본론 3파트·아웃트로) → 톤앤매너 맞춤 설정("대화체", "전문가 톤" 등) → 호스트가 검수·수정(30분).

**편집 자동화**: 녹음 파일 업로드 → AI 음성 편집(무음 구간 제거, 어·음 감소, 볼륨 정규화, 배경 노이즈 제거) → 인트로/아웃트로 음악 자동 삽입 → 최종 파일 자동 생성.

**쇼노트·홍보 자동 생성**: 녹음 내용 자동 전사(Whisper) → AI가 쇼노트 자동 생성(요약, 타임스탬프, 핵심 키워드, 관련 링크) → SNS 홍보용 짧은 문구 3개 자동 생성 → 하이라이트 구간 자동 추출(1분 클립).

**멀티 플랫폼 배포**: 최종 파일 + 쇼노트 → 애플 팟캐스트, 스포티파이, 유튜브 뮤직 동시 배포 자동화 → 배포 완료 시 SNS 자동 홍보 포스팅.

### 데모/사례 (10:00–14:30)
비즈니스 팟캐스트 운영자 P. 주 1회 발행. 자동화 전: 에피소드당 10.5시간, 쇼노트 대충 3줄, SNS 홍보 안 함, 월 4회가 한계. 자동화 후: 에피소드당 3시간(기획 선택 10분+대본 검수 30분+녹음 1시간+편집 검수 20분+나머지 자동). 쇼노트 충실(타임스탬프+키워드+요약). SNS 자동 홍보로 신규 청취자 40%↑. 주 2회 발행 가능.

도구: Make + GPT-4 + Whisper API + Descript + 구글 시트. 월 운영비 약 15만 원.
