## 대본

### 훅 (0:00–0:40)
제품 리콜 한 번이면 수억입니다. 근데 리콜 전에 신호가 있었습니다. 고객 리뷰, CS 문의, SNS 멘션에 이미 나와 있었어요. 아무도 안 봤을 뿐입니다. AI가 이 신호를 실시간으로 감지해서, 리콜 전에 잡는 시스템. 실제로 품질 이슈를 2주 먼저 잡아서 리콜을 막은 사례 보여드리겠습니다.

### 문제 정의 (0:40–3:00)
품질 관리의 병목. 첫째, 신호 분산 — 쿠팡 리뷰, 네이버 리뷰, 인스타 멘션, CS 문의, 커뮤니티 글에 흩어져 있습니다. 둘째, 감지 지연 — QC팀이 수동으로 리뷰를 보는 건 월 1회 수준. 문제가 커진 후에야 인식합니다. 셋째, 원인 추적 — "이 제품 별로다" 리뷰에서 구체적으로 뭐가 문제인지 파악하기 어렵습니다.

### 자동화 설계 (3:00–8:30)
**다채널 리뷰 자동 수집**: 쿠팡, 네이버, 자사몰, SNS 멘션 자동 크롤링 → 제품별 리뷰 통합 DB.

**품질 이슈 자동 감지**: 리뷰 텍스트 AI 분석 → "결함", "고장", "불량", "반품" 등 품질 관련 키워드 감지 → 특정 제품에 부정 키워드 급증 시 즉시 알림.

**이슈 원인 자동 분류**: 감지된 부정 리뷰 → AI가 원인 카테고리 분류(포장 파손, 기능 불량, 소재 문제, 사이즈 오차 등) → 카테고리별 빈도 추적.

**조기 경보 시스템**: 특정 제품 부정 리뷰 비율이 기준치(예: 주간 15%) 초과 → QC팀 + 생산팀에 자동 경보 → AI가 상위 5개 이슈 요약 + 대응 권고.

**추이 대시보드**: 제품별 품질 점수 자동 계산 → 주간/월간 추이 그래프 → 개선 후 변화 추적.
