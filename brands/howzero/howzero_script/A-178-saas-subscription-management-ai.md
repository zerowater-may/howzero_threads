## 대본

### 훅 (0:00–0:40)
우리 회사가 쓰는 SaaS가 몇 개인지 아십니까. 평균 중소기업이 20~40개 SaaS를 사용합니다. 슬랙, 노션, 줌, 구글 워크스페이스, 피그마, 잔디, 먼데이... 문제는 안 쓰는 구독이 30%라는 겁니다. 퇴사한 직원 계정, 무료 대안이 있는 유료 툴, 중복 기능 구독. 연간 수백만 원이 그냥 새고 있습니다. AI로 SaaS 구독 전수 조사부터 비용 최적화까지. 코딩 제로입니다.

### 문제 정의 (0:40–3:30)
SaaS 구독 관리의 3대 병목. 첫째, 파악 불가 — 어떤 SaaS를 쓰는지 전체 목록이 없음. 부서별로 각자 결제. 둘째, 비용 낭비 — 퇴사자 계정 미해지, 사용하지 않는 프리미엄 플랜, 중복 기능 SaaS. 셋째, 갱신 관리 — 자동 갱신 걸어놓고 잊어버림. 갱신 시점에 플랜 재검토 안 함.

### 자동화 설계 (3:30–10:00)
**SaaS 전수 조사 자동화**: 법인카드·경비 처리 데이터에서 구독 결제 자동 추출 → SaaS명·요금·결제 주기·담당 부서 자동 정리 → 전사 SaaS 인벤토리 자동 구축.

**사용량 분석**: SaaS별 활성 사용자 수 추적(로그인 빈도, API 사용량) → 라이선스 수 대비 실제 사용자 비율 자동 산출 → 미사용/저사용 SaaS 자동 식별("노션 라이선스 30개 중 실 사용 18개 — 12개 과잉").

**비용 최적화 제안**: AI가 분석 → 플랜 다운그레이드 제안("슬랙 프로→무료로 변경 시 기능 영향 낮음 — 월 15만 원 절감") → 중복 SaaS 통합 제안("먼데이+아사나 중복 — 하나로 통합 시 월 20만 원 절감") → 대안 SaaS 추천(동일 기능, 저비용).

**갱신 관리 자동화**: SaaS별 갱신일 자동 추적 → D-30 자동 알림("구글 워크스페이스 갱신 30일 전 — 현재 플랜 검토 권장") → 갱신 전 AI가 최근 사용량 기반 플랜 재추천 → 갱신/해지/다운그레이드 의사결정 지원.

### 데모/사례 (10:00–14:30)
스타트업 S. 직원 45명. 자동화 전: 사용 SaaS 파악 안 됨(추정 25개), 퇴사자 계정 7개 미해지, 중복 SaaS 3쌍, 연간 SaaS 비용 4,800만 원. 자동화 후: 전수 조사 결과 SaaS 32개 발견(추정보다 7개 많음). 미사용·퇴사자 계정 정리로 즉시 연 360만 원 절감. 중복 통합으로 연 480만 원 절감. 플랜 최적화로 연 240만 원 절감. 총 연 1,080만 원 절감(22.5%↓).
