## 대본

### 훅 (0:00–0:40)
스마트팜 운영하시는 분, "스마트"라고 하지만 실상은 수동 아닌가요? 센서 데이터 보고 직접 온도 조절하고, 양액 농도 직접 맞추고, 병해 발생하면 그때서야 대응하고. AI가 환경 데이터를 실시간 분석해서 자동 제어하고, 병해를 3일 전에 예측하고, 수확 시점과 수확량까지 자동으로 예측합니다. 수확량 35% 증가, 에너지 비용 25% 절감. 진짜 "스마트"팜은 이겁니다.

### 문제 정의 (0:40–3:00)
스마트팜의 현실적 병목. 첫째, 환경 제어 — 온도·습도·CO2·광량·양액 농도를 실시간으로 최적 조합을 찾아야 합니다. 변수가 너무 많아 경험에 의존합니다. 둘째, 병해충 대응 — 눈에 보이는 시점은 이미 늦습니다. 조기 감지가 핵심인데 사람 눈으로는 한계. 셋째, 에너지 관리 — 난방·냉방·조명 비용이 운영비의 40%. 최적화 없이 운영하면 낭비가 큽니다. 넷째, 수확 타이밍 — 너무 일찍 따면 품질 저하, 너무 늦으면 유통기한 단축.

### 자동화 설계 (3:00–8:30)
**AI 환경 자동 제어**: 온도·습도·CO2·광량·양액 EC/pH 센서 데이터 실시간 수집 → 작물 생육 단계별 최적 환경 조건 AI 학습 → 자동 제어(환기팬·난방·양액 공급·LED 조명) → 외부 기상 예보 연동 사전 제어.

**병해충 AI 조기 감지**: 카메라 이미지 AI 분석(잎 색상·형태 변화 감지) → 환경 데이터 패턴 분석(고온다습 지속 → 곰팡이 위험 예측) → 위험 3일 전 알림 → 예방 조치 자동 실행(환기 강화, 양액 조정).

**에너지 최적화**: 시간대별 전기 요금 + 외부 기온 + 작물 필요 환경 종합 분석 → 난방/냉방 가동 타이밍 자동 최적화 → 야간 저온기 축열, 주간 방열 패턴 AI 설계 → 월간 에너지 리포트 자동 생성.

**수확 예측+품질 관리**: 생육 데이터(높이·잎 수·무게 추정) AI 분석 → 수확 적기 자동 예측(2일 전 알림) → 예상 수확량 자동 산출 → 출하처별 필요 수량 대비 과부족 자동 계산 → 출하 스케줄 자동 생성.
