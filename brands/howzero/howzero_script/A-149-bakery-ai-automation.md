## 대본

### 훅 (0:00–0:40)
빵집 사장님, 오늘 폐기한 빵이 몇 개입니까. 평균적으로 베이커리의 일 폐기율은 15~20%입니다. 매일 만든 빵의 5분의 1을 버리는 겁니다. 월로 환산하면 수백만 원. 반대로 인기 메뉴는 오후 2시면 품절. "아까 왔는데 없었어요"가 반복되면 고객이 떠납니다. AI로 생산량 예측, 재고 관리, 단골 케어까지 자동화한 베이커리 사례를 보여드리겠습니다.

### 문제 정의 (0:40–3:00)
베이커리의 3대 병목. 첫째, 생산량 예측 — 요일별, 날씨별, 시즌별 판매량이 다른데 경험으로 대충 맞춤. 과잉 생산 = 폐기, 과소 생산 = 기회 손실. 둘째, 재고 관리 — 밀가루, 버터, 크림 등 원재료 소진 시기 파악 수동. 급발주 시 단가 상승. 셋째, 고객 관리 — 매일 오는 단골이 누구인지, 선호 메뉴가 뭔지, 생일이 언제인지 모릅니다.

### 자동화 설계 (3:00–8:00)
**AI 생산량 예측**: 일별 판매 데이터를 구글 시트에 축적(메뉴별 판매량, 요일, 날씨, 시즌) → AI가 다음 날 메뉴별 예상 판매량 산출 → 생산 계획표 자동 생성 → 요일/날씨 패턴 학습으로 정확도 지속 향상.

**재고 자동 관리**: 레시피별 원재료 소모량 DB → 일 생산량 기반 원재료 소진 자동 계산 → 안전 재고 이하 시 자동 발주 → 발주 이력·단가 비교 자동.

**단골 관리 자동화**: 포인트 카드/앱으로 구매 이력 수집 → 단골 고객 자동 식별(주 3회+ 방문) → 생일 자동 축하 + 무료 케이크 쿠폰 → 신메뉴 출시 시 단골에게 우선 안내 → 7일 미방문 단골 "보고 싶다" 메시지.

**폐기 최소화**: 오후 시간대 잔여 재고 파악 → 타임 세일 자동 안내(SNS + 카카오) → 마감 할인 자동 → 일별 폐기량 기록 + 폐기율 트렌드 분석.

### 데모/사례 (8:00–13:30)
동네 베이커리 BK. 일 생산 메뉴 25종, 일 평균 생산 300개. 자동화 전: 폐기율 18%(일 54개), 인기 메뉴 품절 일 3건, 원재료 급발주 월 4회, 단골 관리 없음. 자동화 후: AI 예측으로 폐기율 18%→6%(일 18개). 인기 메뉴 품절 일 3건→0.5건(생산량 최적화). 원재료 급발주 0건(자동 발주). 단골 관리로 단골 방문 빈도 20%↑. 월 폐기 비용 180만 원→60만 원.

도구: 구글 시트 + Make + GPT-4 + 카카오 알림톡. 월 운영비 약 6만 원.
