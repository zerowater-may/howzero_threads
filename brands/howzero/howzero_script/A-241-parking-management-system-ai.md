## 대본

### 훅 (0:00–0:40)
주차관제 시스템 운영하시는 분, 장애 콜 하루에 몇 건 받으세요? 차단기 안 열림, 번호판 인식 오류, 정산기 먹통. 현장 기사 보내기 전에 전화 받고, 상황 파악하고, 배차하고. 이것만 하루 2시간입니다. AI가 장애 접수부터 원격 진단, 기사 자동 배차까지 처리하면 어떻게 될까요? 장애 대응 시간 70% 단축. 실제 사례입니다.

### 문제 정의 (0:40–3:00)
주차관제 업체의 병목은 명확합니다. 첫째, 장애 접수 — 관리소에서 전화 오면 상황 파악에 시간이 걸립니다. "차단기가 안 올라가요"만으로는 원인을 모릅니다. 둘째, 기사 배차 — 어느 기사가 가까운지, 어떤 부품이 필요한지 수동으로 판단합니다. 셋째, 유지보수 스케줄 — 수백 개 현장의 정기 점검, 부품 교체 주기를 엑셀로 관리합니다. 넷째, 고객(관리소/건물주) 리포트 — 월간 운영 보고서를 수동으로 작성합니다.

### 자동화 설계 (3:00–8:00)
**AI 장애 접수+원격 진단**: 관리소에서 카카오톡/웹폼으로 장애 접수 → AI가 증상 키워드 분석 + 해당 장비 로그 자동 조회 → 원격 재부팅으로 해결 가능한 건 즉시 처리(전체 장애의 40%) → 현장 출동 필요 건만 기사에게 전달.

**자동 기사 배차**: 장애 위치 + 기사 현재 위치 + 보유 부품 매칭 → 최적 기사 자동 선정 → 기사 앱에 장애 상세+예상 원인+필요 부품 자동 전송 → 도착 예상 시간 관리소에 자동 안내.

**예방 정비 자동화**: 장비별 가동 시간, 장애 이력 AI 분석 → 고장 예측 → 정기 점검 스케줄 자동 생성 → 부품 교체 시기 도래 시 자동 발주.

**고객 리포트 자동 생성**: 월간 장애 건수, 대응 시간, 가동률, 정비 이력 → AI가 자동 집계 + 리포트 생성 → 고객사별 맞춤 발송.

### 데모/사례 (8:00–12:30)
주차관제 업체 Q. 관리 현장 150개소, 기사 12명. 자동화 전: 일 평균 장애 콜 35건, 접수→기사 배차까지 평균 25분, 원격 해결 비율 0%(전부 출동), 월간 리포트 작성 주 3일. 자동화 후: 장애 콜 35건 중 14건 AI 원격 자동 해결, 배차까지 평균 5분, 월간 리포트 자동 생성(1시간).

기사 1인당 처리 건수 일 3건 → 5건. 고객 만족도(장애 대응 속도) 3.2점 → 4.5점. 신규 현장 수주 시 인력 추가 없이 확장 가능.
