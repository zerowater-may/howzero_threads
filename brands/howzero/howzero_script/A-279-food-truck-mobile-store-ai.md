## 대본

### 훅 (0:00–0:40)
푸드트럭 하시는 분, 오늘 어디서 장사할지 어떻게 정하세요? 감으로요? "저번에 여기 잘 됐으니까." 그런데 같은 장소도 날씨, 요일, 주변 행사에 따라 매출이 3배 차이납니다. AI가 날씨·요일·행사·유동인구 데이터를 분석해서 오늘의 최적 입지를 추천합니다. 그리고 메뉴별 수요 예측, 재고 자동 관리, SNS 위치 공지까지. 입지만 바꿔도 매출 2배.

### 문제 정의 (0:40–3:00)
푸드트럭의 병목. 첫째, 입지 선정 — 고정 매장이 아니라 매일 결정해야 합니다. 경험에 의존하면 실패 확률이 높습니다. 둘째, 수요 예측 — 오늘 몇 인분 준비할지. 너무 많으면 폐기, 너무 적으면 매출 손실. 셋째, 고객 알림 — 단골이 "오늘 어디세요?" 매번 물어봅니다. 위치를 알려줄 체계적 방법이 없습니다. 넷째, 운영 기록 — 어떤 날, 어떤 장소에서, 어떤 메뉴가 잘 팔렸는지 기록을 안 합니다. 학습이 안 됩니다.

### 자동화 설계 (3:00–8:00)
**AI 입지 추천**: 과거 매출 데이터(장소별·요일별·날씨별) + 당일 기상 예보 + 주변 행사·축제 캘린더 + 유동인구 데이터 → AI가 당일 최적 입지 TOP 3 자동 추천 + 예상 매출 → 전날 저녁에 알림.

**메뉴별 수요 예측**: 입지별 과거 메뉴 판매 데이터 + 날씨(더운 날 = 음료 증가, 추운 날 = 국물 증가) + 요일 패턴 → 메뉴별 준비 수량 자동 산출 → 식자재 발주량 자동 계산 → 전날 저녁 발주.

**고객 위치 자동 공지**: 당일 위치 확정 → SNS(인스타·카카오 채널) 자동 위치 공지 → "오늘 하우제로 트럭은 여의도 공원 앞! 11시~14시" → 구글 지도 핀 자동 첨부 → 단골 고객 개별 알림.

**운영 데이터 자동 기록**: 일일 매출 자동 기록(POS 연동) → 장소·날씨·요일·메뉴별 자동 분류 → 주간/월간 분석 리포트 자동 생성 → "여의도 수요일 맑은 날 = 평균 매출 85만 원" 같은 인사이트 자동 축적.

### 데모/사례 (8:00–12:30)
푸드트럭 AY. 타코+부리토 메뉴, 1인 운영. 자동화 전: 입지 선정 감(자주 가는 3곳 순환), 폐기율 20%, 일 매출 편차 15만~80만 원, SNS 위치 공지 불규칙, 운영 데이터 미기록. 자동화 후: AI 추천 입지(10곳 중 최적 선택), 폐기율 5%, 일 매출 편차 45만~95만 원(하한 3배 상승), SNS 자동 공지(도달률 2.5배), 운영 데이터 자동 축적.

월 매출: 평균 일 45만 원 → 평균 일 72만 원(60% 증가). 폐기 비용: 월 120만 원 → 30만 원. AI 추천 입지 매출이 기존 입지 대비 평균 1.8배.

도구: 구글 시트 + Make + GPT-4 + 기상청 API + 카카오 알림톡. 월 운영비 약 4만 원.
