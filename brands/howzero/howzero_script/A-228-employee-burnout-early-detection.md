## 대본

### 훅 (0:00–0:40)
핵심 직원이 갑자기 퇴사합니다. "왜 말 안 했어?" 사실 신호는 있었습니다. 야근 빈도 변화, 슬랙 응답 속도 변화, 휴가 패턴 변화, 미팅 참여도 변화. 사람이 보기엔 모르지만 AI는 봅니다. 번아웃 신호를 2~4주 전에 감지해서 알려주는 시스템. 핵심 인재 퇴사율 60% 줄인 사례입니다.

### 문제 정의 (0:40–3:00)
직원 번아웃의 병목. 첫째, 감지 불능 — 본인도 모르는 경우가 많습니다. 관리자는 더 모릅니다. 둘째, 사후 대응 — 퇴사 의사 밝힌 후에야 대응. 이미 늦었습니다. 채용 비용 연봉의 50~200%. 셋째, 프라이버시 — 직원 모니터링은 민감합니다. 감시가 아닌 케어로 설계해야 합니다.

### 자동화 설계 (3:00–8:30)
**업무 패턴 자동 분석**: 출퇴근 시간 변화, 야근 빈도, 주말 근무 패턴 → 개인별 baseline 대비 이상 변화 자동 감지. (중요: 개인 데이터는 익명화, 관리자에게는 팀 단위 트렌드만 공유.)

**커뮤니케이션 패턴 분석**: 슬랙 메시지 빈도·응답 시간 변화(내용은 분석 안 함, 메타데이터만) → 참여도 변화 자동 감지.

**정기 웰빙 체크 자동화**: 격주 1회 → 슬랙 봇으로 3문항 익명 설문(에너지, 업무 만족도, 지원 필요 여부) → AI가 팀 단위 트렌드 분석 → 특정 팀 점수 급락 시 HR 알림.

**조기 개입 추천**: 번아웃 위험 신호 감지 → AI가 관리자에게 "1:1 면담 권장" 알림 → 대화 가이드(질문 리스트) 자동 제공 → 면담 후 조치 기록 → 추적.

**퇴사 예측 모델**: 번아웃 지표 + 근속 기간 + 업무 변화 → AI가 퇴사 위험도 점수 산출(팀 단위) → HR에게 분기별 리스크 리포트.
