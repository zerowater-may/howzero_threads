## 대본

### 훅 (0:00–0:40)
SaaS 서비스 운영하시는 분, 가입 후 30일 이내 이탈률 몇 퍼센트세요? 업계 평균 40~60%입니다. 절반이 한 달도 안 돼서 떠납니다. 이유요? 서비스를 제대로 써보지도 못했으니까. 온보딩이 엉망이면 아무리 좋은 제품도 소용없습니다. AI가 고객별 사용 패턴을 분석해서 맞춤 온보딩을 자동 진행합니다. 30일 이탈률 60% 감소. 온보딩은 매출입니다.

### 문제 정의 (0:40–3:00)
고객 온보딩의 병목. 첫째, 일률적 온보딩 — 모든 고객에게 같은 가이드, 같은 이메일 시퀀스. 마케팅 목적으로 가입한 고객과 데이터 분석 목적 고객에게 같은 안내. 둘째, 활성화 추적 불가 — 고객이 핵심 기능을 실제로 사용했는지 모릅니다. 셋째, 지원 타이밍 — 고객이 막혀서 이탈하는 순간을 감지 못 합니다. 이탈 후에 "어떻게 도와드릴까요?" 이메일. 넷째, CS 병목 — 온보딩 질문이 CS 티켓의 40%를 차지합니다.

### 자동화 설계 (3:00–8:30)
**맞춤 온보딩 경로**: 가입 시 목적·역할·팀 규모 수집(3문항) → AI가 고객 세그먼트 자동 분류 → 세그먼트별 최적 온보딩 시퀀스 자동 실행 → 마케팅 담당자에게는 캠페인 설정 가이드부터, 개발자에게는 API 연동 가이드부터.

**활성화 마일스톤 추적**: 핵심 기능 5개 정의 → 고객별 사용 여부 실시간 추적 → 미사용 기능 안내 자동 발송("아직 리포트 기능을 안 써보셨네요. 3분이면 첫 리포트를 만들 수 있어요") → 전체 마일스톤 완료 시 축하 메시지 + 프리미엄 기능 안내.

**이탈 위험 감지+자동 개입**: 접속 빈도 감소, 핵심 기능 미사용, 에러 반복 발생 패턴 AI 감지 → 이탈 위험도 자동 스코어링 → 저위험: 자동 이메일 → 중위험: 인앱 가이드 팝업 → 고위험: CSM 1:1 연락 자동 트리거.

**온보딩 AI 어시스턴트**: 서비스 내 AI 챗봇이 24시간 온보딩 질문 응대 → "데이터 어떻게 가져오나요?" → 단계별 스크린샷 포함 가이드 자동 안내 → 해결 불가 시 인간 CS 즉시 연결.
