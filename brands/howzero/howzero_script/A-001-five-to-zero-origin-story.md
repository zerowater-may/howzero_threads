## [4:00~7:00] 첫 시도 — 실패 3번 (3분)

**[화면: 화면 녹화 + 실패 스크린샷]**

**대본:**
"처음부터 성공한 건 아닙니다. 3번 망했습니다.

**실패 1: 외주 개발**
처음에는 외주 개발자한테 맡겼습니다. 500만 원 썼는데, 3개월 후에 나온 결과물이 제가 원하던 거랑 완전 달랐어요. 소통 비용이 개발 비용보다 컸습니다.

교훈: '내가 직접 만들어야 한다. 외주는 답이 아니다.'

**실패 2: 노코드 도구**
그 다음에 노코드 도구를 써봤습니다. Bubble, Glide 이런 거요. 근데 이것도 AI 연동이 어려웠고, 제가 원하는 수준의 자동화는 안 됐습니다.

교훈: '도구가 문제가 아니라, 내 프로세스를 먼저 정리해야 한다.'

**실패 3: 과욕**
세 번째는 너무 많은 걸 한 번에 하려고 했습니다. CS도 자동화하고, 마케팅도 자동화하고, 결제도 자동화하고... 결국 아무것도 완성 못 했습니다.

교훈: '가장 쉽고 가장 효과 큰 것 하나부터. Low-hanging fruit.'"

## [7:00~11:00] 전환점 — GPT-3 + 프로세스 맵핑 (4분)

**[화면: 프로세스 맵 시각화 + GPT-3 화면 녹화]**

**대본:**
"전환점은 두 가지가 동시에 일어났을 때 왔습니다.

첫째, 프로세스 맵을 그렸습니다. 종이 한 장에 우리 회사의 모든 반복 업무를 쭉 적었어요.

(화면에 프로세스 맵 보여주기)

CS 문의 접수 → 분류 → 담당자 배정 → 응답 작성 → 발송 → 후속 관리

이 6단계 중에 '사람이 꼭 해야 하는 것'만 남기고 나머지를 체크했더니... 6개 중 4개가 자동화 가능했습니다.

둘째, GPT-3 API를 발견했습니다. 이건 ChatGPT 같은 대화형이 아니라, 프로그램 안에 AI를 집어넣을 수 있는 도구입니다. 근데 중요한 건 — 코딩 없이도 연결할 수 있었다는 거예요.

어떻게요? Claude Code 같은 자동화 도구로 '이메일이 오면 → GPT가 분류하고 → 답변 초안 만들어서 → 담당자한테 보내줘'라는 흐름을 드래그앤드롭으로 만들 수 있었거든요.

이걸 깨달은 순간, 모든 게 바뀌었습니다."

## [11:00~14:00] 구축 과정 — 5가지를 0으로 (3분)

**[화면: Claude Code 워크플로우 실제 화면 녹화]**

**대본:**
"그래서 저의 5가지 반복 업무를 하나씩 자동화했습니다.

(화면: 카운트다운 애니메이션 5→4→3→2→1→0)

**1번: CS 문의 분류** — 하루 2시간 → 0시간
이메일이 오면 AI가 자동으로 '환불', '배송', '제품문의', '기타'로 분류합니다. 정확도 92%.

**2번: 고객 응답 초안** — 하루 1시간 → 0시간
분류된 문의에 맞는 응답 초안을 AI가 만들어줍니다. 담당자는 검토만 하면 돼요.

**3번: 리뷰 모니터링** — 하루 30분 → 0분
리뷰가 올라오면 자동 수집하고, 감성 분석해서 부정 리뷰만 알림 보내줍니다.

**4번: 주간 보고서** — 주 2시간 → 0시간
데이터를 자동으로 모아서 AI가 요약 보고서를 만들고, 이메일로 발송합니다.

**5번: 데이터 정리** — 하루 1시간 → 0시간
엑셀 수동 입력이 전부 자동화됐습니다.

5가지, 전부 0으로 만들었습니다.
절감 시간: 주 40시간 이상.
직원 5명이 할 일을 AI 하나가 합니다."
