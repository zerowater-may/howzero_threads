### 훅 (0:00–0:40)

### 문제 정의 (0:40–3:00)
고객 인터뷰 분석의 3대 병목. 첫째, 녹취록 정리 — 녹음 파일을 텍스트로 변환하고 정리하는 데만 인터뷰당 2시간. 둘째, 코딩(정성 분석) — 발언을 카테고리별로 분류하고 빈도 체크하는 작업. 10건이면 2~3일. 셋째, 인사이트 도출 — 개별 발언에서 공통 패턴을 찾고, 실행 가능한 개선 방향을 뽑아내는 것. 분석자의 역량에 따라 결과가 완전히 달라짐.

### 자동화 설계 (3:00–8:30)
**녹취록 자동 변환**: 녹음 파일 → Whisper API로 자동 텍스트 변환 → 화자 분리 → 구글 독스에 자동 저장. 정확도 95%+. 1시간 인터뷰 5분에 변환.

**AI 자동 코딩**: 녹취록을 GPT-4에 투입 → 사전 정의한 분석 프레임워크(JTBD, 페인 포인트, 기능 요청, 감정 분석)에 따라 자동 분류 → 구글 시트에 구조화 저장 — 발언 원문, 카테고리, 감정(긍정/부정/중립), 키워드.

**패턴 분석 자동화**: 10건의 코딩 결과를 종합 → "가장 많이 언급된 페인 포인트 Top 5", "기능 요청 빈도 순위", "감정 변화 패턴" 자동 산출 → 시각화 차트 자동 생성.

**인사이트 리포트 자동 생성**: 분석 결과를 바탕으로 GPT-4가 "핵심 발견 3가지", "즉시 실행 가능한 개선안", "추가 검증이 필요한 가설"을 리포트로 정리 → 구글 독스 + 슬랙 자동 공유.

SaaS 스타트업 IV. 분기 1회 고객 인터뷰 15건 실시. 자동화 전: 녹취록 정리 30시간, 코딩 분석 24시간, 리포트 작성 8시간 = 총 62시간. 자동화 후: 녹취록 변환 75분(자동), 코딩 30분(자동+검수), 리포트 생성 15분 = 총 2시간. 시간 절감 96%. 분석 프레임워크가 일관되어 분기별 비교 가능.
