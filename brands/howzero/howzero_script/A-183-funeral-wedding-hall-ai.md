## 대본

### 훅 (0:00–0:40)
장례식장과 웨딩홀. 전혀 다른 사업 같지만 운영 구조는 거의 동일합니다. 홀 예약 관리, 케이터링 발주, 인력 배치, 견적서 작성. 그리고 공통점 하나 더 — 대부분 수작업입니다. 예약 겹침 사고 한 번이면 신뢰 추락. AI로 홀 배정부터 자동 견적, 고객 응대까지 자동화한 시스템. 코딩 제로입니다.

### 문제 정의 (0:40–3:30)
장례식장·웨딩홀의 3대 병목. 첫째, 예약·홀 배정 — 홀 3~10개, 날짜·시간·규모 조합 복잡. 엑셀이나 수기 관리. 겹침·누락 리스크. 둘째, 견적 — 패키지, 옵션, 인원수에 따라 가격 다름. 견적서 작성에 건당 30분~1시간. 둘째, 고객 응대 — 문의 전화 하루 20통+. 같은 질문("5월 셋째 주 토요일 가능한가요?") 반복.

### 자동화 설계 (3:30–9:30)
**예약·홀 배정 자동화**: 홀별 수용 인원·시설 정보 DB → 문의 접수 시 가능 일자·홀 실시간 조회 → 규모(예상 하객수)에 맞는 홀 자동 추천 → 예약 확정 시 캘린더 자동 등록 → 겹침 방지 자동 체크 → D-7, D-1 양측(고객+운영팀) 자동 리마인드.

**자동 견적서**: 패키지 선택 + 옵션(꽃장식, 사진, 영상, 음식) + 인원수 입력 → 견적서 자동 생성(PDF) → 고객에게 자동 발송 → 옵션 변경 시 자동 재산출 → 계약 확정 시 계약서 자동 생성.

**고객 응대 자동화**: 카카오 챗봇으로 24시간 기본 문의 자동 응대(가능 일자 조회, 패키지 안내, 가격 안내, 위치·주차 안내) → 상담 필요 건만 담당자 연결 → 문의→예약 전환률 자동 추적.

**운영 자동화**: 확정 예약 기반 → 케이터링 발주 자동 생성 → 필요 인력 자동 산출(홀 규모·인원수 기반) → 인력 배치 자동 스케줄링 → 이벤트 종료 후 고객 만족도 자동 설문.
