## 대본

### 훅 (0:00–0:40)
회사에서 문서 하나 찾는 데 평균 몇 분 걸리세요? 맥킨지 보고서에 따르면 직장인이 문서를 찾는 데 업무 시간의 19%를 쓴다고 합니다. 주 5일 중 1일을 문서 찾기에 쓰는 겁니다. 구글 드라이브, 슬랙, 이메일, 노션에 흩어진 수만 개 문서. AI가 자동으로 분류하고, 자연어로 "지난달 삼성 프로젝트 제안서" 하면 5초 안에 찾아줍니다.

### 문제 정의 (0:40–3:00)
사내 문서 관리의 현실. 첫째, 문서 산재 — 구글 드라이브, 노션, 슬랙, 이메일, 로컬 PC에 분산되어 있습니다. 어디에 있는지 모릅니다. 둘째, 분류 체계 없음 — 폴더 구조가 사람마다 다르고, 파일명 규칙이 없습니다. "최종", "최종_수정", "진짜최종". 셋째, 검색 불가 — 키워드 검색으로는 내용 기반 검색이 안 됩니다. 넷째, 지식 유실 — 퇴사자의 문서가 어디 있는지 모릅니다. 프로젝트 히스토리가 사라집니다.

### 자동화 설계 (3:00–8:00)
**문서 자동 수집+통합**: 구글 드라이브 + 노션 + 슬랙 + 이메일 첨부파일 자동 스캔 → 신규/수정 문서 자동 감지 → 중앙 인덱스에 자동 등록.

**AI 자동 분류**: 문서 내용 AI 분석 → 카테고리 자동 태깅(프로젝트명·부서·문서유형·날짜·고객사) → 중복 문서 자동 감지 → 버전 관리 자동(최신 버전 표시).

**자연어 검색**: "지난달 마케팅팀이 만든 경쟁사 분석 자료" → AI가 의미 기반 검색 → 관련도 순 결과 제공 → 문서 요약 미리보기 → 원문 바로 접근.

**지식 자동 연결**: 관련 문서 자동 추천("이 제안서와 관련된 이전 프로젝트 결과보고서 3건") → 프로젝트별 문서 타임라인 자동 생성 → 신규 직원용 온보딩 문서 패키지 자동 큐레이션.

### 데모/사례 (8:00–12:30)
마케팅 대행사 AD. 직원 35명, 총 문서 약 45,000개. 자동화 전: 문서 검색 평균 8분, 못 찾는 경우 20%, 중복 문서 비율 추정 30%, 신규 직원 온보딩 문서 찾기 3일. 자동화 후: 문서 검색 평균 15초, 못 찾는 경우 2%, 중복 문서 자동 정리(8,000개 제거), 신규 직원 온보딩 문서 자동 제공(즉시).

직원 1인당 절감 시간: 일 45분. 35명 × 월 20일 × 45분 = 월 525시간 절감. 시급 3만 원 환산 약 1,575만 원/월 가치.

도구: 구글 시트 + Make + GPT-4 + 임베딩 API + 벡터 DB(Pinecone). 월 운영비 약 15만 원.
