## 대본

### 훅 (0:00–0:40)
편의점 점주분들, 유통기한 지나서 폐기하는 상품 월 얼마나 됩니까. 평균 매출의 3~5%입니다. 3호점 이상 운영하면 연 수백만 원이 그냥 쓰레기통으로 갑니다. POS 데이터를 AI가 분석해서 발주량을 자동 최적화하고, 시프트까지 자동 편성하는 시스템. 코딩 제로로 만든 사례를 보여드리겠습니다.

### 문제 정의 (0:40–3:30)
편의점·소매점의 3대 병목. 첫째, 발주 — 경험과 감으로 발주. 날씨, 요일, 주변 행사 반영 안 됨. 과잉 발주→폐기, 과소 발주→기회 손실. 둘째, 시프트 관리 — 알바생 스케줄 조율에 점주 주 3시간+. 대타 요청 시 카톡 돌리기. 셋째, POS 데이터 방치 — 매출 데이터가 쌓이지만 분석 안 함. 어떤 상품이 시간대별로 잘 팔리는지 모름.

### 자동화 설계 (3:30–9:30)
**POS 데이터 자동 분석**: POS 매출 데이터 일별 자동 수집 → 시간대별·카테고리별·상품별 매출 분석 → 요일·날씨·주변 행사 변수 반영 → 주간 인사이트 리포트 자동 생성("화요일 18시 삼각김밥 매출 40%↑, 비 오는 날 우산 매출 300%↑").

**AI 발주 최적화**: 지난 90일 판매 데이터 + 날씨 예보 + 요일 패턴 → AI가 상품별 적정 발주량 자동 산출 → 발주서 자동 생성 → 점주가 검수 후 원클릭 발주. 폐기 예상 상품 사전 경고.

**시프트 자동 편성**: 알바생 가능 시간 수집(구글 폼) → 시간대별 필요 인원(POS 매출 기반 산출) → AI가 최적 시프트표 자동 생성 → 대타 필요 시 가능한 알바생에게 자동 문자 → 확정 시 시프트표 자동 업데이트.

**유통기한 관리**: 입고 시 유통기한 등록 → D-3 자동 경고 → 할인 판매 자동 등록 → 폐기 시 자동 기록 → 월간 폐기율 리포트.

### 데모/사례 (9:30–13:30)
편의점 3개 점포 운영 K 점주. 자동화 전: 발주 감으로, 폐기 월 45만 원, 시프트 조율 주 4시간, POS 분석 안 함. 자동화 후: 폐기 월 45만→12만 원(73%↓). 발주 정확도 78%→94%. 시프트 편성 자동화로 주 4시간→20분. 시간대별 매출 분석으로 진열 최적화 → 매출 8%↑.

도구: POS 데이터 연동 + 구글 시트 + Make + GPT-4 + 날씨 API. 월 운영비 약 7만 원.
