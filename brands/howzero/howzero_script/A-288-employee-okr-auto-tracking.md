## 대본

### 훅 (0:00–0:40)
OKR 도입했는데 형식만 남았다고요? 분기 시작할 때 목표 세우고, 분기 끝날 때 "아 못 했네" 하고 끝. OKR이 실패하는 이유는 추적이 안 되기 때문입니다. 매주 진행 상황을 체크하고, 지연되면 코칭하고, 팀 간 정렬 상태를 봐야 합니다. 사람이 하기엔 너무 많습니다. AI가 매주 자동으로 진척률을 추적하고, 지연 신호를 감지하면 코칭 넛지를 보냅니다. 달성률 2배. 코딩 제로.

### 문제 정의 (0:40–3:00)
OKR 운영의 병목. 첫째, 추적 부재 — 분기 초 설정 후 방치. 중간 점검이 없습니다. 둘째, 정렬 실패 — 팀 OKR과 개인 OKR이 따로 놉니다. 회사 목표와 연결이 안 보입니다. 셋째, 코칭 부재 — 지연되는 사람에게 피드백 타이밍을 놓칩니다. 넷째, 데이터 부재 — 달성률, 지연 원인, 반복 패턴이 축적되지 않습니다. 매 분기 같은 실수를 반복합니다.

### 자동화 설계 (3:00–8:30)
**OKR 자동 추적**: 주간 단위 자동 체크인 → 슬랙/이메일로 "이번 주 Key Result 진척률 업데이트해 주세요" 자동 발송 → 응답 자동 수집 → 진척률 대시보드 자동 갱신.

**지연 감지+자동 코칭**: 진척률이 계획 대비 20% 이상 뒤처지면 자동 플래그 → 해당 직원에게 "목표 X가 지연 중입니다. 방해 요인이 뭔가요?" 코칭 넛지 자동 발송 → 방해 요인 패턴 자동 분석 → 매니저에게 리포트.

**팀-회사 정렬 자동 시각화**: 회사 OKR → 팀 OKR → 개인 OKR 계층 구조 자동 매핑 → "회사 목표 A의 달성률이 낮은 이유: 팀 B의 KR-3이 30%에서 정체" 자동 진단.

**분기 회고 자동 리포트**: 분기 종료 시 달성률·지연 원인·코칭 반응률·개선 트렌드 자동 분석 → 팀별/개인별 회고 리포트 자동 생성 → 다음 분기 OKR 설정 시 AI 추천.
