## 대본

### 훅 (0:00–0:40)
AI 자동화 열심히 하시는 분들, 질문 하나. 자동화 시스템이 처리하는 고객 데이터, 어디에 저장되고, 누가 접근할 수 있고, 얼마나 보관되는지 정확히 알고 계세요? 대부분 모릅니다. 자동화를 많이 할수록 데이터는 더 많이 흐르고, 관리 안 하면 개인정보 사고, 과징금, 신뢰 상실. 오늘은 AI 자동화와 데이터 거버넌스를 함께 가져가는 실전 방법을 알려드립니다.

### 문제 정의 (0:40–3:30)
자동화가 많아지면 생기는 데이터 문제. 첫째, 데이터 산재 — 구글 시트, Make, GPT API, 카카오톡 등 여러 도구에 고객 데이터가 분산됩니다. 전체 데이터 흐름을 파악하는 사람이 없습니다. 둘째, 개인정보 리스크 — 자동화가 고객 이름, 연락처, 구매 이력을 자동으로 처리합니다. GDPR, 개인정보보호법 위반 시 매출의 3%까지 과징금. 셋째, 데이터 품질 저하 — 자동화가 쌓는 데이터에 중복, 오류, 포맷 불일치가 축적됩니다. 넷째, 접근 권한 관리 부재 — 누가 어떤 데이터에 접근 가능한지 통제가 없습니다.

### 실전 프레임워크 (3:30–9:00)
**1단계: 데이터 흐름 지도 만들기**: 모든 자동화 시나리오에서 어떤 데이터가 어디서 어디로 이동하는지 시각화. 입력 → 처리 → 저장 → 출력 각 단계에서 개인정보 포함 여부 체크. AI가 자동화 시나리오 분석 → 데이터 흐름도 자동 생성.

**2단계: 분류+등급화**: 데이터를 민감도별로 자동 분류. 1등급(개인 식별 정보) → 암호화 필수, 접근 로그 필수. 2등급(구매 이력 등) → 접근 제한. 3등급(집계 데이터) → 자유 활용.

**3단계: 자동 정책 적용**: 보관 기간 초과 데이터 자동 삭제/익명화 → 접근 권한 분기별 자동 점검 → 데이터 반출 시 자동 알림 → 개인정보 처리 동의 만료 전 자동 재동의 요청.

**4단계: 모니터링+감사**: 데이터 접근 로그 자동 수집 → 이상 접근 패턴 AI 감지 → 월간 거버넌스 보고서 자동 생성 → 법규 변경 시 영향 받는 자동화 시나리오 자동 식별.

### 실전 사례 (9:00–14:00)
이커머스 기업 V. 자동화 시나리오 23개 운영 중. 거버넌스 적용 전: 고객 데이터가 구글 시트 18개에 분산, 개인정보 보관 기간 미설정, 접근 권한 관리 없음, 개인정보 처리방침과 실제 처리가 불일치. 잠재 과징금 리스크 약 2,000만 원.

거버넌스 적용 후: 데이터 흐름도 자동 생성 → 구글 시트 18개를 5개로 통합 → 개인정보 6개월 초과 데이터 자동 익명화 → 접근 권한 월 자동 점검 → 과징금 리스크 0.

추가 효과: 데이터 품질 개선으로 AI 자동화 정확도 15% 향상. 중복 데이터 제거로 API 비용 20% 절감.

도구: 구글 시트 + Make + GPT-4. 추가 비용 월 3만 원(기존 자동화 인프라 활용).

### 체크리스트 (14:00–16:00)
AI 자동화 거버넌스 5대 체크리스트:
1. 모든 자동화의 데이터 흐름도가 있는가?
2. 개인정보 포함 데이터가 분류되어 있는가?
3. 보관 기간과 삭제 정책이 자동 적용되는가?
4. 접근 권한이 분기별로 점검되는가?
5. 데이터 접근 로그가 수집·분석되는가?

이 5개 중 3개 이상 "아니오"면, 자동화를 더 만들기 전에 거버넌스부터 잡으세요.
