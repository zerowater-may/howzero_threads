### 훅 (0:00–0:40)

### 문제 정의 (0:40–3:00)
입찰 관리의 병목. 첫째, 공고 모니터링 — 나라장터, 국방조달, 지방자치단체 등 사이트가 분산되어 있습니다. 매일 확인해도 놓칩니다. 둘째, 적격 공고 필터링 — 자격 요건(실적·면허·재무조건)을 일일이 대조합니다. 맞지 않는 공고에 시간 낭비. 셋째, 가격 분석 — 예정가격, 과거 낙찰률, 경쟁사 분석을 사람이 합니다. 넷째, 서류 준비 — 매 건마다 비슷하지만 미묘하게 다른 제안서, 서류를 처음부터 만듭니다.

### 자동화 설계 (3:00–8:30)
**24시간 공고 모니터링**: 나라장터 + 국방조달 + 지자체 입찰 사이트 자동 크롤링 → 신규 공고 발생 시 즉시 감지 → 회사 프로필(업종·면허·실적) 기반 적격 공고 자동 필터링 → 담당자에게 즉시 알림(카카오톡+이메일).

**자격 요건 자동 대조**: 공고의 참가 자격(실적 요건·면허·재무 조건) 자동 추출 → 회사 보유 자격과 자동 대조 → 충족/미충족 항목 명확 표시 → 미충족 시 공동수급 파트너 자동 추천.

**낙찰 확률 분석**: 해당 발주처 과거 3년간 유사 공고 낙찰 데이터 AI 분석 → 예정가격 대비 낙찰률 분포 → 경쟁 강도 예측(과거 참여 업체 수) → 최적 투찰 가격 범위 자동 산출.

**제안서 초안 자동 생성**: 공고 요구사항 자동 분석 → 과거 수주 제안서 DB에서 유사 건 자동 매칭 → 제안서 골격 자동 생성 → 핵심 차별화 포인트 자동 하이라이트.
