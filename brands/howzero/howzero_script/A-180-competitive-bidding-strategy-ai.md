# A-180: 유튜브 롱폼 — "AI로 경쟁 입찰 전략 자동 분석"
> **유형**: 유튜브 롱폼 | **길이**: 18분
## 메타데이터
- **제목**: AI로 경쟁 입찰 전략 자동 분석 — 낙찰 확률을 높이는 데이터 시스템
- **썸네일 텍스트**: 입찰 전략 AI | 낙찰률↑ 데이터 분석
- **태그**: 입찰, 낙찰, 경쟁 입찰, 조달, 제안서, 전략, AI, 자동화, 하우제로
## 대본

### 훅 (0:00–0:40)
공공 입찰이든 민간 입찰이든, 제안서 쓰는 데 2~3주, 투입 인력 3~5명. 그런데 낙찰률이 20% 안 됩니다. 5번 중 4번은 헛수고입니다. 문제는 감으로 입찰에 참여한다는 겁니다. 과거 낙찰 데이터를 분석하면 패턴이 있습니다. AI로 입찰 기회 필터링부터 경쟁사 분석, 최적 가격 산출까지 자동화한 시스템. 코딩 제로입니다.

### 문제 정의 (0:40–3:30)
입찰 관리의 3대 병목. 첫째, 기회 발굴 — 나라장터, 조달청 등에서 매일 수백 건 공고. 우리에게 맞는 건 찾기 어려움. 둘째, 입찰 의사결정 — 이길 수 있는 입찰인지 판단 기준 없음. 모든 건에 다 뛰어듦 → 제안 품질 하락. 셋째, 가격 전략 — "이 정도면 되겠지"로 가격 결정. 과거 낙찰 가격 분석 안 함.

### 자동화 설계 (3:30–10:00)
**입찰 기회 자동 필터링**: 나라장터·조달청·민간 입찰 사이트 공고 자동 수집 → 우리 회사 역량(업종, 실적, 자격) 매칭 스코어 자동 산출 → 매칭 80% 이상 건만 자동 알림 → 주간 입찰 기회 리포트.

**경쟁사 분석**: 과거 입찰 결과 데이터 수집(낙찰자, 낙찰 금액, 투찰 업체 수) → 동일 분야 경쟁사 패턴 분석(주로 참여하는 규모, 평균 투찰률) → 특정 건에 예상 경쟁사·경쟁 강도 자동 산출.

**최적 가격 자동 산출**: 과거 유사 건 낙찰가율(예정가 대비 낙찰가 비율) 분석 → 경쟁 강도 반영 → AI가 최적 투찰 가격 범위 제안("예정가의 87.5~89.2% 권장 — 유사 건 낙찰가율 평균 88.3%") → 시나리오별 낙찰 확률 시뮬레이션.

**제안서 초안 자동 생성**: 입찰 공고 요구사항 자동 분석 → 과거 제안서 DB에서 유사 섹션 자동 매칭 → 제안서 초안 자동 생성(기술 부문, 관리 부문) → 담당자 검수·보완.

### 데모/사례 (10:00–14:30)
