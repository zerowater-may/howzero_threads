## 대본

### 훅 (0:00–0:40)

### 문제 정의 (0:40–3:30)
반찬·밀키트의 3대 병목. 첫째, 생산 계획 — 뭘 얼마나 만들지 감으로 결정. 과잉 생산→폐기(매출의 8~15%), 과소 생산→기회 손실. 둘째, 재료 발주 — 메뉴별 재료 소요량 수기 계산. 발주 누락, 과잉 발주 빈발. 셋째, 배송·고객 관리 — 정기 배송 고객 관리 수작업. 메뉴 변경·일시 중지·주소 변경 대응 번거로움.

### 자동화 설계 (3:30–9:30)
**수요 예측+생산 계획**: 과거 판매 데이터 + 요일 패턴 + 날씨(더운 날 냉반찬↑) + 명절·연휴 영향 → AI가 메뉴별 일별 예상 판매량 산출 → 생산 계획 자동 생성("내일: 잡채 50인분, 제육 40인분, 샐러드 60인분") → 사장이 검수 후 확정.

**재료 자동 발주**: 확정된 생산 계획 → 메뉴별 레시피 DB에서 필요 재료·수량 자동 산출 → 현재 재고 차감 → 부족분 발주서 자동 생성 → 거래처별 자동 전송 → 입고 확인 자동 기록.

**정기 배송 자동 관리**: 고객별 구독 정보(메뉴, 배송 요일, 주소) 자동 관리 → 매주 배송 리스트 자동 생성 → 배송 동선 자동 최적화 → 배송 출발/완료 고객 자동 알림 → 메뉴 변경·중지 요청 카카오 챗봇으로 자동 처리.

**폐기·원가 분석**: 일별 생산량 대비 판매량 자동 비교 → 폐기율 자동 추적 → 메뉴별 원가율 자동 산출 → 저마진 메뉴 경고 → 월간 원가 리포트.

### 데모/사례 (9:30–13:30)
