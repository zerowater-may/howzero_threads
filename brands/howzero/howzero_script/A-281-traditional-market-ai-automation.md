## 대본

### 훅 (0:00–0:40)
전통시장 상인분들, 단골 고객이 점점 줄고 있다고요? 대형마트, 쿠팡에 밀린다고요? 근데 전통시장의 진짜 강점은 '관계'입니다. 문제는 그 관계를 데이터로 관리 안 한다는 겁니다. 단골이 뭘 좋아하는지, 언제 오는지, 뭘 사는지 — 다 사장님 머릿속에만 있습니다. AI가 이걸 시스템으로 만들면? 단골 재방문율 2배, 배달 매출 추가, 폐기 손실 절반. 코딩 제로.

### 문제 정의 (0:40–3:00)
전통시장의 핵심 병목. 첫째, 단골 관리 — 사장님 기억에만 의존합니다. 사장님이 아프면 단골 데이터가 사라집니다. 둘째, 재고 관리 — 날씨·요일별 수요 변화를 감으로 맞춥니다. 비 오는 날 과일을 잔뜩 깔면 폐기입니다. 셋째, 온라인 채널 부재 — 젊은 고객은 네이버 지도에서 검색합니다. 정보가 없으면 존재하지 않는 겁니다. 넷째, 배달 주문 — 주변 아파트 단지에서 수요가 있는데 체계가 없습니다.

### 자동화 설계 (3:00–8:30)
**단골 고객 자동 관리**: 카카오톡 채널 + 간단 POS 연동 → 고객별 구매 이력 자동 기록 → 주기적 방문 고객 자동 식별 → "김 사모님, 이번 주 제철 딸기 입고됐어요" 개인화 알림 → 재방문율 증가.

**AI 수요 예측+재고 관리**: 요일·날씨·시즌·행사 데이터 + 과거 매출 → 일별 품목별 적정 입고량 자동 추천 → 폐기 손실 최소화 → "비 오는 수요일 = 국거리용 고기 30% 감소, 찌개 재료 20% 증가" 패턴 자동 학습.

**온라인 노출 자동화**: 네이버 플레이스 자동 업데이트(오늘의 추천 상품, 가격) → 인스타그램 자동 포스팅(입고된 신선 식재료 사진) → 블로그 후기 자동 수집+답변.

**동네 배달 자동화**: 카카오톡 채널로 주문 접수 → 시간대별 배달 자동 묶음 처리 → "오후 2시 배달: 5건 묶음" → 배달 경로 자동 최적화 → 배달 완료 알림.
